# v1.0.4 OAuth silent SSO 会话归档

## 会话目标

推进 Feishu IAM 下一版本，支持第三方应用在 Base Portal iframe 场景下安全使用 OAuth silent SSO。

## 用户关键要求

- Base Portal 生产地址为 `https://base-portal.riversoft.com.cn`。
- SSO Demo 生产地址为 `https://feishu-iam-sso-demo.riversoft.com.cn/`。
- Base Portal 不传 token、cookie、authorization code 或 secret。
- Feishu IAM 不要求 Base Portal 代理第三方鉴权。
- `prompt=none` 已登录时直接签发 code，未登录或需要交互时回跳稳定错误，不在 iframe 中渲染 IAM 登录页。
- 不降低 `redirect_uri` 精确匹配、client、scope、code TTL 和一次性使用校验。
- 不读取或输出真实 secret、token、cookie、authorization code 或 env 值。

## 关键设计决策

- 新增 OAuth browser session，不复用管理员 session。
- browser session 只保存 hash、飞书用户和过期/撤销状态，不保存明文 cookie。
- 应用新增 `silent_sso_enabled` 和 `silent_sso_allowed_origins`，默认关闭。
- 迁移为 `feishu-iam-sso-demo` 预置 `https://feishu-iam-sso-demo.riversoft.com.cn`。
- 普通 OAuth 登录成功后设置 httpOnly、Secure cookie。
- `prompt=none` 路径只使用 302，不输出 iframe HTML 登录页。
- OAuth HTML 错误页设置 `X-Frame-Options: DENY` 和 `Content-Security-Policy: frame-ancestors 'none'`。
