## Findings

### P2 - `oauth.authorize` 失败分支缺少审计

影响：版本完成标准要求 authorize 成功/失败可回溯。原实现只记录无登录失败和成功，invalid client/redirect、disabled application 会返回错误但不会写入 `audit_logs`。

处理：

- 在 `server/src/modules/oauth/oauthRoutes.ts` 为 invalid client/redirect 补 `oauth.authorize` failure audit。
- 为 disabled application 补 `oauth.authorize` failure audit。
- 在 `server/tests/oauth.test.ts` 增加 invalid redirect failure audit 断言。

复验：

- `TEST_DATABASE_URL=postgres://postgres:postgres@127.0.0.1:55432/feishu_iam_test npm run server:test -- server/tests/oauth.test.ts`
