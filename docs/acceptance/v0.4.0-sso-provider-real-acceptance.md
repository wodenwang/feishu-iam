# v0.4.0 SSO Provider 真实验收流程

本文档用于在本机 Docker 环境中验收 Feishu IAM `v0.4.0` SSO Provider 最小闭环。验收目标是确认一个真实浏览器用户可以经 Feishu IAM 跳转飞书登录，第三方应用可以换取 Feishu IAM access token，并读取用户信息和权限清单。

## 环境状态

- Feishu IAM API：`http://localhost:3000`
- 第三方回调捕获服务：`http://localhost:3999`
- Feishu IAM 飞书 OAuth 回调地址：`http://localhost:3000/oauth/feishu/callback`
- 本机敏感配置文件：`.env.local`、`.env.acceptance.local`

`.env.acceptance.local` 被 `.gitignore` 覆盖，只允许保存在本机，不进入仓库、文档、截图或聊天消息。

## 飞书后台前置配置

在飞书开放平台的 Feishu IAM 企业自建应用中，确认 OAuth 重定向 URL 包含：

```text
http://localhost:3000/oauth/feishu/callback
```

如果飞书后台不允许 `localhost` 回调，需要先改用可从浏览器访问并已加入白名单的内网域名或临时 HTTPS 隧道，再同步更新：

- Docker 启动环境变量 `FEISHU_OAUTH_REDIRECT_URI`
- `.env.acceptance.local` 中的 `FEISHU_IAM_BASE_URL` 和授权入口
- Feishu IAM 后台登记的第三方应用 `redirect_uri`

## 启动和健康检查

从仓库根目录执行：

```bash
set -a
source .env.local
set +a
export FEISHU_OAUTH_REDIRECT_URI="${FEISHU_OAUTH_REDIRECT_URI:-http://localhost:3000/oauth/feishu/callback}"
pnpm compose:down
pnpm compose:up
```

检查服务状态：

```bash
docker compose -f deploy/docker-compose.yml ps
curl -sS http://localhost:3000/health
curl -sS http://localhost:3000/ready
curl -sS http://localhost:3000/version
```

通过标准：

- `postgres` 为 `healthy`。
- `api` 为 `Up`。
- `/health` 返回 `status=ok`。
- `/ready` 返回 `status=ready`。
- `/version` 返回 `version=0.4.0`。

## 验收数据准备

通过平台 API 准备一个专用验收应用：

- app key：由准备脚本生成，例如 `ssoaccept0516xxxx`。
- 环境：`dev`。
- 第三方回调地址：`http://localhost:3999/callback`。
- client：只在创建时返回一次 `client_secret`。
- 权限数据：一个权限组、两个权限点、一个 IAM 角色，并把角色绑定到一个真实可登录飞书用户。

准备完成后，本机生成 `.env.acceptance.local`，包含验收应用、client、授权入口和平台 token。不要提交或展示该文件内容。

回调捕获服务会在成功获取 `/oauth/userinfo` 后，把验收 IAM 角色临时绑定到当前登录飞书用户，然后再查询 `/api/v1/apps/{app_key}/me/permissions`。这样可以避免预置用户与实际浏览器登录用户不一致导致权限为空；该绑定会通过平台 API 写入审计日志。

## 浏览器主流程验收

启动第三方回调捕获服务：

```bash
node tools/acceptance/sso-callback-server.mjs
```

当前本机验收环境已通过 `launchctl` 托管该服务，查看或停止命令如下：

```bash
launchctl list | grep feishu-iam-sso-callback
launchctl remove feishu-iam-sso-callback
```

打开：

```text
http://localhost:3999
```

点击“开始 SSO 验收登录”，按浏览器提示完成飞书登录。成功后，页面必须展示：

- token 响应，且 `access_token` 已被页面打码。
- `/oauth/userinfo` 返回当前飞书用户。
- 验收 IAM 角色已临时绑定到当前登录用户。
- `/api/v1/apps/{app_key}/me/permissions` 返回当前应用权限组、权限点和匹配角色。
- `/oauth/revoke` 返回撤销成功。

通过标准：

- 浏览器经历 Feishu IAM -> 飞书 -> Feishu IAM -> 第三方回调服务的完整跳转。
- 第三方回调收到的 `state` 与本地配置一致。
- 授权码可以换取 token。
- 同一个 token 可以读取 userinfo 和当前应用权限。
- 页面不展示明文 `client_secret` 或 access token。
- token 撤销后不能继续访问 userinfo 或权限接口。

## 负向验收

### 1. redirect_uri 精确匹配

把 authorize URL 中的 `redirect_uri` 临时改成未登记地址，例如：

```text
http://localhost:3999/callback-bad
```

通过标准：

- Feishu IAM 展示统一错误页。
- 页面包含 request id。
- 不展示裸 JSON、堆栈或框架默认错误。
- `security_events` 记录 `OAUTH_REDIRECT_URI_UNTRUSTED`。

### 2. state 必填

删除 authorize URL 中的 `state`。

通过标准：

- Feishu IAM 展示统一错误页。
- `security_events` 记录 `OAUTH_STATE_REQUIRED`。

### 3. 授权码只能使用一次

主流程成功后，使用同一个 `code` 再调用一次 `/oauth/token`。

通过标准：

- 返回 `OAUTH_CODE_USED` 或 `OAUTH_CODE_INVALID`。
- 不生成新的 access token。

### 4. client_secret 错误

用错误 `client_secret` 调用 `/oauth/token` 或 `/oauth/revoke`。

通过标准：

- 返回 `OAUTH_CLIENT_CREDENTIALS_INVALID`。
- `security_events` 记录失败事件。

### 5. token 撤销后不可用

主流程页面已经调用 `/oauth/revoke`。用撤销后的 token 再调用：

```bash
curl -sS http://localhost:3000/oauth/userinfo \
  -H "Authorization: Bearer <revoked_access_token>"
```

通过标准：

- 返回 `OAUTH_TOKEN_REVOKED`。

## 数据库证据

查看安全事件：

```bash
docker compose -f deploy/docker-compose.yml exec -T postgres \
  psql -U feishu_iam -d feishu_iam \
  -c "select event_type,result,reason_code,summary,created_at from security_events order by created_at desc limit 20;"
```

查看审计日志：

```bash
docker compose -f deploy/docker-compose.yml exec -T postgres \
  psql -U feishu_iam -d feishu_iam \
  -c "select resource_type,action,result,created_at from audit_logs order by created_at desc limit 20;"
```

通过标准：

- 登录成功、token exchange、token revoke 有安全事件。
- 回调地址、client、角色和权限配置有审计日志。
- 失败路径不会因为审计写入失败而掩盖原始业务错误。
