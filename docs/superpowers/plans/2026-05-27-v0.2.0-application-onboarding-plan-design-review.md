# v0.2.0 应用接入生产化 Plan Design Review

日期：2026-05-27

## 结论

通过，但需要补 v0.2 Pencil 原型后再进入工程评审。

`v0.2.0` 的产品方向成立：把 v0.1 已跑通的 demo 接入链路升级为可长期维护的应用接入配置面。现有 `design/pencil-input-v0.1.0.md`、`DESIGN.md` 和已实现的应用详情页可以复用信息架构基线，但不足以直接支撑 v0.2 实现，因为 redirect URI CRUD、secret reset/rotate 的一次性明文展示、多应用管理员维护、配置完整性状态和审计回溯入口还没有被画成明确的页面、Drawer、Modal 和状态。

本次设计完整度初始评分：6/10。

如果按本文补齐 v0.2 原型和状态说明，可进入工程评审，目标评分为 8/10。未生成新 mockup，因为当前工作区 `gstack designer` binary 不可用。

## NOT in scope

- `/directory` 编辑、导入、导出或目录治理：v0.2 聚焦应用接入配置，不扩大组织数据治理面。
- 飞书 webhook、event sync、incremental sync、同步告警：属于同步可靠性版本，不混入应用配置生产化。
- 完整 OIDC discovery、JWKS、PKCE、refresh token、复杂 consent：属于后续协议产品化，不进入第一版应用配置维护。
- SDK、CLI、Helm、Terraform、install/upgrade 产品化：属于开发者体验和部署产品化，不进入本版本。
- 多租户 SaaS IdP：当前项目仍是单专用飞书应用边界。
- username/password、本地 root、本地超级管理员：继续违反项目硬边界，禁止进入设计。

## Review Passes

### Pass 1：Information Architecture

评分：7/10。

现有 Tabs 方向正确，但 v0.2 需要重新组织 `应用详情` 的信息优先级：

- `概览` 应展示配置完整性摘要，而不是只展示基础字段。
- `接入配置` 应拆成 OAuth 配置、Application API 配置和安全状态。
- `应用管理员` 应从只读 Descriptions 升级为可维护 Table。
- `审计记录` 应支持配置变更动作的快速筛选。

建议 v0.2 Pencil 原型保留一个主页面：`/applications/:id`。不要新增独立“配置中心”菜单，否则会把应用上下文拆散。

### Pass 2：Interaction State Coverage

评分：5/10。

缺口集中在危险操作和配置维护：

- redirect URI 添加、停用、恢复、删除的 loading、校验失败、重复 URI、无权限和空状态未定义。
- secret rotate/reset 后的一次性明文展示、关闭后不可再见、复制审计、失败回滚未定义。
- 多应用管理员添加、移除、最后一个管理员保护、用户不存在、跨应用无权限未定义。
- 配置完整性状态缺少 `未配置 / 部分完成 / 可接入 / 异常` 的展示规则。

这些状态必须在 Pencil 输入中补齐，否则实现阶段会自由发挥。

### Pass 3：User Journey

评分：7/10。

主路径应固定为：

```text
应用列表 -> 应用详情 -> 查看配置完整性 -> 配置 redirect URI -> 复制或轮换 secret -> 维护应用管理员 -> 查看审计 -> 运行 v0.2 验收脚本
```

用户的关键心理负担是“不知道这个应用是否已经可以安全接入”。因此页面第一屏应回答：

- OAuth 回调是否配置完整。
- Application API secret 是否已签发。
- appSecret/apiSecret 是否最近轮换。
- 是否有至少一个应用管理员。
- 最近是否有失败的 OAuth/token/API 调用。

### Pass 4：AI Slop Risk

评分：8/10。

风险可控，因为本项目已有明确后台设计系统和 Ant Design 组件映射。需要避免两个常见退化：

- 不要把配置状态做成装饰性大卡片 Dashboard。
- 不要把 redirect URI 和应用管理员维护塞进一个过长表单。

推荐使用紧凑的 `Descriptions` + `Table` + `Drawer/Form` + `Modal`，沿用当前 Admin Console 密度。

### Pass 5：Design System Alignment

评分：8/10。

与 `DESIGN.md` 对齐。推荐组件：

| 交互 | Ant Design 组件 |
|---|---|
| 配置完整性摘要 | `Descriptions` / 小型 `Card` / `Tag` |
| redirect URI 列表 | `Table` |
| 新增 redirect URI | `Drawer + Form` |
| 停用/恢复 URI | `Popconfirm` |
| secret rotate/reset | `Modal.confirm` + 一次性结果 Modal |
| 多应用管理员 | `Table` + `Drawer + Select` |
| 配置变更审计 | `Table` + action filter |

### Pass 6：Responsive & Accessibility

评分：6/10。

现有页面有 1440/1280/768 验证传统，但 v0.2 需要补以下规则：

- 768px 下 Tabs 不应导致危险操作按钮挤压或换行失控。
- redirect URI 长 URL 必须等宽、可复制、可省略展示，不能撑破表格。
- secret 一次性明文 Modal 必须支持键盘复制、清晰关闭确认和不可恢复提示。
- 所有危险操作按钮需要明确文本，不只用图标。

### Pass 7：Unresolved Design Decisions

需要在 Pencil 前确认：

1. secret 操作命名：统一用 `轮换` 还是 `重置`。建议 UI 用 `轮换 secret`，说明新 secret 生效后旧 secret 立即失效。
2. redirect URI 删除策略：建议第一版只做 `停用 / 恢复`，不做硬删除，便于审计回溯。
3. 应用管理员最后一人保护：建议不允许移除最后一个应用管理员，除非平台管理员先添加替代管理员。
4. 配置完整性状态是否影响应用启用：建议 v0.2 只提示，不自动停用应用，避免误伤已有接入。

## Required v0.2 Pencil Coverage

现有 v0.1 Pencil 原型不足。v0.2 应补一份 `design/pencil-input-v0.2.0.md` 和对应 `.pen`，至少覆盖：

1. 应用详情概览第一屏：配置完整性摘要、风险提示、最近调用、最近审计。
2. 接入配置 Tab：OAuth redirect URI Table、Application API 状态、secret 状态。
3. 新增 redirect URI Drawer：URL 输入、环境标签、校验错误、重复错误。
4. redirect URI 停用/恢复 Popconfirm。
5. secret 轮换确认 Modal：影响说明、二次确认、操作 loading。
6. secret 轮换结果 Modal：只显示一次明文、复制按钮、安全提示、关闭后不可恢复。
7. 应用管理员 Tab：管理员 Table、新增管理员 Drawer、移除确认、最后一人保护错误。
8. 审计记录 Tab：配置变更 action filter 和详情 Drawer 脱敏展示。
9. 768px 视口下长 URL、Tabs、危险操作区的布局。
