## 目标

`v0.1.9` 是 `v0.1.8` 真实飞书登录后的部署补丁，目标是避免 macOS AppleDouble 元数据文件和其他非编号 SQL 文件被 migration runner 当成数据库迁移执行。

本版本只修复部署稳定性，不新增 IAM 业务能力，不改变飞书登录、session、权限或审计模型。

## CI 状态

- `codex/v0.1.9-migration-file-filter` 分支 CI：通过。
- `main@1aa7254` CI：失败。
- 失败原因：`src/pages/Roles/index.test.tsx` 中授权变更摘要用例显式设置 `10_000ms` timeout，在 GitHub runner 上超过 10 秒。
- 本地复现：`npm test -- src/pages/Roles/index.test.tsx` 通过，`9 tests passed`，耗时约 13 秒。
- 收口修复：将该慢用例显式 timeout 提升到 `20_000ms`，避免 main CI 被环境性能波动阻塞。
