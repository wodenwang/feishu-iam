# v0.1.15 Feishu Directory Sync Runtime 工程评审

## 结论

通过。`v0.1.15` 只交付飞书通讯录同步 runtime 的最小闭环：平台管理员手动触发一次 full sync，后端用专用自建飞书应用读取部门和用户，幂等写入本地目录投影，并让现有 `/sync` 页面在 HTTP mode 中显示同步历史、结果和失败原因。

本版本不重开 `/directory` CRUD，不做增量同步、定时任务、事件订阅、复杂删除策略、OAuth/OIDC 增强或应用管理员维护 UI。

## 范围锁定

In scope:

- 新增 `sync_runs` runtime 表，记录 trigger、status、operator、requestId、diff summary、错误和耗时。
- 新增 Feishu directory sync adapter 边界，生产使用 tenant access token 调用飞书通讯录 API，测试使用 fake adapter。
- `POST /api/sync/runs` 由平台管理员触发 full sync。
- `GET /api/sync/runs` 返回分页同步历史。
- 同步数据幂等 upsert 到 `feishu_users`、`directory_departments`、`directory_users`。
- 不在仓库保存飞书凭证、tenant token、导出用户列表或同步快照。
- 同步开始、成功、失败写入审计。
- 前端 HTTP mode 接通现有 `SyncPage` 的列表、手动同步和重试。
- README、CHANGELOG、VERSION 更新到 `v0.1.15`。

Out of scope:

- 增量同步、定时同步、事件订阅。
- 全字段通讯录同步、手机号等敏感字段兜底采集。
- 完整删除策略；本版只把本次缺失的既有用户标记为 `resigned`。
- `/directory` 编辑能力或目录权限扩大。
- 应用管理员触发同步；同步仍是平台管理员能力。
- OAuth/OIDC/redirect URI/secret rotation。

## 架构决策

### 1. Adapter 边界隔离飞书 API

`server/src/modules/sync/directorySyncAdapter.ts` 定义内部 snapshot shape。真实飞书实现只负责把官方接口响应映射为部门和用户快照，业务写库、diff、审计都在 sync service 中完成。

### 2. Full sync 先闭环，增量留后续版本

v0.1 需要一个可信的通讯录投影来源。Full sync 能证明 IAM 从飞书读取组织/用户并服务授权链路。增量、事件订阅和调度器属于后续运维产品化。

### 3. 后端权限只允许平台管理员同步

应用管理员可以管理自己应用下角色，但不能触发全局通讯录同步。`/api/sync/runs` 读取和写入都要求 platform admin。

### 4. 不持久化 token 和原始快照

tenant access token 只在内存中用于本次请求，不写数据库、日志或审计 metadata。同步结果只保存最小目录投影和统计。

## 风险与控制

| 风险 | 控制 |
|---|---|
| 飞书 API 权限不足导致同步失败 | 同步 run 标记 failed，保存错误摘要，审计 `sync.run` failure。 |
| 同步把已有用户误删 | 本版不删除用户；只在本次快照明确缺失时标记 `resigned`。 |
| 敏感字段泄露 | 不保存 token、secret、原始 payload；手机号字段只在飞书返回且前端已有脱敏展示时映射。 |
| 并发重复触发 | v0.1.15 不引入队列；运行中检查后拒绝新同步。 |
| 前端 HTTP mode 仍调用 unsupported method | 补齐 `listSyncRuns`、`startManualSync`、`retrySyncRun` 和 DTO mapper。 |

## 测试策略

必须覆盖：

- 平台管理员可触发同步并写入部门、用户和 `sync_runs`。
- 同步重复执行时幂等更新而不是重复插入。
- 本次缺失的既有用户标记为 `resigned`。
- 普通用户和应用管理员不能触发或查看同步。
- adapter 抛错时 run 标记 failed 并写失败审计。
- migration 包含 `006_sync_runs`。
- 前端 HTTP API 映射同步列表和触发动作。

## 完成标准

- `/api/sync/runs` 支持 list、manual run、retry run。
- 同步结果能驱动 `/directory` 页面看到真实 HTTP 投影。
- 同步成功、失败、重试均可在 `/sync` 页面和审计日志中追踪。
- 自动化测试、构建、浏览器验证、QA、review 通过。
- tag `v0.1.15`、GitHub Release、远端 Docker Compose runtime 和 `/api/health` 验证完成。
