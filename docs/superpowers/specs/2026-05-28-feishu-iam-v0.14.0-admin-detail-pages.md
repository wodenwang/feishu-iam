# Feishu IAM v0.14.0 后台导航补齐与重交互详情页规格

日期：2026-05-28
状态：设计和工程评审前规格

## 1. 背景

`v0.13.1` 已完成系统管理一级分组和二级入口的基本层级表达，但最新 GitLab issue 反馈显示后台仍有两个问题需要一起收口：

1. `系统管理` 父级交互和树形层级仍不够明确，二级菜单缺少图标。
2. 应用详情和角色详情都已经承载多 Tab、多表单、多列表和保存确认等重交互，不应继续作为右侧抽屉默认入口。

因此 `v0.14.0` 不再拆 `v0.13.2` 小补丁，而是把 `#16/#6/#7/#9` 合并为一个后台体验版本：先补齐系统管理导航，再把应用详情和角色详情升级为独立详情页。

## 2. 输入来源

- `AGENTS.md`：当前阶段、Admin UI 开发规范、Browser 自检硬约束。
- `DESIGN.md`：shadcn/ui + tweakcn + Tailwind 后台设计基线，复杂配置优先 full page 或 Tab 化详情。
- GitLab issue `#16`：系统管理父级展开收起、二级菜单图标和树形层级。
- GitLab issue `#6`：应用管理提示词复制、编辑、停用、状态展示，以及 2026-05-27 补充的独立应用详情页要求。
- GitLab issue `#7`：角色详情组织树、左待选右已选、权限组绑定，以及 2026-05-27 补充的独立角色详情页要求。
- GitLab issue `#9`：角色元数据归应用管理，权限管理聚焦授权绑定。
- `design/v0.11.2-application-ops-prototype.md` 和 `design/v0.11.3-role-authorization-prototype.md`：既有应用/角色工作区原型基线。
- `docs/superpowers/specs/2026-05-28-feishu-iam-v0.13.1-sidebar-nav.md`：系统管理导航当前基线。

## 3. 版本号

推荐版本号：`v0.14.0`。

原因：

- `#16` 独立成 `v0.13.2` 范围过窄，且导航补齐会影响后续详情页的整体后台壳层体验。
- `#6/#7/#9` 的新反馈已经从局部抽屉调整升级为页面级交互承载重构，属于新版本范围。
- 本版本仍不改变后端权限模型、SSO 协议或部署拓扑。

## 4. 目标

交付一个更符合后台系统使用方式的管理端：

- `系统管理` 菜单像树一样可理解、可展开收起、二级入口有图标。
- 应用详情从右侧抽屉升级为可深链的独立详情页。
- 角色详情从右侧抽屉升级为可深链的独立详情页。
- 从列表进入详情、切换 Tab、刷新页面和返回列表时，筛选上下文尽量保留。
- 应用管理继续承载角色元数据生命周期；权限管理只承载授权绑定。

## 5. 纳入范围

### 5.1 系统管理导航补齐

- 点击 `系统管理` 父级整行执行展开 / 收起，不再只靠旁边小 chevron。
- 当前处于系统管理二级页时，父级保持当前分组标识，展开状态可理解。
- `飞书同步`、`管理员授权`、`操作审计`、`系统信息` 二级菜单补充合适图标。
- 二级菜单相对父级有稳定缩进、层级线或等价视觉线索。
- 桌面收缩态和移动端 Sheet 保持可访问、可理解。
- 前端测试覆盖父级展开收起、二级图标渲染和当前二级页状态。

### 5.2 应用详情独立页

- 应用列表 `详情` 进入 `/admin/applications/:appKey` 独立详情页。
- 详情页顶部提供返回应用列表按钮，返回时尽量保留搜索、状态、分页和排序。
- 详情页按独立页面承载应用名称、基础信息、回调地址、OAuth 凭证、Developer credential、接入提示词、应用角色和状态操作。
- 支持刷新后仍定位到当前应用。
- 创建应用成功后的接入提示词仍保留一键复制能力，不记录明文 secret、token 或 cookie。
- 角色元数据新增、编辑和启停继续归应用详情页。

### 5.3 角色详情独立页

- 权限管理角色列表 `详情` 进入 `/admin/permissions/:appKey/roles/:roleId` 独立详情页。
- 详情页顶部提供返回权限管理列表按钮，返回时尽量保留应用筛选、角色查询、状态、分页和排序。
- 详情页使用 Tab 组织：
  - 总览
  - 组织与用户绑定
  - 权限组绑定
  - 基础信息
  - 操作说明
- 刷新后仍能定位到对应角色和当前 Tab。
- 组织与用户绑定继续保持左待选、右已选、组织/用户分区和保存前变更摘要。
- 权限组绑定继续在同一详情页内完成。

## 6. 不纳入范围

- 不恢复权限管理里的角色新增、编辑、启停。
- 不把权限点 CRUD 暴露为后台主流程。
- 不新增后端 DDL，除非实施前确认现有接口无法支撑已登记验收。
- 不实现完整 OIDC、SAML、ABAC、资源级权限、飞书角色同步、飞书用户组同步。
- 不新增 HTTPS、反向代理、高可用或滚动升级。
- 不改变生产管理员 session、管理员权限校验或破窗登录边界。

## 7. URL 和状态约定

### 7.1 应用管理

- 列表页：`/admin/applications?q=&status=&page=&pageSize=&sort=`
- 应用详情页：`/admin/applications/:appKey?tab=overview`
- 允许的应用详情 Tab：
  - `overview`
  - `basic`
  - `redirects`
  - `credentials`
  - `prompt`
  - `roles`
  - `danger`
- 从列表进入详情时，把当前列表查询串作为 `from` 参数保存，或使用等价方式保留返回上下文。

### 7.2 权限管理

- 列表页：`/admin/permissions?appKey=&q=&status=&page=&pageSize=&sort=`
- 角色详情页：`/admin/permissions/:appKey/roles/:roleId?tab=subjects`
- 允许的角色详情 Tab：
  - `overview`
  - `subjects`
  - `groups`
  - `base`
  - `audit`
- 从列表进入详情时，保留应用筛选、角色查询、状态、分页和排序上下文。

## 8. 验收标准

- `系统管理` 父级整行可展开 / 收起，二级菜单有图标和清楚树形层级。
- `/admin/system/audit` 刷新后父级展开，`操作审计` 高亮，收缩态和移动端无错位。
- 应用列表点击 `详情` 后进入独立详情页，不打开右侧抽屉。
- 应用详情刷新后仍在对应应用；返回列表后保留查询上下文。
- 应用详情中基础信息、回调地址、接入提示词、角色元数据和状态操作可用。
- 权限管理角色列表点击 `详情` 后进入独立详情页，不打开右侧抽屉。
- 角色详情刷新后仍在对应角色和 Tab；返回列表后保留筛选上下文。
- 角色详情中组织与用户绑定、权限组绑定、保存前摘要可用。
- 权限管理首页不出现角色元数据新增、编辑或启停入口。
- 浏览器 console 无新增错误，Network 无非预期失败请求。
- `pnpm check`、前端定向测试、构建、响应式检查和 Browser 自检通过。
