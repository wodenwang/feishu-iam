# Feishu IAM v1.0.0 Riversoft 正式版线上 canary 观察

## 会话目标

使用 `gstack /canary` 对 Feishu IAM v1.0.0 Riversoft 正式版完成发布后线上观察，确认公网入口、后台未登录统一问题提示页、飞书登录入口、管理员初始化、敏感凭证边界和容器健康状态。

## 用户关键要求摘要

- 线上入口：`https://feishu-iam.riversoft.com.cn/`。
- 部署位置：`bpmt@120.24.236.92:/home/bpmt/feishu-iam`。
- GitHub Release：`https://github.com/wodenwang/feishu-iam/releases/tag/v1.0.0`。
- 重点观察 `/ready`、`/version`、后台未登录统一问题提示页、飞书登录入口、王文哲平台管理员权限、应用详情不暴露敏感凭证、request id 追踪入口和容器健康状态。
- 不输出或记录真实 secret、token、cookie、飞书 app_secret、数据库密码或生产导出数据。

## 本次使用的关键约束

- v1.0.0 是内部正式版发布收口。
- 只能做发布后观察，不扩大功能范围。
- 数据初始化只处理 IAM 管理权限，确保“王文哲”成为平台管理员。
- 不初始化第三方应用权限、demo client、回调地址或业务角色。

## 观察方式

- 使用 Playwright 对公网入口执行三轮浏览器观察。
- 视口覆盖：
  - 1440px：`/admin`
  - 768px：`/admin/applications/crm?from=/admin/applications&tab=development`
  - 390px：`/admin/system/audit?tab=trace&requestId=req_prod_v100_canary`
- 使用 SSH 在部署目录执行只读容器状态和数据库边界查询。

## 关键结果

- `/ready` 返回 `200`，状态为 ready，数据库检查为 ok。
- `/version` 返回 `200`，版本为 `1.0.0`。
- 后台未登录页面展示统一问题提示页，包含错误码、request id、复制 request id 和飞书登录入口。
- 正式飞书登录入口为 `/admin/auth/login`，线上返回 `302` 并跳转飞书账号域名。
- 1440、768、390 三个视口未发现横向溢出、旧品牌文案、敏感凭证文本或非预期 console error。未登录态接口的 `401 Unauthorized` 属于预期行为。
- 应用详情深链在未登录态没有暴露应用详情数据或敏感凭证；线上初始化边界同时确认没有初始化应用、client、回调地址或业务角色数据。
- 远端容器状态：
  - `feishu-iam-db-1`：running，healthy。
  - `feishu-iam-web-1`：running。
- 管理员初始化边界：
  - “王文哲”为 active 管理员，并拥有 `platform_admin`。
  - `applications | application_clients | application_redirect_uris | iam_roles` 计数为 `0|0|0|0`。
- v1.0.0 迁移记录存在：`Riversoft 正式版 UI 翻新与平台管理员初始化`。
