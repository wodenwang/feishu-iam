# 2026-06-13 13:40 v1.0.2 前端 UI/UX 审计与迭代方案

## 会话目标

基于线上服务 `https://feishu-iam.riversoft.com.cn/` 审视当前前端 UI/UX 的优化项，形成下一版本迭代方案，并补强 `DESIGN.md` 等设计文档。

## 用户关键要求

- 使用 Product Design 插件审视线上前端 UI/UX。
- 输出下一版本迭代方案。
- 可以优化 `DESIGN.md` 等设计文档。
- 不改变 shadcn 的前端技术架构。
- UI 和调色方案不设限制，配色可以按需大改。

## 本次使用或形成的关键约束

- 继续遵守项目 `AGENTS.md`：中文文档、shadcn/ui + tweakcn + Tailwind 基线、公开错误页不得暴露敏感信息。
- 线上审计只保存不含登录二维码、token、cookie、授权码和 `state` 的截图。
- 登录跳转进入飞书扫码页，截图可能包含一次性登录二维码和 `state`，已删除该截图，不纳入仓库证据。

## 重要设计决策

- 将下一版本命名为 `v1.0.2` 前端 UI/UX 迭代规划。
- 用户指出不能只审首页后，已从首页点击 `飞书登录` 并进入真实后台控制台，审计范围扩展到完整控制台。
- 首个优先切片调整为移动端列表和 Tab 修复：390px 应用管理列表表格被压碎，操作审计 Tab 溢出。
- 公开问题页仍保留为下一版 P0：统一 OAuth 错误页和后台问题提示页，并移除 `复制问题信息`。
- 允许下一版本大幅调整配色，但所有变化必须通过 CSS variables / Tailwind token 收口，不在业务页面硬编码。
- 登录页截图可能包含一次性二维码和 `state`，已删除未入库。
