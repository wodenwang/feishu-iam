# Codex 会话归档：v1.0.0 Riversoft UI 翻新与管理员初始化第一切片

## 会话目标

执行 `IMPLEMENTATION_PLAN.md` 中的第一个 vertical slice：Riversoft token、项目级组件、三个证明页面、v1.0.0 管理员初始化和验证证据。

## 用户关键要求摘要

- 只实现 D2-A：Token + 项目级组件 + 3 个证明页面 + 管理员初始化。
- 不改变 UX 流程、信息架构、路由、表单流程、数据契约、CRUD 行为、权限逻辑或业务逻辑。
- Riversoft token 必须集中落到 shadcn/ui + tweakcn + Tailwind CSS variables，不允许页面内散落硬编码颜色。
- 数据初始化只确保“王文哲”获得 IAM 平台管理员权限，不初始化第三方应用权限、demo client、回调地址或业务角色。
- 不提交真实 secret、token、cookie、飞书 app_secret、数据库密码或生产导出数据。
- 执行完成后必须运行测试、敏感信息扫描和 Browser/Playwright 自检。

## 本次关键约束

- 保留现有 AppShell、路由、Tab、表格、详情页、弹窗、确认框、分页和后端数据契约。
- 应用详情页安全表达改为“OAuth 凭证 / 凭证状态 / 最近轮换”，不在可见 UI 展示敏感字段名或真实值。
- v1.0.0 初始化迁移只做 `platform_admin` 幂等授予，不撤销其他管理员。
- 本次 Browser 目检使用项目响应式脚本中的 API mock；直接启动完整 API 因本地缺少加密 key 无法完成，不使用真实密钥绕过。

## 重要设计和工程决策

- Riversoft 主题通过 `apps/admin-web/src/index.css` 的 CSS variables 落地，并在 `apps/admin-web/src/theme.ts` 中保留集中常量。
- 语义状态色通过 `--status-success`、`--status-warning`、`--status-info` 等变量和 `StatusBadge` tone 收敛。
- `DataTable`、`PageState`、`AppShell`、`DetailSheet`、`ConfirmDialog`、`ProblemFeedbackPage`、`TraceResultPanel` 作为项目级组件优先升级。
- 版本元数据同步到 `1.0.0`，并修复 `/version` fallback 测试从旧 `0.16.1-dev` 到 `1.0.0-dev` 的不一致。
