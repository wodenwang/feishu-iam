# Feishu IAM v0.13.0 飞书同步运维控制台原型说明

## 目标

本原型用于进入 `v0.13.0` 飞书同步运维控制台的后续设计评审和工程评审。范围锁定 GitLab issue `#13`：让管理员能在后台查看飞书本地镜像、定位同步问题，并执行用户级和部门级轻量同步；生产验收默认不触发真实全量同步。

## 交互决策

- 默认信息架构为紧凑同步总览 + `组织与用户` / `同步历史` / `字段诊断` / `高级操作` 四标签。
- 主旅程是排障定位，不把“触发同步”作为第一视觉中心。
- 本地查询采用左侧部门树和右侧详情工作区，避免默认三栏挤压。
- 用户级和部门级轻量同步在本版本作为真实可用能力呈现，并要求写入审计。
- 全量同步放入高级操作，必须经过强确认；生产验收默认只验证预确认路径。
- 安全错误展示只暴露稳定中文说明、错误码和 request id，不展示 secret、token、cookie、堆栈或飞书敏感原始响应。
- 窄屏不横向塞桌面表格，采用单列钻取并保留返回后的上下文。

## 后续评审重点

- 工程评审已锁定接口契约：健康摘要、部门树、用户详情、历史详情、字段诊断、轻量同步、全量同步预确认和审计写入。

## 原型复评修正记录

本轮按 plan-design-review 维度复查后，已修正以下进入工程评审前必须解决的问题：

- 将同步健康条中的 `全量同步` 日常入口改为 `高级操作`，避免重操作成为全局默认动作。
- 同步历史列表补齐触发来源、开始结束时间、统计摘要、错误码和筛选提示，避免历史详情只能靠右侧面板理解。
- 同步历史详情补齐权限校验、tenant token、审计写入、稳定错误码和排查建议。
- 字段诊断页补齐轻量同步运行状态要求、权限不足提示、审计字段和无权限边界。
- 全量同步强确认补齐影响范围、输入当前 run id、默认禁用继续按钮和后端二次权限校验说明。

复评结论：当前原型已进入并通过工程评审。后续进入 Superpowers `writing-plans`，实施计划必须覆盖 API 契约、权限矩阵、审计事件结构、运行中互斥、轻量同步真实落地和生产验收不触发真实全量同步的测试边界。
