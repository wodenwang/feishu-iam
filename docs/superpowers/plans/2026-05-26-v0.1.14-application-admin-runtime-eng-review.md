# v0.1.14 Application Admin Runtime 工程评审

## 结论

通过。`v0.1.14` 只交付应用管理员 runtime 最小闭环：平台管理员创建应用时绑定一个飞书用户为应用管理员；该用户登录后只访问自己负责的应用、角色授权、权限注册和应用审计。

本版本不重开 `v0.1.13` OAuth 小收口，不做 redirect URI 管理 UI，不做 Sync runtime，不扩展 `/directory`，不做多人应用管理员维护 UI。

## 范围锁定

In scope:

- 新增 `application_admins` runtime 表。
- `POST /api/applications` 支持可选 `ownerFeishuUserId`。
- 绑定应用管理员时要求目标飞书用户已存在。
- `/api/session/current` 返回 `application_admin`、权限码和 scoped `applicationIds`。
- 应用、角色、审计 API 对 application admin 做后端 scope 校验。
- 前端 HTTP mode 创建应用传递 `ownerFeishuUserId`，应用详情显示真实 owner。
- README、CHANGELOG、VERSION 更新到 `v0.1.14`。

Out of scope:

- OAuth 协议继续增强、OIDC discovery、JWKS、PKCE、refresh token。
- redirect URI 管理 UI。
- Sync runtime、tenant token、通讯录全量同步、事件订阅。
- `/directory` 能力扩展。
- 创建后应用管理员增删改 UI、多人管理员、邀请审批。
- username/password 登录。

## 架构决策

### 1. 使用独立 `application_admins` 表

`applications.created_by_feishu_user_id` 表示创建人，不等同于应用管理员。新增绑定表可以表达应用负责人 scope，并为后续多人应用管理员留入口。

第一版仍只通过创建应用时指定一个主要应用管理员，不做维护 UI。

### 2. Scope 校验必须在后端完成

前端已有 `getScopedApplicationIds`，但只能优化 UX。后端必须在所有相关 Admin API 中判断：

- platform admin：全局允许。
- application admin：仅允许访问绑定到自己的 application。
- 普通 Feishu user：拒绝 Admin API。

### 3. 角色管理按 application scope 收窄

应用管理员可以管理自己负责应用下的角色和授权，这是产品规格里的核心职责。`GET /api/roles/permission-tree` 对应用管理员只返回自己负责应用的权限点。

### 4. 审计读取按 target application 收窄

应用管理员只能读取 `targetType=application&targetId=<owned app>` 的审计日志。不允许读取全局审计，也不允许用空 target 或其他 targetType 绕过。

## 风险与控制

| 风险 | 控制 |
|---|---|
| 跨应用越权 | 所有详情、权限注册、secret copy、角色、审计接口统一走 application scope helper。 |
| 创建孤儿管理员 | `ownerFeishuUserId` 必须存在于 `feishu_users`，否则 400。 |
| 平台管理员行为回退 | platform admin 保持现有全局权限与 `applicationIds: []`。 |
| 前端误以为 HTTP 创建支持 code/callback | v0.1.14 只传 `name` 和 `ownerFeishuUserId`，不引入 redirect URI UI。 |
| 角色 permission tree 泄露其他应用权限点 | application admin 查询时按 owned application ids 过滤。 |

## 测试策略

必须覆盖：

- 创建应用时绑定应用管理员并写 `application.admin.bind` 审计。
- 绑定不存在飞书用户时返回 400。
- 应用管理员登录后 session 返回 scoped role、permissions、applicationIds。
- 应用管理员只能 list/read 自己应用，不能读其他应用。
- 应用管理员只能读自己应用的 permission registrations 和 secret-copy audit。
- 应用管理员只能管理自己应用下的角色和授权。
- 应用管理员不能读取全局审计或其他应用审计。
- platform admin 现有行为不回退。
