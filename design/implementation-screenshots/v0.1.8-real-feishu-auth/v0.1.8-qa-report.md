# v0.1.8 QA Report - Real Feishu Auth

日期：2026-05-24
范围：真实飞书 Admin Console 登录闭环、本地 mock 回归、HTTP runtime 主路径。

## 结论

通过。真实飞书 OAuth route 使用 fake adapter 完成自动化验证；本地 HTTP runtime E2E 主路径继续通过。

## QA 覆盖

- `/api/auth/feishu/start` 设置短期 state cookie 并 redirect 飞书授权地址。
- `/api/auth/feishu/callback` 拒绝缺失 code/state/cookie 和 state mismatch。
- callback 成功后创建 IAM session，写入 `feishu_users` / `directory_users` 最小投影，并写审计。
- 飞书 token/user info 失败时不创建 session，并写失败审计。
- 生产 cookie 模式下 OAuth state cookie 带 `Secure`。
- mock 登录回归通过。
- `/login -> /initialize -> /applications -> /audit-logs` E2E 通过。
- `/roles` 创建、授权、停用 E2E 通过。

## 发现与修复

1. 发现：旧 `v0.1.3` E2E 仍假设 HTTP mode 下 `/roles` 会重定向到 `/applications`。
   修复：删除过期断言。`v0.1.5` 已经实现 `/roles` HTTP runtime，进入 `/roles` 是当前正确行为。

2. 发现：Playwright E2E 会重写旧 `v0.1.5` 截图。
   修复：丢弃本轮验证生成的旧截图改动，只保留 `v0.1.8` 专属截图。
