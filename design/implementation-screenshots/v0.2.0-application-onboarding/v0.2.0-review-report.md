# v0.2.0 应用接入生产化 Pre-Landing Review

日期：2026-05-27

## 结论

通过，可进入 ship 收口。未发现需要阻塞落地的 SQL、安全、权限边界、secret 泄漏或测试缺口。

## 重点检查

| 维度 | 结论 |
|---|---|
| 领域边界 | 保持飞书唯一身份源；没有 username/password、本地 root、多租户 SaaS IdP。 |
| SQL/migration | 新增 008 migration 幂等；并发 migration 测试已补 008 期望。 |
| Secret 安全 | 轮换接口只返回一次 plaintext；审计 metadata 只记录 kind/appKey，不记录 secret。 |
| OAuth 边界 | authorize 查询追加 active redirect URI 限制；disabled URI 无法 authorize。 |
| Application API 边界 | apiSecret 轮换后旧 HMAC 失败，新 HMAC 成功。 |
| 权限边界 | redirect URI、secret、管理员维护均限制 platform admin；application admin 只读自己的应用范围。 |
| 最后管理员保护 | 后端 409 + 前端明确错误文案已验证。 |
| 前端实现 | 使用 Ant Design 组件；未引入其他 UI 框架；保留企业后台表格/Drawer/Modal 模式。 |
| 测试覆盖 | 后端、前端 service、页面结构、验收脚本和浏览器截图均覆盖。 |

## 剩余风险

- 当前自动验收运行在 mock Feishu runtime；真实飞书开放平台的 redirect URI、通讯录权限和白名单仍需人工配置后做生产环境 canary。
- push、PR、tag、release、deploy 仍需用户明确授权。
