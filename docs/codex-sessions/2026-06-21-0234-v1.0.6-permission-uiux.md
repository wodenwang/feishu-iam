# v1.0.6 权限管理 UI/UX 小版本

## 会话目标

规划并实现一个小版本，主要解决 `权限管理` 模块 UI/UX 问题，纳入用户在生产页面标注的两个问题，并尽可能找出其它值得优化的前端内容。

## 用户关键要求

- 使用 Product Design 对应技能调研 `https://feishu-iam.riversoft.com.cn/` 权限管理模块。
- 当前会话两个截图标注必须纳入小版本。
- 其余值得优化的内容尽可能找出。
- 务必策划好前端。
- 使用 my-harness 流程从头到尾实现。

## 关键约束

- 当前会话没有可调用的 Product Design `get-context` / `ideate` 工具，因此使用生产标注、`DESIGN.md`、v1.0.5 定稿规格和 Playwright 三视口审计作为替代设计证据。
- 本版本只处理前端 UI/UX，不新增 DDL，不改变权限模型、权限计算、管理员 session 或审计语义。
- 未经用户授权，不执行 stage、commit、push、release 或部署。

## 设计决策

- `组织与用户` 和 `应用权限` 双栏布局使用起始对齐，避免左侧工作区被右侧内容撑高。
- `应用权限` 当前应用切换改为纵向 tab，展示应用名称、`appKey`、应用状态和绑定状态。
- 角色工作台内部标题改为 `角色上下文`，总览第一块改为 `基础信息概览`，角色名称作为字段展示，避免重复标题。
- `绑定结果预览` 桌面端使用 sticky 右栏、最大高度和内部滚动。
- `权限点对比` 保留在主界面，并增加最大高度和内部滚动，不降级为弹窗。
