# v1.0.3 Base Portal 接入包与完整提示词验收记录

## 版本范围

`v1.0.3` 收口 Base Portal 接入包、完整提示词主流程、secret 按需轮换、接入预检和 iframe 无感 SSO 验收说明。

## 不变边界

- 不新增 DDL。
- 不实现完整 OIDC、refresh token、SAML、ABAC、资源级权限或 deny 规则。
- 不长期明文保存 developer API token。
- 不放宽 `redirect_uri` 精确匹配。
- 不改变管理员 session、权限裁剪、飞书通讯录同步或生产部署拓扑。
- 本验收文档不记录真实 secret、token、cookie、authorization、授权码、hash、ciphertext 或飞书 `app_secret`。

## 功能验收

- 应用详情 `开发信息` Tab 提供接入预检，覆盖启用回调地址、OAuth 登录凭证、Developer API 凭证和 request id 排障边界。
- 管理员通过强确认触发 `刷新凭证并生成完整提示词`，后端在同一个数据库事务中轮换 OAuth `client_secret` 和 developer API token，避免只轮换其中一个凭证后无法返回完整提示词。
- 完整提示词包含第三方 Codex 项目接入所需的 Feishu IAM URL、`app_key`、`client_id`、`client_secret`、`developer_api_token`、回调地址、OAuth 流程、权限查询和 Developer API 边界。
- `base-portal` 应用提示词包含 Portal preset：菜单权限点建议、打开方式约束和 iframe 无感 SSO 验收矩阵。
- developer API token 仍只保存 hash；忘记或变更 token 时，通过再次刷新完整提示词生成新 token。
