# v0.3.0 飞书事件同步 Design Review

## 结论

通过。新增 UI 保持在现有 `/sync` 页面内，使用 Ant Design `Card`、`Statistic`、`Alert`、`Table` 和 `Tag`，符合企业后台高密度、状态可扫读、表格优先的项目规范。

## 检查点

- 信息架构：事件同步作为同步运营的一部分，放在同步任务列表之前，符合“先看健康，再看任务”的工作流。
- 视觉密度：事件计数使用四列统计，桌面和窄屏均可扫读，没有改成卡片网格。
- 状态表达：`待同步`、`失败`、`已处理`、`已忽略` 使用 Tag 呈现；健康状态使用卡片右上角 Tag。
- 响应式：1440、1280、768 截图已覆盖，表格在窄屏使用横向滚动。
- 文案：回调地址明确为 `/api/feishu/events`，未在页面展示 token、secret 或 signature。

## 边界确认

- 未引入新的设计语言或 UI 框架。
- 未改变既有手动同步、预检、同步任务表和详情 Drawer 结构。
- 未新增低密度说明页或营销式布局。
