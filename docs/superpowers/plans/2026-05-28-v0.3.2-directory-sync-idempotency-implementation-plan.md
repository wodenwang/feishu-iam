# v0.3.2 Directory Sync Idempotency Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix GitHub issue #27 so full sync disables stale directory departments, keeps current Feishu departments idempotent by stable ID, preserves legal same-name departments, and updates version metadata to `v0.3.2`.

**Architecture:** Keep the existing `directory_departments.id` primary-key model. `applyDirectorySnapshot()` treats each successful full sync as the current active directory snapshot: upsert snapshot departments, mark active departments outside the snapshot as `disabled`, then upsert users only against snapshot department IDs. Directory read APIs return only active department trees and active-derived user paths.

**Tech Stack:** TypeScript, Fastify, PostgreSQL, Vitest, React/Vite version metadata docs.

- [ ] **Step 2: Add helper below `upsertDepartment()`**

Add this helper:

```ts
async function disableDepartmentsOutsideSnapshot(client: DbClient, departmentIds: Set<string>): Promise<number> {
  const result = await client.query(
    `
      update directory_departments
      set status = 'disabled', updated_at = now()
      where status = 'active'
        and not (id = any($1::text[]))
    `,
    [[...departmentIds]],
  );
  return result.rowCount ?? 0;
}
```

- [ ] **Step 3: Filter user path department tree to active departments**

In the user list `department_tree` CTE, use:

```sql

from directory_departments
where parent_id is null and status = 'active'

       child.parent_id,
       (department_tree.path || ' / ' || child.name)::text as path
from directory_departments child
join department_tree on department_tree.id = child.parent_id
where child.status = 'active'
```
