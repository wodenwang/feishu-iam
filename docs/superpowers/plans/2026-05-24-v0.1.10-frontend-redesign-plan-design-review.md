## System Audit

- 当前仓库版本线已到 `v0.1.9`，`main` 与 `origin/main` 同步。
- `DESIGN.md` 存在，并明确本项目是企业级 Admin Console，不是营销页或创意型 UI。
- `design/feishu-iam-v0.1.0-admin-console.pen` 和历史截图存在，但没有 `v0.1.10` 的新原型。

### 外部参照

- Ant Design Pro 文档建议中后台布局通过整体风格、主题色、导航模式、内容区宽度、固定 Header、固定侧边菜单等配置统一处理；右上角用户信息、头像和退出登录可作为布局右侧内容自定义。

当前计划是 **6/10**。

已有方向明确：Ant Design Pro 化、JoyBell 参考色、logo、登录页、退出登录、抖动修复。缺口是还没有把这些方向转成可实现的信息架构、状态表、响应式规则、具体 token、组件边界和验收截图清单。

- 1440 / 1280 / 768 三档响应式约束。
- 进入 Pencil 原型的页面清单、状态清单和截图验收清单。

## Pass 1: Information Architecture

评分：**5/10 -> 8/10**

### 阻塞问题

1. 当前计划没有定义 Admin Shell 的稳定信息层级，导致 Header、Sider、Breadcrumb、Content 可能继续各自为政。

   修正要求：

   ```text
   AdminShell
   ├── Sider 224px fixed
   │   ├── BrandLogo 56px high
   │   └── Menu fixed item height
   └── Main
       ├── Header 56px fixed
       │   ├── Left: page context / environment tag
       │   └── Right: avatar + username + role + dropdown
       └── Content
           ├── Breadcrumb 32px area
           └── Page content 24px padding
   ```

2. 登录页还没有结构约束。

   修正要求：

   ```text
   LoginShell
   ├── Left panel: brand, product promise, Feishu-only identity statement
   └── Right panel: login card
       ├── BrandLogo
       ├── 使用飞书登录 primary button
       ├── deployment/runtime info
       ├── error/callback state
       └── dev-only mock entry
   ```

### 重要建议

- HTTP runtime 下菜单仍可只显示 `/applications`、`/roles`、`/directory`、`/audit-logs`，但禁用路由跳转应有清晰 fallback 文案，避免用户以为页面丢失。
- Breadcrumb 不应承担页面标题职责。每个页面应有稳定 PageHeader 区域，否则页面切换容易显得跳。

## Pass 2: Interaction State Coverage

评分：**4/10 -> 8/10**

### 阻塞问题

必须为以下功能补状态表，Pencil 原型至少展示正常、加载、错误、无权限四类核心状态。

| Feature | Loading | Empty | Error | Success | Partial |
|---|---|---|---|---|---|
| AdminLayout session | 固定全屏骨架，不改变 Header/Sider 宽度 | 不适用 | Alert + 重新登录 | 显示 Shell | 会话过期但路由保留 |
| Header user menu | Avatar skeleton | 不适用 | 隐藏敏感信息，只保留重试/登录 | 用户名 + 角色 + 下拉 | 长用户名省略 |
| Login page | Callback processing panel | 不适用 | config/auth/access 状态卡 | 飞书登录入口 | DEV mock 入口弱化 |
| Logout | 按钮 loading | 不适用 | message + 不清除页面 | 清 cookie 后跳 `/login` | API 失败仍允许本地跳转需评估 |
| Main tables | 表格骨架或固定高度 Spin | EmptyState + primary action | ErrorState + requestId + Retry | 数据表格 | search empty |

### 重要建议

- 所有 loading 态必须保留容器尺寸，避免页面抖动。
- 表格 `scroll.x`、操作列宽、空状态高度应固定。
- API error 必须继续显示 `requestId`。

## Pass 3: User Journey & Emotional Arc

评分：**5/10 -> 8/10**

### 阻塞问题

当前计划描述了功能，但没有定义用户从登录到后台的信任建立过程。

必须补 storyboard：

| Step | User does | User feels | Design requirement |
|---|---|---|---|
| 1 | 打开 `/login` | 判断这是正式系统还是临时 demo | 品牌、部署、飞书唯一身份源同时出现 |
| 2 | 点击飞书登录 | 担心是否跳错应用 | 按钮文案明确，展示专用自建应用边界 |
| 3 | callback 返回 | 等待身份确认 | callback loading 文案固定，不闪白 |
| 4 | 进入后台 | 需要确认自己是谁、有什么权限 | Header 显示用户、角色、环境 |
| 5 | 退出登录 | 需要可控结束会话 | 用户菜单提供明确退出入口和确认 |

### 重要建议

- 登录页不做 username/password，也不要出现类似“账号登录”的视觉暗示。
- Mock 登录只在开发模式出现，样式降级为次要链接或次按钮，并标注“本地开发”。

## Pass 4: AI Slop Risk

评分：**7/10 -> 9/10**

### 重要问题

- 计划中“参考 JoyBell 的 logo 和主题色”容易被误读为直接复制企业官网视觉。应明确只借鉴色彩气质，不复制 JoyBell logo、字体造型或企业资产。
- 登录页可以品牌化，但不能变成营销落地页。第一屏只服务登录、身份边界和部署识别。

### 修正要求

- Logo 方向：原创 `feishu-iam` 标识，表达“飞书身份 + IAM 权限网关”，不使用 JoyBell 的“金”字或现有图形。
- 主题方向：深蓝可信、浅蓝灰背景、橙色只作强调和告警，不做大面积装饰。
- 禁止 3-column feature grid、装饰性图标圆圈、夸张渐变、中心化营销文案。

## Pass 5: Design System Alignment

评分：**6/10 -> 8/10**

### 阻塞问题

主题 token 需要集中定义，不能散落在页面 style 里。

建议 token：

```ts
colorPrimary: '#0f4c81'
colorInfo: '#1f67b2'
colorWarning: '#f28c28'
colorBgLayout: '#eef4f8'
colorTextBase: '#122230'
borderRadius: 6
fontSize: 14
```

组件级建议：

- `Layout.headerBg = '#ffffff'`
- `Layout.siderBg = '#ffffff'`
- `Menu.itemSelectedBg` 使用浅蓝底，不用高饱和色块。
- `Table.headerBg` 使用接近 `#f6f9fb` 的浅灰蓝。
- `Button.primary` 只使用主蓝。
- `Tag` 继续表达环境和角色，不做品牌装饰。

### 重要建议

- 新增 `BrandLogo` / `AppLogo` 项目级组件。
- 新增 `UserMenu` 或 `HeaderRightContent` 项目级组件。
- 登录页品牌视觉应复用同一个 logo 组件，而不是单独画一套。

## Pass 6: Responsive & Accessibility

评分：**4/10 -> 8/10**

### 阻塞问题

必须明确响应式规则，否则右上角对齐和页面抖动会在 768px 继续复发。

建议规则：

| Viewport | Shell behavior | Header right behavior | Content behavior |
|---|---|---|---|
| 1440 | 224px Sider，56px Header | Avatar + username + role + dropdown | 24px padding，表格可横向滚动 |
| 1280 | 同 1440 | feishu user id 不直接展示，放入 dropdown | 24px padding |
| 768 | Sider 可 collapsed 或 drawer 化 | 只显示 Avatar + dropdown，角色放下拉 | 16px padding，表格固定 `scroll.x` |

Accessibility 要求：

- 退出登录必须可键盘触发。
- User dropdown 要有可识别 label。
- Logo 图片必须有 `alt`。
- 登录主按钮触控高度不低于 44px。
- body text 对比度不低于 4.5:1。
- 错误状态不能只靠颜色表达。

## Pass 7: Unresolved Design Decisions

评分：**5/10 -> 8/10**

### 进入 Pencil 前必须决策

| Decision needed | Recommendation | If deferred |
|---|---|---|
| Logo 是文字组合还是图形 + 文字 | 图形 + `feishu-iam` 文字，后台 Sider 使用紧凑横向版，登录页使用完整横向版 | 实现阶段继续出现纯文字品牌，视觉问题不解决 |
| Header 右侧显示哪些信息 | 只显示 Avatar、用户名、主角色，feishu user id 和退出放 Dropdown | 长 ID 撑破 Header，右上角继续不齐 |
| 退出是否需要确认 | 用户菜单点击“退出登录”后弹 Popconfirm 或 Modal，确认后调用 logout | 误点会直接丢 session |
| 移动端 Sider 模式 | 768px 使用 collapsed Sider 或 Drawer，需要 Pencil 明确 | 实现阶段随意压缩，菜单和内容重叠 |
| 登录页是否使用品牌插画 | 不使用大插画，使用低调品牌面板和信息块 | 登录页容易偏营销或变空洞 |

## Blocking Findings

1. `v0.1.10` 不能直接进入实现。必须先产出新 Pencil 原型或等价设计稿，覆盖 Admin Shell、Login、UserMenu/Logout 和三视口布局。
2. 主题不能只改 `colorPrimary`。必须同步 Layout、Menu、Table、Button、Tag 等组件 token，否则页面仍会读成默认 Ant Design。
3. 退出登录不是纯前端菜单项。需要至少定义后端清 cookie API、前端 query cache 清理、跳转和失败反馈。
4. 页面抖动必须作为设计约束写入原型：固定 Header/Sider、表格容器高度、loading skeleton、右上角内容最大宽度和省略策略。

## Important Findings

1. 登录页应从 `Result` 组件主导改为专门的登录布局；`Result` 适合结果页，不适合正式产品入口。
2. 当前 `DESIGN.md` 说“不要做大 logo”，但用户现在明确要求补 logo。新规则应覆盖旧规则：后台用紧凑 logo，登录页可用完整 logo。
3. 历史 compact viewport 学习表明，768px 表格隐藏字段是合理的，但必须通过详情 Drawer 保证信息可达。
4. `Ant Design App.useApp()` 的通知约束要继续保留，避免动态主题下静态 message 警告复发。

## Optional Findings

1. 可以在登录页展示“当前部署 URL / HTTP runtime / 专用飞书应用”三项元信息，但不要堆成说明文。
2. 可以增加 Footer 版权或版本号，但不应占用主要操作路径。
3. 可考虑后续加入顶部全局搜索，但不属于本版本。

## Pencil 原型输入要求

`v0.1.10` Pencil 原型至少包含：

1. Admin Shell 1440px：Sider、Header、右上角 UserMenu 打开态。
2. Admin Shell 768px：菜单折叠/抽屉态、Header 右侧紧凑态。
3. Login 正常态：飞书登录主入口。
4. Login callback processing 态。
5. Login error 态：配置缺失、飞书登录失败、无后台权限。
6. Logout confirm 态。
7. 典型表格页 loading / empty / error / data 态，优先 `/applications`。
