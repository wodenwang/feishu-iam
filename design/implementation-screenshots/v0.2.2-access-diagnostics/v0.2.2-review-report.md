# v0.2.2 接入诊断 Pre-landing Review

## 结论

通过，未发现阻塞发布的问题。

## Diff 风险检查

- 权限边界：诊断 API 复用 `requireAdminActor` 和 `requireApplicationScope`，应用管理员只能查看自己 scope 内应用诊断。
- 敏感信息：后端 diagnostics 返回状态、计数、端点和 requestId，不返回 secret、token、authorization code、signature、cookie 或 hash 原文；复制审计 route 不接收诊断包正文。
- 数据边界：不新增 schema，不新增本地 username/password，不改变飞书唯一身份源。
- UI 边界：新能力挂在应用详情 `接入诊断` Tab，不重构既有 v0.2 接入配置、权限注册、应用管理员和审计路径。
- 版本边界：不实现 OIDC/JWKS/PKCE/refresh、SDK/CLI、webhook/incremental sync、部署产品化或目录编辑。

## 测试覆盖

- 后端：healthy、warning、failed、复制审计和脱敏断言。
- 前端：HTTP adapter、mock service、详情页 Tab 和 Markdown 诊断包生成。
- Runtime：新增 `scripts/verify-v0.2-access-diagnostics.sh` 覆盖 warning -> healthy -> failed 和复制审计。

## 发布注意

- GitHub release notes 应强调这是 `v0.2.2` 诊断补强版本，不是新协议或部署产品化版本。
- 120 部署后必须用线上 `/api/health` 和应用详情页面做最终确认。
