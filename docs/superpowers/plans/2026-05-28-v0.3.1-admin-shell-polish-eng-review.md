# feishu-iam v0.3.1 Admin Shell 外框与顶部栏修复工程评审

日期：2026-05-28

## 结论

通过，按补丁版本执行。`v0.3.1` 只处理 GitHub issue #25 中的 Admin Shell 外框、顶部栏、账户入口和侧边栏收缩交互问题，不重开同步、OAuth、应用接入、目录治理或协议产品化范围。

## 版本边界

### In Scope

- 梳理 `AdminLayout` 顶部栏信息层级，保留产品语义和页面操作效率。
- 修复侧边栏收缩按钮，使其能切换 collapsed 状态，并同步菜单、Logo 和内容区布局。
- 简化 `UserMenu` 顶部触发器，只保留必要用户身份入口。
- 将环境和 runtime 信息迁移到低干扰位置，避免在首页首屏显眼展示。
- 更新 `AdminLayout` / `UserMenu` 测试。
- 补充 `v0.3.1` 截图、设计 QA、功能 QA、pre-landing review、ship 和 deploy 证据。

### Out Of Scope

- `/directory` 编辑、导入、导出或目录治理。
- 完整 event-driven sync worker、事件积压调度器、告警平台。
- OIDC/JWKS/PKCE/refresh token、SDK、CLI、Helm、Terraform。
- 角色授权、应用接入配置或第三方 Demo 新能力。
- username/password、本地 root 或独立账号体系。

## 工程决策

- D1：不重做 Pencil 原型。该切片是既有 Ant Design Admin Shell 的缺陷修复，已有 v0.1.10 外框设计基线和 issue 验收标准足够约束实现。
- D2：侧边栏 collapsed 使用组件 state 驱动，避免只由 viewport 推导导致按钮无法生效。
- D3：顶部栏不直接展示 `HTTP runtime`、`Mock data`、`生产环境`、`本地开发` 等调试/运行态标签；这些信息只在用户下拉详情中展示。
- D4：账户入口保持轻量：桌面显示头像和用户名，窄屏只显示头像；角色、环境、open_id 和退出操作进入下拉层。

## 验收策略

- 单元测试覆盖顶部栏不显眼展示 runtime/environment、用户下拉仍能查看低干扰运行信息、侧边栏按钮可切换 collapsed。
- 构建验证覆盖 TypeScript 和 Vite。
- 浏览器验证覆盖 `/applications` desktop、laptop、tablet，检查顶部栏、账户入口、折叠/展开、菜单和内容区无错位。
- QA / design review / pre-landing review 记录到 `design/implementation-screenshots/v0.3.1-admin-shell-polish/`。
