## Findings

1. Docker runtime 最初依赖 `dist-server`，但 `npm run server:build` 没有 emit JavaScript。
   - 风险：CI 中 build 通过，但镜像运行时找不到 server entrypoint，部署后容器持续重启。
   - 修复：`server:build` 改为 typecheck + Vite SSR bundle，并复制 migrations。
   - 复验：Docker build 和本地 compose smoke 通过。
2. Docker build 最初没有设置 `VITE_IAM_API_MODE=http`。
   - 风险：部署环境展示 mock-only 前端，用户看到成功页面但不会访问真实 Fastify runtime。
   - 修复：Dockerfile build stage 默认设置 `VITE_IAM_API_MODE=http`。
   - 复验：Playwright 截图显示 `HTTP runtime`。
3. CI frontend tests 使用默认 5 秒 timeout，在 GitHub runner 上对既有 Ant Design 多步交互测试不够。
   - 风险：本地通过但 PR checks 偶发或稳定失败，阻塞自动 land。
   - 修复：`vitest.config.ts` 设置 `testTimeout: 10_000`。
   - 复验：重新触发 GitHub Actions。
