# Feishu IAM v0.13.1 左侧导航层级修复规格

日期：2026-05-28
状态：设计评审前规格

## 1. 背景

`v0.13.0` 已发布并完成 `系统管理 / 飞书同步` 运维控制台。当前仍打开的 GitLab issue 中，`#12` 指向后台壳层的基础体验问题：左侧导航虽然已有一级和二级路由数据，但视觉和交互上仍更像扁平列表，不能稳定表达“系统管理”与其二级菜单的层级关系。

本版本建议作为 `v0.13.1` 小版本推进，目标是修复导航组件结构和交互表达，不新增业务能力。

## 2. 输入来源

- `AGENTS.md`：当前阶段、前端 Admin UI 开发规范和 Browser 自检硬约束。
- `DESIGN.md`：shadcn/ui + tweakcn 设计基线，特别是左侧导航必须使用能表达一二级目录的组件结构。
- GitLab issue `#12`：`v0.11.0 菜单样式无法体现一二级目录，需要更换合适导航组件`。
- `docs/superpowers/plans/2026-05-26-feishu-iam-v0.11.0-system-management-baseline.md`：系统管理 IA 的原始实施边界。
- `README.md`：`v0.13.0` 已发布的版本事实。

## 3. 版本号

推荐版本号：`v0.13.1`。

原因：

- 这是 `v0.13.0` 之后的后台壳层体验修复，不引入新业务域。
- 范围集中在 GitLab issue `#12`，适合小版本。
- 不应直接升到 `v0.14.0`，除非另开全新产品能力。

## 4. 目标

交付一个可理解、可展开收起、刷新后状态正确、收缩态仍不丢失上下文的后台左侧导航。

用户在 `系统管理 / 飞书同步`、`系统管理 / 管理员授权`、`系统管理 / 操作审计`、`系统管理 / 系统信息` 页面之间切换时，应明确知道这些页面属于同一个一级模块。

## 5. 纳入范围

- 重做或强化 `apps/admin-web/src/components/admin/AppShell.tsx` 中的导航渲染结构。
- 一级导航包含：`工作台`、`应用管理`、`权限管理`、`系统管理`。
- `系统管理` 作为可展开/收起的一级分组，二级菜单包括：
  - `飞书同步`
  - `管理员授权`
  - `操作审计`
  - `系统信息`
- 进入任一系统管理二级页面后：
  - `系统管理` 分组保持展开。
  - 当前二级菜单高亮。
  - 一级分组也显示 active 上下文。
- 桌面展开态使用常驻二级导航，不用普通 `DropdownMenu` 替代。
- 桌面收缩态使用图标、tooltip 和可理解的分组提示；不得让系统管理的二级入口完全不可发现。
- 移动端导航 Sheet 保留完整层级和当前项高亮。
- 保留旧路由兼容：`/admin/records`、`/admin/admins`、`/admin/settings`。
- 补充或更新组件测试、App 路由测试和响应式/浏览器自检。

## 6. 不纳入范围

- 不新增后端 DDL、Prisma schema 或 API。
- 不扩大 SSO 协议面，不实现完整 OIDC、SAML、ABAC、资源级权限、HTTPS、高可用或滚动升级。
- 不修改飞书同步业务能力。
- 不恢复权限管理中的角色元数据新增、编辑或启停。
- 不改变生产路径的管理员 session 和管理员权限校验机制。
- 不处理 GitLab `#6/#7/#9` 的剩余关闭状态，除非本轮验证发现它们确实仍回归。

## 7. 设计要求

### 7.1 展开态

- 使用项目级 `SidebarNav` / `NavItem` 或等价结构表达父子层级。
- 父级分组具备明确的展开/收起按钮或可触发区域。
- 二级菜单相对父级有缩进、连接线、弱化背景或其他稳定层级信号。
- 当前二级菜单高亮时，父级分组也具备上下文高亮。
- 父级展开/收起状态不应造成主内容区域尺寸跳动。

### 7.2 收缩态

- 一级菜单保留图标按钮和 tooltip。
- 含二级菜单的一级项需要让用户理解其下存在子项，至少通过 tooltip、弹出层或可访问名称表达。
- 当前位于系统管理二级页面时，收缩态的 `系统管理` 图标仍应处于 active 状态。

### 7.3 移动端

- 移动端继续使用左侧 Sheet。
- Sheet 内展示完整一级和二级层级。
- 当前二级项高亮，父级展开。
- 触控目标不小于既有后台导航标准。

### 7.4 可访问性

- 一级分组展开按钮必须有稳定中文 accessible name。
- 展开状态应通过 `aria-expanded` 或等价语义表达。
- 当前页面仍使用 `aria-current="page"`。
- 收缩态 tooltip 不替代 accessible name。

## 8. 验收标准

- 桌面端左侧导航能清楚看到一级目录和二级目录归属。
- `系统管理` 可展开/收起，二级菜单收缩在其下。
- 刷新 `/admin/system/audit` 后，`系统管理` 保持展开，`操作审计` 高亮。
- 收缩侧边栏后，`系统管理` 图标可通过 tooltip 或合理弹出方式理解其分组和子项。
- 移动端 Sheet 可打开并展示完整系统管理二级菜单。
- 面包屑、页面标题、URL 和菜单高亮一致。
- 浏览器 console 无新增错误，Network 无非预期失败请求。
- `pnpm check` 或更窄的等价前端检查通过。
- 完成 `@Browser` 自检，覆盖桌面、窄屏、展开/收起、刷新保持和键盘焦点。
