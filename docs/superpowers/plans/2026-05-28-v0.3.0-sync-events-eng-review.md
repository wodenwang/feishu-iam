# feishu-iam v0.3.0 飞书事件同步可靠性工程评审

日期：2026-05-28

## 结论

通过，按窄切片执行。`v0.3.0` 从 `v0.2.x` 应用接入生产化切到同步可靠性主线，只交付飞书事件订阅的接收、校验、去重、状态记录和重试入口，不展开目录治理、告警平台或完整事件驱动同步。

## 版本边界

### In Scope

- `POST /api/feishu/events` 接收飞书事件订阅回调。
- 支持 URL verification challenge。
- 支持 Verification Token 校验；配置 Encrypt Key 时校验 `X-Lark-Request-Timestamp`、`X-Lark-Request-Nonce`、`X-Lark-Signature`，并支持解密 `encrypt` 事件体。
- 新增 `sync_events` 表记录事件 id、事件类型、资源标识、状态、requestId、关联同步 run 和失败原因。
- 新增平台管理员事件状态、列表和重试 API。
- `/sync` 页面新增飞书事件同步健康区。
- 新增自动验收脚本 `scripts/verify-v0.3-sync-events.sh`。

### Out Of Scope

- `/directory` 编辑、导入、导出或目录治理。
- 飞书机器人、Slack 或告警平台。
- 完整 incremental sync worker、事件积压调度器或 trace/metrics 系统。
- OIDC/JWKS/PKCE/refresh token。
- SDK、CLI、Helm、Terraform。
- username/password、本地 root 或独立账号体系。

## 工程决策

- D1：事件回调 route 不使用 IAM session，但必须通过飞书 Verification Token 和可选 Encrypt Key 签名校验保护。
- D2：事件默认只记录为 `pending_sync`，由平台管理员在同步页重试处理；本切片不在回调请求内直接执行通讯录同步，避免飞书回调超时。
- D3：重复 `event_id` 幂等返回，不重复写事件或触发同步。
- D4：事件重试使用 system actor 触发同步 run，不伪造飞书用户。
- D5：事件记录只保存类型、资源标识、requestId 和摘要，不保存 token、secret、signature 或完整原始事件体。

## 验收策略

- 后端测试覆盖 URL verification、事件接收、去重、事件状态/列表、重试同步和非平台管理员禁止读取事件状态。
- 前端构建覆盖同步页事件区类型接线。
- 验收脚本覆盖 runtime challenge、事件 pending、重复 event id、列表、重试和敏感信息扫描。
