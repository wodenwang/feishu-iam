# Feishu IAM v0.15.1 权限可解释性与后台操作一致性规格

日期：2026-05-28
状态：实施前规格

## 1. 背景

`v0.15.0` 已收口组织树与组织用户选择器，并明确把 GitLab issue `#21` 留到后续小版本处理。当前真实打开的 GitLab issue 是：

- `#21`：角色权限点绑定需支持权限组内权限点查看和最终权限点搜索一览。
- `#25`：左侧导航一级菜单 hover 宽度与带二级菜单项不一致。

本轮用户还发现一个新的后台回归：应用详情页 `角色管理` 列表中，`停用` 按钮仍会变形。该问题与 `v0.15.0` 已处理的 `#24` 同属操作列稳定性问题，但本轮要求更明确：应用详情角色管理的启用/停用操作应统一为图标按钮形式，而不是继续使用容易被压缩变形的文字按钮。

本版本定位为 `v0.15.1`，主线是权限可解释性；同时合并两个后台 UI 一致性回归修复，避免为很小的视觉问题拆出单独 patch 版本。

## 2. 输入来源

- `AGENTS.md`：当前阶段、Admin UI 开发规范、Browser 自检硬约束和 `v0.15.0` 不回退边界。
- `DESIGN.md`：shadcn/ui + tweakcn + Tailwind 基线、后台 CRUD、表格操作列、图标按钮、tooltip 和响应式要求。
- `README.md`、`CHANGELOG.md`：当前最新发布版本为 `v0.15.0`。
- `docs/superpowers/specs/2026-05-28-feishu-iam-v0.15.0-org-tree-selector.md`：明确建议 `v0.15.1` 处理 `#21`。
- GitLab issue `#21`：权限组内权限点查看、角色最终权限点一览、权限点搜索和来源展示。
- GitLab issue `#25`：左侧导航普通一级菜单与带二级菜单父级 hover / active / focus-visible 宽度不一致。
- 用户本轮新增反馈：应用详情 `角色管理` 列表中的启用/停用按钮需要统一为图标形式，解决停用按钮变形问题。

## 3. 版本号

推荐版本号：`v0.15.1`。

原因：

- `#21` 是 `v0.15.0` 后明确留下的小版本范围，属于权限管理体验增强，不需要升到 `v0.16.0`。
- `#25` 和应用详情角色管理停用按钮图标化都是后台 UI 一致性回归，单独发布价值偏低，适合随 `#21` 一起收口。
- 本版本不改变权限判定语义、SSO 协议、飞书同步模型或部署拓扑，仍然符合 patch 小版本边界。

## 4. 目标

- 让管理员在角色详情中能理解“这个角色最终拥有哪些具体权限点”，并能快速按权限点 key 或标题搜索。
- 让管理员能查看权限组内包含的权限点，避免只看到权限组名称而无法核对实际授权。
- 清楚展示权限点来源：直接绑定、来自一个或多个权限组，或同时存在直接绑定和权限组来源。
- 统一后台左侧导航一级菜单 hover / active / focus-visible 的行宽、圆角、内边距、对齐和点击区域。
- 统一应用详情 `角色管理` 列表操作列的启用/停用为图标按钮，避免文字按钮被压缩、换行或变形。

## 5. 纳入范围

### 5.1 GitLab issue #21：权限组内权限点查看

- 在角色权限绑定工作区中，权限组列表必须提供查看权限组内权限点的入口。
- 权限组内权限点查看可以采用展开行、详情面板、右侧详情区或等价轻量承载方式，但不能隐藏到复杂表单深处。
- 权限组内权限点列表至少展示：
  - 权限点 key。
  - 权限点标题或名称。
  - 权限点状态。
  - 描述或必要摘要。
- 权限组权限点为空、加载中、加载失败和搜索无结果时，必须展示稳定中文状态。
- 如果现有接口缺少权限组内权限点字段，可补充最小 API 支持；不新增 DDL，不改变权限模型。

### 5.2 GitLab issue #21：角色最终权限点一览

- 在角色详情中提供“最终权限点”一览区域，汇总展示当前角色最终拥有的具体权限点。
- 最终权限点必须合并直接绑定权限点和通过权限组获得的权限点。
- 同一权限点来自多个来源时只展示一条最终结果，来源信息聚合展示，不能让重复行造成误解。
- 来源展示至少能区分：
  - 直接绑定。
  - 来自某个权限组。
  - 同时直接绑定且来自一个或多个权限组。
- 最终权限点列表至少展示：
  - 权限点 key。
  - 权限点标题或名称。
  - 来源类型。
  - 来源权限组。
  - 状态或描述。
- 最终权限点区域应靠近权限组/权限点绑定主流程，可以作为独立 Tab、子区块或稳定固定区域；实现计划阶段应优先选择与现有角色详情 Tab 结构一致的承载方式。

### 5.3 GitLab issue #21：权限点搜索

- 最终权限点一览必须支持搜索。
- 搜索字段至少覆盖权限点 key 和权限点标题/名称。
- 搜索输入应靠近最终权限点列表。
- 搜索可以先做前端过滤；如果现有数据量或接口返回不适合一次性加载，可在实施计划中选择最小服务端搜索。
- 搜索无结果时展示稳定中文空状态。
- 搜索不能破坏角色详情当前上下文、Tab 状态、分页状态和返回路径。

### 5.4 GitLab issue #25：左侧导航一级菜单 hover 宽度一致

- 普通一级菜单和带二级菜单的父级菜单必须使用一致的行宽、圆角、内边距和背景范围。
- hover、active、focus-visible 状态必须覆盖整行可点击区域，而不是只包裹图标和文字。
- 展开箭头存在与否不应导致背景宽度、点击区域或文字对齐变化。
- 桌面展开态、桌面收缩态和移动端 Sheet 导航都不能出现宽度跳变、视觉错位或层级误解。
- 本修复不改变后台导航信息架构，不改变 `系统管理` 二级菜单展开/收起业务行为。

### 5.5 应用详情角色管理启停用图标化

- 应用详情页 `角色管理` Tab 的角色列表操作列中，启用/停用必须统一为图标按钮。
- 图标按钮应使用项目已有 lucide 图标体系，优先沿用管理员授权列表中启停用图标按钮的模式。
- 图标按钮必须具备稳定的 `aria-label`、`title` 或 tooltip，文案能让用户明确知道当前操作是启用还是停用。
- 停用仍然是危险操作，必须保留危险语义、确认弹窗、提交中状态、失败提示和审计含义说明。
- 操作列宽度、按钮间距、按钮尺寸和行高必须稳定；不允许启用/停用文字在按钮内部换行或把按钮压成竖向形态。
- 本修复不改变应用内角色管理的业务模型、后端接口、角色状态语义或信息架构。

## 6. 不纳入范围

- 不改变权限判定语义。
- 不新增 deny、ABAC、资源级权限或数据范围权限。
- 不实现飞书角色同步、飞书用户组同步或飞书实时通讯录树。
- 不改变 SSO、OAuth、管理员 session、管理员权限校验或生产部署拓扑。
- 不新增 Prisma DDL。
- 不把角色元数据新增、编辑或启停放回权限管理；这些能力继续归应用管理。
- 不把权限点和权限组 CRUD 扩展成新的后台常驻主流程。
- 不系统性重构所有后台表格操作列；本版本只处理应用详情角色管理列表中已发现的启停用按钮变形。
- 不修改 `#25` 以外的导航 IA 或路由结构。

## 7. 组件和数据边界

### 7.1 权限可解释性边界

权限可解释性应围绕“最终权限点”这个可理解结果组织，而不是把原始绑定关系直接平铺给管理员。实现时应明确区分：

- 输入绑定：直接权限点绑定、权限组绑定。
- 聚合结果：角色最终拥有的权限点集合。
- 来源说明：每个最终权限点来自哪些绑定。

如果后端已有足够数据，可以在前端聚合；如果现有接口会导致多次请求或来源计算不稳定，应补充最小后端聚合接口或字段。无论采用哪种方式，都不能改变后端授权判定语义。

### 7.2 UI 一致性边界

`#25` 和启停用图标化都属于后台 UI 一致性修复，应优先复用现有 `AppShell`、表格、按钮、tooltip、确认弹窗和 lucide 图标模式。不要为这两个修复引入新的视觉体系或一次性组件。

应用详情角色管理操作列应和管理员授权、权限管理等已有 icon button 操作列保持一致：图标承载动作，tooltip / aria label 承载说明，确认弹窗承载危险操作确认。

## 8. 错误处理和状态

- 权限组内权限点和最终权限点列表必须覆盖加载中、空状态、错误状态和搜索无结果状态。
- 接口失败展示稳定中文说明，不暴露堆栈、token、cookie、secret 或框架默认错误。
- 搜索输入为空时恢复默认最终权限点列表。
- 最终权限点来源较多时，来源展示必须可读，不能撑破表格或造成横向溢出。
- 启停用操作提交中必须禁用重复提交，失败后保留用户所在页面和当前 Tab。
- 导航 hover / active / focus-visible 修复不能降低键盘可访问性。

## 9. 验收标准

- 角色详情权限绑定工作区可以查看任一权限组内包含的权限点。
- 角色详情可以一览该角色最终拥有的具体权限点。
- 最终权限点一览支持通过权限点 key 搜索。
- 最终权限点一览支持通过权限点标题或名称搜索。
- 权限点来源展示清晰，能区分直接绑定、来自权限组和二者都有。
- 同一权限点来自多个权限组或直接绑定时，最终列表不重复展示为多条误导性记录。
- 桌面和 390px 窄屏下，权限组列表、最终权限点列表、搜索框、来源说明无错位、遮挡、溢出或异常空白。
- 鼠标移上 `工作台`、`应用管理`、`权限管理` 等普通一级菜单时，hover 背景宽度与 `系统管理` 父级菜单一致。
- 普通一级菜单和可展开父级菜单在 hover、active、focus-visible 状态下无宽度跳变或视觉错位。
- 应用详情 `角色管理` Tab 中，启用/停用操作显示为图标按钮，不再出现文字按钮换行、压缩或竖向变形。
- 图标按钮具备明确 tooltip、`title` 或 `aria-label`，危险操作仍打开确认弹窗。
- 浏览器 console 无新增错误，Network 无非预期失败请求。
- `pnpm check`、相关前端测试、管理后台构建、响应式检查和 Browser 自检通过。

## 11. 后续版本建议

- 如果 `v0.15.1` 之后继续发现表格操作列视觉不一致，应单独开 issue 做后台操作列规范化，不要把未登记的全局整理塞进本版本。
- 如果最终权限点查询在真实数据量下性能不足，后续可做服务端分页、索引或专用聚合接口优化；这属于性能增强，不改变 `v0.15.1` 的功能验收边界。
