# Feishu IAM 管理后台设计系统契约

本文档是 `design/admin-console.pen` 的工程实现契约。Pencil 文件负责视觉方向，本文档负责前端落地时必须保持一致的颜色、密度、状态和响应式规则。

## 视觉基调

- 管理后台采用深青绿色侧栏、深色顶部栏、白色内容面板和浅灰绿页面底色。
- 视觉重点是稳定、克制、可扫描，不使用营销式 hero、大面积渐变、装饰卡片堆叠或泛用三列大卡片。
- 项目真实 logo 使用 `apps/admin-web/src/assets/feishu-iam-logo.png`，Pencil 中的 TST IAM logo 仅为占位。

## 颜色

- 侧栏背景：`#082528` 到 `#0b2f33`，激活项背景 `#0f3b3f`。
- 顶部栏背景：`#0b2f33`，底部强调线 `#20b386`。
- 主品牌绿色：`#168a60`，高亮青绿：`#20b386`。
- 页面背景：`#f4f7f6`，面板背景：`#ffffff`。
- 正文：`#18212f`，次级文字：`#526172` / `#667085`。
- 边框：`#d9e1ea`，输入边框：`#c8d3df`。
- 成功：文字 `#067647`，背景 `#ecfdf3`。
- 警告：文字 `#b54708`，背景 `#fffaeb`。
- 错误：文字 `#b42318`，背景 `#fef3f2`。
- 禁用：文字 `#475467`，背景 `#f2f4f7`。

## 字体和排版

- 默认字体使用系统无衬线字体；如果后续引入 Pencil 字体，可映射为 `Funnel Sans` 和 `Geist Mono`，但不能阻塞功能。
- 一级页面标题 28px，模块标题 22px，表格和表单正文 14px，辅助说明 12px 到 13px。
- 字距保持默认值，不使用负字距，不随视口宽度缩放字号。
- `app_key`、`client_id`、URL、request id 和 token 占位文本必须可换行或横向滚动，不能撑破容器。

## 间距和圆角

- 页面内边距桌面 24px，窄屏 16px 到 18px。
- 面板、表格、抽屉、表单块间距以 12px、16px、24px 为主。
- 卡片和面板圆角固定 8px，输入和按钮圆角固定 6px。
- 卡片只用于真实信息块；列表、筛选、分页和操作区优先使用表格或紧凑面板。

## 信息架构

- 六个一级模块固定为：工作台、应用管理、权限管理、管理员授权、记录查询、系统设置。
- 一级模块默认采用“清单 -> 右侧详情抽屉”范式。
- 详情抽屉关闭后必须保留筛选条件、分页、排序和当前列表上下文。
- 复杂编辑可在抽屉内使用 tab 或表单弹窗，但不能回到每页自由选择详情页、内联表单或裸弹窗。

## 表格密度

- 表头 14px，行高保持紧凑，单元格上下内边距 12px。
- 操作列宽度稳定，危险操作位置固定，不能随数据长度左右跳动。
- 表格外层必须支持横向滚动，长字段优先截断、换行、tooltip 或复制。
- 空状态、错误状态和加载状态必须出现在表格工作区内，不使用页面级空白占位。

## 抽屉

- 桌面右侧详情抽屉宽度为 `min(720px, calc(100vw - 248px))`。
- 960px 以下抽屉宽度不超过 720px。
- 760px 以下抽屉全宽显示，内容继续保持表格横向滚动。
- 抽屉顶部必须包含标题、状态、说明和关闭按钮。

## 按钮和确认弹框

- 主操作按钮使用青绿色实底，次操作按钮使用白底描边。
- 危险操作使用红色语义按钮，并必须进入统一确认弹框。
- 高风险操作包括：停用应用、轮换 secret、查看 secret、停用角色、启停管理员、导出记录、立即同步。
- 确认弹框必须展示影响说明、审计提示、取消按钮、确认按钮、提交中状态和失败信息。

## Secret 展示

- `client_secret` 和开发者 API token 只允许在创建、查看或轮换后短时展示。
- Secret 展示区必须说明“关闭或超时后会从页面清除”，并提供关闭按钮。
- 普通列表、日志、错误信息、审计详情和会话归档不得出现明文 secret、token、cookie 或密码。

## 页面状态

- 列表、抽屉、表单、搜索候选、导出、secret 展示和同步触发都必须有加载、空、错误和提交中状态。
- 错误信息使用稳定中文说明，不显示堆栈、框架默认错误、明文凭证或原始后端 detail。
- 部分成功状态必须明确哪些项成功、哪些项失败，以及下一步可执行动作。

## 响应式规则

- 桌面保持左侧导航、顶部用户区、内容区、表格工作区的稳定结构。
- 760px 以下左侧导航转换为顶部品牌区和横向滚动模块栏，当前模块必须始终可见。
- 窄屏不使用汉堡菜单，也不声明只支持桌面。
- 390px 宽度下，筛选区纵向排列，表格横向滚动，抽屉全宽，弹窗按钮可换行。
- 页面级 `documentElement.scrollWidth` 不得大于视口宽度；长表格、长 URL、request id 和 key 的横向滚动必须收敛在具体表格或代码块容器内。
- 响应式回归以 `apps/admin-web/test/run-responsive-overflow-check.mjs` 为最低自动化门禁，覆盖 390px、760px、960px 和桌面宽度。

## 禁用模式

- 权限不足或状态不允许时，按钮应禁用并保留稳定提示，不依赖前端假权限绕过后端校验。
- 已停用资源仍可查看详情和操作记录，但不能执行会产生新登录、授权或同步影响的动作。
