# Feishu IAM v0.5.1 真实飞书验收清单

本文档用于在 `http://192.168.2.112:3000` 验收 Feishu IAM `v0.5.1` 的内网 Docker Compose 部署、飞书通讯录同步、管理后台飞书登录和第三方 SSO 主链路。

验收过程中不要把真实密钥、token、cookie、密码、authorization code、access token 或 client secret 写入仓库、文档、截图、聊天消息、工单或会话归档。临时明文 client secret 只允许保存在当前终端会话或受控密钥介质中，用完即清理。

## 基础服务检查

在服务器或可访问内网地址的机器上执行：

```bash
curl -fsS http://192.168.2.112:3000/health
curl -fsS http://192.168.2.112:3000/ready
curl -fsS http://192.168.2.112:3000/version
```

通过标准：

- `/health` 返回服务健康。
- `/ready` 返回数据库和迁移就绪。
- `/version` 返回 `0.5.1`。
- 浏览器访问 `http://192.168.2.112:3000` 可以打开管理后台。

## 飞书同步检查

使用管理后台飞书同步页面或管理端接口完成检查：

1. 确认飞书应用配置状态为已配置。
2. 触发一次飞书通讯录同步。
3. 确认同步 run 结果为成功。
4. 确认 `activeUsers > 0`。
5. 抽查准备用于管理员登录的飞书用户处于可登录状态。

通过标准：

- 飞书同步不需要任何写飞书通讯录权限。
- 同步失败时页面或接口返回稳定错误和 request id。
- 日志和验收记录不包含真实飞书应用凭证。

## 管理员登录检查

首次部署需要用破窗入口绑定第一个 `platform_admin`：

1. 访问 `http://192.168.2.112:3000/admin/auth/bootstrap`。
2. 使用服务器本地配置的破窗账号登录。
3. 绑定真实飞书用户为 `platform_admin`。
4. 退出破窗入口。

随后使用飞书管理员身份登录：

1. 访问 `http://192.168.2.112:3000`。
2. 点击飞书登录并完成授权。
3. 登录成功后调用或观察管理端请求 `GET /api/v1/admin/me`。
4. 确认返回的管理员角色包含 `platform_admin`。
5. 确认可以访问飞书同步、应用管理、权限管理、接入配置、审计日志和安全事件页面。

通过标准：

- 日常管理使用飞书管理员身份，不继续依赖破窗入口。
- 管理员 session 使用 HttpOnly cookie，文档和日志不记录 cookie 值。
- `/api/v1/admin/me` 未登录时返回稳定认证错误。

## 测试应用配置

在管理后台创建或确认以下验收应用配置：

- app key：`acceptance`
- 环境：`dev`
- redirect URI：`http://192.168.2.112:3000/acceptance/oauth/callback`
- scope：`openid profile permissions`
- client：为 `acceptance` 的 `dev` 环境创建一个启用状态的 client。

`dev` 环境允许 `localhost` 和 RFC1918 私有 IPv4 的 HTTP 回调，用于当前内网验收；`test` 和 `prod` 环境仍然必须使用 HTTPS 回调地址。

创建 client 后，明文 client secret 只会展示一次。验收时可在当前终端临时保存为环境变量，不能写入文件：

```bash
FEISHU_IAM_URL='http://192.168.2.112:3000'
CLIENT_ID='<client_id>'
CLIENT_SECRET='<client_secret>'
REDIRECT_URI='http://192.168.2.112:3000/acceptance/oauth/callback'
FORM_EQ='='
```

## 浏览器授权

在浏览器打开以下 URL。`redirect_uri` 已按目标回调地址编码：

```text
http://192.168.2.112:3000/oauth/authorize?response_type=code&client_id=<client_id>&redirect_uri=http%3A%2F%2F192.168.2.112%3A3000%2Facceptance%2Foauth%2Fcallback&state=v051-acceptance&scope=openid%20profile%20permissions
```

预期流程：

1. Feishu IAM 校验 client、回调地址和 state。
2. 浏览器跳转到飞书登录。
3. 飞书回调 `http://192.168.2.112:3000/oauth/feishu/callback`。
4. Feishu IAM 回调测试应用地址 `http://192.168.2.112:3000/acceptance/oauth/callback`。
5. 回调页显示 `code` 和 `state`，不保存、不换 token、不展示敏感凭证。

记录从回调页取得的授权码时，只放在当前终端变量中：

```bash
AUTH_CODE='<code>'
```

## 换取 token

使用授权码换取 Feishu IAM access token：

```bash
curl -fsS -X POST "$FEISHU_IAM_URL/oauth/token" \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  --data-urlencode 'grant_type=authorization_code' \
  --data-urlencode "code=${AUTH_CODE}" \
  --data-urlencode "redirect_uri=${REDIRECT_URI}" \
  --data-urlencode "client_id=${CLIENT_ID}" \
  --data-urlencode "client_secret${FORM_EQ}${CLIENT_SECRET}"
```

通过标准：

- 返回 access token、token type、过期时间和 scope。
- 响应不包含内部哈希、飞书应用凭证或数据库连接信息。
- 授权码只能使用一次。

把返回的 access token 只保存到当前终端变量：

```bash
ACCESS_TOKEN='<access_token>'
```

## 查询 userinfo

```bash
curl -fsS "$FEISHU_IAM_URL/oauth/userinfo" \
  -H "Authorization: Bearer ${ACCESS_TOKEN}"
```

通过标准：

- 返回当前飞书登录用户的稳定身份信息。
- 不返回不应暴露的内部字段或敏感凭证。
- token 无效、过期或已撤销时返回稳定认证错误。

## 查询应用权限

```bash
curl -fsS "$FEISHU_IAM_URL/api/v1/apps/acceptance/me/permissions" \
  -H "Authorization: Bearer ${ACCESS_TOKEN}"
```

通过标准：

- 返回 `acceptance` 应用范围内的权限组和权限点清单。
- 不返回其他应用的权限数据。
- 如果当前用户未被授予验收应用权限，响应应为空清单或稳定的业务结果，而不是服务错误。

## 撤销 token

```bash
curl -fsS -X POST "$FEISHU_IAM_URL/oauth/revoke" \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  --data-urlencode "token${FORM_EQ}${ACCESS_TOKEN}" \
  --data-urlencode "client_id=${CLIENT_ID}" \
  --data-urlencode "client_secret${FORM_EQ}${CLIENT_SECRET}"
```

通过标准：

- 返回撤销成功语义。
- 对未知 token 不泄露是否存在。
- 审计日志和安全事件不记录明文 token 或 client secret。

## 撤销后复查

撤销后再次调用 userinfo：

```bash
curl -sS "$FEISHU_IAM_URL/oauth/userinfo" \
  -H "Authorization: Bearer ${ACCESS_TOKEN}"
```

撤销后再次调用权限接口：

```bash
curl -sS "$FEISHU_IAM_URL/api/v1/apps/acceptance/me/permissions" \
  -H "Authorization: Bearer ${ACCESS_TOKEN}"
```

通过标准：

- 两个请求都不能继续返回有效用户或权限数据。
- 返回稳定认证错误，例如 token 已撤销、无效或未授权。
- 错误响应包含可排查的 request id。
- 错误响应不展示堆栈、框架默认错误页或敏感凭证。

## 清理终端变量

完整验收流程结束后，清理当前终端中保存的临时变量：

```bash
unset AUTH_CODE ACCESS_TOKEN FEISHU_IAM_URL REDIRECT_URI CLIENT_ID CLIENT_SECRET FORM_EQ
```
