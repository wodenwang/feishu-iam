# Feishu IAM v1.0.2 UI/UX P0 验收记录

日期：2026-06-13

## 版本范围

`v1.0.2` 修复正式版后的三个 P0 UI/UX 问题：

- 390px 管理后台资源列表不再把 `app_key`、角色 key、request id 等长字段压缩成逐字竖排。
- 操作审计、应用详情和角色详情等多 Tab 页面在 390px 下不再静态撑破视口。
- OAuth 公开 HTML 错误页只复制 `request id`，不再复制整段问题信息。

本版本不新增 DDL，不改变 OAuth 协议、管理员 session、权限模型、审计语义或第三方应用接入契约。

## Browser 环境说明

- 本地前端 Browser 自检使用 `http://localhost:5173`，通过项目内 Playwright 脚本安装 API mocks 后验证后台页面布局。
- 本地 API Browser 自检尝试使用 `PORT=3001 NODE_ENV=test CLIENT_SECRET_ENCRYPTION_KEY=0123456789abcdef0123456789abcdef pnpm --filter @feishu-iam/api dev` 启动，但 Nest 初始化被 Prisma `DATABASE_URL` 前置校验阻断。
- 因本次不读取或猜测本地数据库连接与密钥，公开 OAuth 错误页先以 API 自动化测试作为本地证据，发布后通过生产 canary 复核真实 HTML。

## 安全边界

- 公开错误页不出现 `复制问题信息`。
- 公开错误页不包含 `data-feedback`。
- 文档不要求终端用户复制 token、cookie、authorization、授权码、token hash、state hash 或 raw payload。
- 移动端列表只展示已有脱敏字段，不新增敏感字段。
