## 部署动作

1. 使用 `git archive --format=tar HEAD` 将仓库受控文件同步到远端部署目录。
2. 保留远端 `.env` 与 `data/postgres`，并创建 `.env.backup-v0.1.15-20260526220813`。
3. 将远端 `.env` 中的 `FEISHU_IAM_IMAGE_TAG` 更新为 `v0.1.15`。
4. 在远端执行 `docker compose up -d --build` 完成镜像构建、容器重建和启动。

## 边界确认

- 本版本只交付手动飞书通讯录同步 Runtime，没有引入定时同步、事件订阅同步或本地用户密码体系。
- 平台管理员仍绑定飞书用户，同步入口只对平台管理员开放。
- 远端 `.env`、飞书应用凭证、tokens、同步快照和 PostgreSQL 数据目录未纳入仓库同步。
