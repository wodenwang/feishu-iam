# v0.1 后端 API 实施计划

## 目标

实现支撑 v0.1 前端闭环的后端最小 API：飞书登录态、平台管理员初始化、应用管理、角色授权、飞书目录同步、审计日志和统一错误返回。

## 范围

- 飞书是唯一身份与组织数据源，不实现 username / password。
- 系统管理员和应用管理员都必须绑定飞书用户。
- 所有飞书 API 调用使用项目绑定的专用自建飞书应用。
- 不提交 app secret、tenant token、同步快照、导出用户列表或真实审计日志。

## 任务拆分

1. 建立后端骨架
   - 选择项目确认的 Node.js / TypeScript 后端框架。
   - 增加 `server/` 独立入口、配置加载、健康检查和测试脚本。
   - `.env.example` 只放变量名和示例占位值。

2. 实现飞书登录与会话
   - `GET /api/auth/feishu/authorize-url` 返回飞书 OAuth 授权地址。
   - `GET /api/auth/feishu/callback` 只接受飞书 code 换取用户身份。
   - `GET /api/session/current` 返回当前飞书用户、角色、权限和可管理应用范围。
   - 未初始化时引导到平台管理员绑定流程。

3. 实现平台初始化
   - `GET /api/initialization/status` 返回是否已有平台管理员。
   - `POST /api/initialization/bind-platform-admin` 将当前飞书用户绑定为平台管理员。
   - 禁止手工输入任意用户 ID 伪造管理员。

4. 实现应用管理 API
   - `GET /api/applications` 支持分页、关键词、状态筛选。
   - `POST /api/applications` 创建第三方应用，并生成仅可显示一次的运行时 secret。
   - `GET /api/applications/:id` 返回详情、权限组、权限点和最近调用摘要。
   - `POST /api/applications/:id/disable` 停用应用。

5. 实现角色授权 API
   - `GET /api/roles` 支持应用维度筛选。
   - `POST /api/roles`、`PATCH /api/roles/:id` 管理角色基础信息。
   - `PUT /api/roles/:id/authorization` 保存权限点和组织范围。
   - 应用管理员只能操作自己负责的应用范围。

6. 实现目录同步 API
   - `GET /api/directory/departments` 返回飞书部门树投影。
   - `GET /api/directory/users` 返回飞书用户只读投影。
   - `POST /api/sync/runs` 触发飞书同步。
   - `GET /api/sync/runs` 和 `GET /api/sync/runs/:id` 返回同步运行和失败详情。

7. 实现审计与错误
   - 所有写操作写入审计日志，包含 actor、action、target、requestId 和结果。
   - `GET /api/audit-logs` 支持筛选和分页。
   - 统一错误格式包含 `requestId`、`code`、`message`、`details`。

## 预期输出

- 后端 API 能支撑当前前端 mock services 替换为真实 services。
- OpenAPI 文档覆盖 v0.1 所有前端页面所需接口。
- 审计日志能追踪应用创建、停用、角色授权、同步触发和失败重试。

## 完成标准

- 所有新增 API 有单元或集成测试。
- 非飞书身份入口不存在。
- 平台管理员绑定只能来自当前飞书登录用户。
- `npm run server:test`、`npm run server:build`、`npm run test`、`npm run e2e` 通过。
- 仓库中没有真实飞书凭证、token、导出用户或同步快照。

## 风险与边界

- 飞书 OAuth 和通讯录权限需要在专用自建应用中提前开通。
- v0.1 不实现复杂审批流，只提供应用、角色、目录、审计的最小闭环。
- v0.1 不实现多租户；一个部署实例绑定一个飞书租户和一个专用飞书应用。
