# v1.0.3 Base Portal 接入包与完整提示词会话归档

## 会话目标

使用 `my-harness` 推进并完成 Feishu IAM `v1.0.3`：Base Portal 接入包与完整提示词收敛。

## 用户关键要求

- 配合 Base Portal 做必要改动。
- 简化应用 secret 等信息获取，不再拆得太细。
- 统一让用户复制完整提示词。
- 如果 secret 变了，则让用户重新刷新提示词。
- 提示词应包含详细 secret 等密钥，否则提示词没有展示价值。
- 不实现完整 OIDC、refresh token、SAML、ABAC、资源级权限或长期明文保存 developer API token。

## 关键约束

- 文档使用中文。
- 不长期明文保存 developer API token。
- 不放宽 `redirect_uri` 精确匹配。
- 不记录真实 secret、token、cookie、authorization、授权码、hash 或 ciphertext。
- 不改变管理员 session、权限模型、飞书通讯录同步或生产部署拓扑。

## 重要设计决策

- `v1.0.3` 采用完整提示词主路径。
- 应用详情新增 `刷新凭证并生成完整提示词`，一次确认后在同一个数据库事务中轮换 OAuth `client_secret` 和 developer API token。
- 旧安全版提示词保留后端兼容接口，但不再作为前端主展示。
- `base-portal` 作为 preset 写入菜单权限点建议、打开方式和 iframe 无感 SSO 验收矩阵。
- developer API token 只保存 hash；需要完整提示词时重新轮换。
