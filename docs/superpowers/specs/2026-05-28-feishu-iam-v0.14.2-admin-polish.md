# Feishu IAM v0.14.2 后台信息密度与控件稳定性规格

日期：2026-05-28
状态：实施前规格

## 1. 背景

`v0.14.0` 已把应用详情和角色详情升级为独立详情页，`v0.14.1` 已把应用详情页改为 `详细资料 / 角色管理 / 开发信息 / 危险操作` 四个 Tab。最新 GitLab issue 反馈显示，后台仍有三类局部体验问题需要在 `v0.14.x` 线上体验链路内收口：

- 应用详情中新增回调地址按钮出现文字换行和形态变形。
- 权限管理角色列表中存在重复且价值不足的应用快捷查询区域。
- 应用详情页 `角色管理` Tab 使用大卡片堆叠展示角色，信息密度低，不符合普通后台 CRUD 使用习惯。

`v0.14.2` 定位为 `v0.14.1` 之后的后台体验补丁版本，只处理 GitLab issue `#18/#22/#23`。本版本不改变后端接口、权限模型、SSO 协议或部署拓扑。

## 2. 输入来源

- `AGENTS.md`：当前阶段、Admin UI 开发规范、Browser 自检硬约束。
- `DESIGN.md`：后台 CRUD 页面、表格列宽、操作列、长字段、响应式和 shadcn/ui 基线。
- `docs/superpowers/specs/2026-05-28-feishu-iam-v0.14.1-application-detail-tabs.md`：应用详情 Tab 化基线。
- GitLab issue `#18`：应用管理新增回调地址按钮文字换行变形，需纳入前端设计规范检查。
- GitLab issue `#22`：权限管理角色列表快捷查询区域无价值应移除。
- GitLab issue `#23`：应用内角色管理应改回普通列表 CRUD 模式。

## 3. 版本号

推荐版本号：`v0.14.2`。

原因：

- 这是 `v0.14.0` 和 `v0.14.1` 后台详情页体验链路的补丁，不是新产品模块。
- 范围集中在管理后台现有页面的信息密度、按钮稳定性和列表 CRUD 形态。
- 不新增 DDL，不改变 SSO、权限模型或部署拓扑，适合补丁版本发布和 `192.168.2.112` 停机升级验证。

## 4. 目标

- 应用详情的新增回调地址操作在桌面和窄屏下稳定展示，不出现按钮内部断行、挤压、遮挡或异常高度。
- 前端设计规范补齐按钮文字、操作区和响应式控件稳定性检查要求，避免同类问题重复出现。
- 权限管理角色列表移除应用快捷查询区域，首屏更聚焦筛选栏和角色表格。
- 应用详情页 `角色管理` Tab 改为普通后台列表 CRUD，便于扫描、比较、新增、编辑、查看和启停角色。
- 继续保持 `v0.14.1` 应用详情独立页面、四个 Tab 和 `tab` URL 状态不回退。

## 5. 纳入范围

### 5.1 GitLab issue #18：回调地址按钮与设计规范

- 修复应用详情开发信息区域中新增回调地址按钮的文字换行和形态变形问题。
- 按钮文字默认不在按钮内部断行。
- 输入框和操作按钮在桌面下保持一行或稳定的组合布局；空间不足时允许整个操作组换行，但不允许单个按钮文字断裂。
- 窄屏下可采用整组换行、短文案、图标加 tooltip 或等价策略，但必须保持可读、可点击和可访问。
- `DESIGN.md` 增补按钮文字、工具栏、表单操作区、表格操作列和响应式断点检查规则。

### 5.2 GitLab issue #22：权限管理快捷查询移除

- 移除权限管理角色列表筛选区下方的应用快捷查询区域。
- 保留并验证现有 `应用` 下拉筛选作为选择应用的主入口。
- 角色查询、状态、排序、查询、重置和分页语义保持不变。
- 移除后角色表格上移，首屏角色列表可见性提升。

### 5.3 GitLab issue #23：应用内角色管理列表 CRUD

- 应用详情页 `角色管理` Tab 不再使用大卡片堆叠展示角色。
- 角色列表改为表格或等价高密度列表。
- 列表至少展示：
  - 角色名称
  - 角色 key
  - 状态
  - 描述摘要
  - 创建时间
  - 更新时间
  - 操作列：查看、编辑、启用或停用
- 顶部保留 `新增角色` 主操作。
- 新增角色和编辑角色使用弹框或等价轻量表单，包含字段级校验、提交中状态、成功反馈和失败提示。
- 启用或停用角色必须保留确认流程和稳定状态反馈。
- 查看入口可以使用轻量详情、跳转到权限管理角色详情页或当前页面内只读详情，但不得把角色元数据维护入口重新放回权限管理。
- 长 role key 需要截断、tooltip、复制或等价处理，不得撑破表格或弹窗。

## 6. 不纳入范围

- 不纳入 GitLab issue `#19/#20/#21`。
- 不改飞书同步着陆页结构，不移除飞书同步页面内部重复分栏。
- 不引入成熟 Tree / TreeSelect 组件。
- 不重做飞书同步部门树。
- 不重做角色组织与用户绑定选择器。
- 不新增角色最终权限点一览。
- 不新增权限组内权限点查看。
- 不改变后端 API、Prisma、DDL、权限模型、SSO 协议、管理员 session 或生产部署拓扑。
- 不恢复权限管理中的角色元数据新增、编辑或启停；角色元数据继续归应用管理承载。
- 不实现完整 OIDC、SAML、ABAC、资源级权限、数据范围权限、飞书角色同步、飞书用户组同步、HTTPS、反向代理、高可用或滚动升级。

## 7. 设计和工程风险

- `ApplicationDetailSheet` 当前同时承载应用详情页面和旧抽屉兼容入口；修改 `角色管理` Tab 时必须保持独立页和旧 `sheet=app:*` 入口都可用。
- `v0.14.1` 已要求应用详情 Tab 状态进入 URL。本版本不能破坏 `tab=roles`、`tab=development`、`tab=danger` 的刷新保留能力。
- 应用内角色管理只能维护角色元数据；组织与用户绑定、权限组绑定和权限点可解释性不在本版本处理，避免和 `#19/#20/#21` 混线。
- `#18` 不应只做局部 CSS 修补；必须把按钮内部断行、操作区拥挤和响应式降级规则写入 `DESIGN.md`。
- 移除权限管理快捷查询后，需要确认无应用、应用加载失败、权限不足和空角色列表仍有清晰提示。
- 角色列表改成表格后，窄屏必须支持横向滚动或稳定降级，不能牺牲操作列可用性。

## 8. 验收标准

- 应用详情 `开发信息` Tab 中新增回调地址按钮在常见桌面宽度下不换行、不变形。
- 390px 窄屏下新增回调地址输入和按钮区域仍可用，文字、图标和点击区域无遮挡或异常高度。
- `DESIGN.md` 已补充按钮文字、操作区和响应式控件稳定性规则。
- 权限管理页面筛选栏下方不再出现应用快捷查询按钮区域。
- 权限管理的应用筛选、角色查询、状态筛选、排序、查询、重置和分页不回退。
- 应用详情 `角色管理` Tab 使用普通列表或表格展示角色，不再大卡片堆叠。
- 可以从 `角色管理` Tab 新增角色，新增流程有字段校验、提交中状态、成功反馈和失败提示。
- 可以从 `角色管理` Tab 编辑角色，编辑流程能加载当前值并保存。
- 可以从 `角色管理` Tab 查看角色关键信息。
- 可以从 `角色管理` Tab 启用或停用角色，危险操作有确认流程。
- 桌面和 390px 窄屏下，表格、操作列、弹框、分页、按钮和长 role key 无明显错位、遮挡、溢出或文字变形。
- 浏览器 console 无新增错误，Network 无非预期失败请求。
- `pnpm check`、前端定向测试、管理后台构建、响应式检查和 Browser 自检通过。
