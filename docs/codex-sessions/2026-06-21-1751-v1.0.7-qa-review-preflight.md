## 会话目标

继续推进 Feishu IAM `v1.0.7 - 权限管理信息架构收敛` 的 my-harness 后半段流程，完成实现后视觉 QA、系统化功能 QA、代码 review 修复和 Step 13 ship preflight。

## 用户关键要求摘要

- 一个版本完成 S1 角色工作台减负和 S2 权限矩阵 MVP，不拆分到 `v1.0.8`。
- 严格遵守 `AGENTS.md`、`DESIGN.md`、PRD、设计审计、视觉目标、工程计划和已完成 review 文件。
- 按 gstack / my-harness 流程推进，但不要进入 Plan mode，也不要调用交互式选择工具。
- 除非明确授权，不执行 push、tag、release、镜像上传或 deploy。
- `.my-harness/` 只保存快速索引和简短摘要，第三方制品保留在其规范目录。

## 本次使用或形成的关键约束

- `v1.0.7` 不新增 DDL，不扩大权限模型，不新增 ABAC、资源级权限、deny 规则或数据范围权限。
- 权限管理必须继续遵守 shadcn/ui + tweakcn 后台系统基线。
- 角色-应用绑定软解除使用现有 `iam_role_applications.status=disabled`，不可硬删除。
- 权限矩阵是只读 MVP，负责最终权限和来源解释，不提供编辑入口。
- 响应式至少覆盖 390px 窄屏。

## 重要设计和工程决策

- 修复 review 发现的 P1：disabled 角色-应用绑定不得继续作为 active 关联。
  - 后端 app-scoped `listRoles` 和 `getRole` 只接受 active binding。
  - 前端角色应用 tab、角色列表应用摘要和 Dialog 已关联列表只展示 active binding。
  - disabled binding 进入可添加列表，并标记为可恢复绑定。
- 修复 review 发现的 P2：`管理角色关联应用` Dialog 增加按应用名称 / app key 搜索。
- Step 13 preflight 确认 v1.0.6 历史记录和 v1.0.7 本次业务改动需要在 ship 时明确提交边界。
