## Runtime 验收覆盖

- `/api/health` 通过。
- `/api/feishu/events` 可原样返回 URL verification challenge。
- 通讯录事件写入 `sync_events`，状态为 `pending_sync`。
- 相同 `event_id` 重复投递幂等返回，不重复写入。
- 平台管理员可查看事件健康状态和事件列表。
- 事件重试使用 system actor 触发同步 run，并把事件标记为 `processed`。
- 验收脚本扫描输出，不包含 app secret、tenant access token、authorization code、bearer token、cookie 或事件签名。

## 剩余风险

- 本版本不实现后台自动消费队列或告警平台；事件处理仍以平台管理员可见和手动重试为第一刀。
- 真实飞书事件订阅仍需要在飞书开放平台手动配置回调地址、Verification Token、Encrypt Key、通讯录事件权限和部署环境白名单。
