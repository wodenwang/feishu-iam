# v0.1 Access Loop Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the v0.1 "第三方系统接入闭环" foundation: a product spec, a testable Ant Design Admin Console prototype, and acceptance gates for OAuth/SSO, Application API, role authorization, sync, audit, and the third-party demo.

**Architecture:** Work docs-first because the repository is still a skeleton and the approved design says the v0.1 product spec is the next gate. The first executable slice locks the v0.1 contract in `docs/v0.1-product-spec.md`, then frontend work uses typed mock services and TanStack Query so the UI can later swap to real APIs without rewriting pages.

**Tech Stack:** React, TypeScript, Vite, Ant Design, React Router, TanStack Query, Vitest, Testing Library, Playwright, Markdown documentation.

---

## Scope Check

v0.1 contains several subsystems: product spec, Admin Console frontend, backend OAuth/Application API, Feishu sync, third-party demo, Docker distribution. Treat this file as the v0.1 master plan. Execute it in slices; each slice must leave the repo testable and shippable as a checkpoint.

Recommended execution order:

1. Product spec contract.
2. Admin Console mock frontend.
3. Backend API and persistence.
4. `feishu-iam-thirdpart-demo`.
5. Docker Compose, install, upgrade, and release docs.

Do not start backend or frontend code until Task 1 is committed. The spec defines route names, API boundaries, secret handling, and demo acceptance.

```markdown
# feishu-iam v0.1 产品规格说明

状态：APPROVED_FOR_IMPLEMENTATION
适用版本：v0.1.0
产品主线：第三方系统接入闭环

## 1. 目标

v0.1.0 的目标是证明一个第三方业务系统可以完整接入 `feishu-iam`：

1. 平台管理员通过飞书登录 IAM Admin Console。
2. 平台管理员完成首次超级管理员绑定和飞书通讯录同步。
3. 平台管理员创建一个 demo 应用。
4. IAM 生成应用接入配置、OAuth 配置、Application API 凭证和 Agent Prompt。
5. 第三方 demo 系统按配置完成 IAM 登录。
6. 第三方 demo 系统通过 Application API 注册权限组和权限点。
7. 平台管理员或应用管理员创建角色，并把角色绑定到飞书组织或用户。
8. demo 用户通过飞书登录 demo 系统。
9. demo 系统向 IAM 查询当前用户在当前应用下的权限点。
10. demo 系统按权限点控制页面或接口访问。
11. IAM 审计日志能回溯登录、授权、权限查询、应用 API 调用和同步任务。

## 2. 不可协商边界

- 飞书是组织结构和用户的唯一来源。
- 不新增独立 username/password 登录系统。
- 所有登录和认证流程依赖飞书。
- 系统超级管理员身份绑定到飞书用户。
- 系统绑定一个专用自建飞书应用。
- 所有飞书 API 权限来自该专用自建飞书应用。
- 飞书应用凭证、tokens、导出的用户列表、同步快照不得提交到仓库。
- 项目文档和代码注释使用中文，技术名词保留英文。

## 3. 用户角色

### 平台管理员

- 通过飞书登录 Admin Console。
- 完成首次初始化和超级管理员绑定。
- 创建、查看、停用应用。
- 管理应用管理员。
- 触发飞书组织用户同步。
- 查看同步结果和失败原因。
- 创建角色并绑定权限组、权限点、飞书组织、飞书用户。
- 查看全局审计日志。
- 复制应用接入配置和运行时密钥。

### 应用管理员

- 只能查看自己所属应用。
- 查看应用接入信息。
- 复制本应用 Agent Prompt。
- 复制本应用运行时密钥。
- 查看本应用注册的权限组和权限点。
- 管理本应用下的角色授权。
- 查看本应用相关审计日志。
- 不能创建或停用应用。
- 不能触发飞书全局同步。
- 不能查看其他应用。
- 不能查看全局审计日志。

### 第三方系统开发人员

- 不必须登录 Admin Console。
- 通过应用管理员或平台管理员导出的 Agent Prompt、`.env` 配置和 API 文档完成接入。

## 4. 最短接入路径

```text
平台管理员飞书登录
  -> 首次初始化并绑定 Feishu super admin
  -> 手动同步飞书组织和用户
  -> 创建 demo 应用
  -> 复制 OAuth 配置、Application API 配置、Agent Prompt
  -> demo 系统写入环境变量
  -> demo 系统发起 IAM OAuth 登录
  -> demo 系统注册权限组和权限点
  -> IAM 创建角色并绑定飞书组织或用户
  -> demo 用户飞书登录 demo 系统
  -> demo 系统查询当前用户权限点
  -> demo 页面或接口按权限点生效
  -> IAM 审计日志可查询全链路
```

## 5. 页面与 API 对照

| 步骤 | 用户 | Admin Console 页面 | IAM API | 产物 |
|---|---|---|---|---|
| 飞书登录 | 平台管理员 | `/login` | `GET /api/auth/feishu/start`, `GET /api/auth/feishu/callback` | IAM session |
| 首次初始化 | 平台管理员 | `/initialize` | `POST /api/bootstrap/super-admin` | Feishu user 绑定记录 |
| 同步通讯录 | 平台管理员 | `/sync` | `POST /api/sync/runs`, `GET /api/sync/runs` | 组织和用户投影 |
| 创建应用 | 平台管理员 | `/applications` | `POST /api/admin/applications` | app key、回调配置 |
| 查看接入配置 | 平台管理员、应用管理员 | `/applications/:id` | `GET /api/admin/applications/:id/integration` | OAuth 和 API 配置 |
| 生成 Agent Prompt | 平台管理员、应用管理员 | `/applications/onboarding` | `GET /api/admin/applications/:id/agent-prompt` | 默认不含真实 secret 的提示词 |
| 注册权限组 | 第三方系统 | API 文档 | `PUT /api/application/permission-groups` | permission group |
| 注册权限点 | 第三方系统 | API 文档 | `PUT /api/application/permission-points` | permission point |
| 角色授权 | 平台管理员、应用管理员 | `/roles` | `POST /api/admin/roles`, `PUT /api/admin/roles/:id/bindings` | 授权关系 |
| 查询用户权限 | 第三方系统 | API 文档 | `GET /api/application/me/permissions` | permission point codes |
| 审计回溯 | 平台管理员、应用管理员 | `/audit-logs` | `GET /api/admin/audit-logs` | 审计记录 |

## 6. OAuth / SSO v0.1 边界

v0.1 实现 OAuth2 Authorization Code 风格的最小闭环。

包含：

- IAM Admin Console 使用飞书登录。
- 第三方应用发起 IAM OAuth 登录。
- IAM 校验第三方应用的 `client_id`、回调地址和状态参数。
- IAM 使用飞书身份解析用户。
- IAM 为第三方应用建立授权结果。
- 第三方应用可通过 IAM 查询当前用户权限点。

不包含：

- SAML。
- 多租户 SaaS IdP。
- 完整 OIDC discovery。
- 本地 username/password 登录。
- 第三方社交登录。
- 复杂 consent 页面。
- refresh token 轮换策略。

## 7. Secret 处理

- 默认 Agent Prompt 使用环境变量占位符，例如 `IAM_APP_SECRET` 和 `IAM_API_SECRET`。
- 默认 Agent Prompt 不包含真实 secret。
- 查看、复制、轮换、重置 secret 必须二次确认。
- 查看、复制、轮换、重置 secret 必须写审计日志。
- UI 必须提示不得把 secret 写入 `AGENTS.md`、`CLAUDE.md`、README、示例代码或 Git。

import { applications, auditLogs, platformAdminSession } from './mockData';
import type { Application, AuditLog, CreateApplicationInput, CurrentSession, PageRequest, PageResult } from './types';

const wait = (ms = 80) => new Promise((resolve) => window.setTimeout(resolve, ms));

const paginate = <T>(items: T[], request: PageRequest): PageResult<T> => {
  const start = (request.page - 1) * request.pageSize;
  return {
    items: items.slice(start, start + request.pageSize),
    total: items.length,
    page: request.page,
    pageSize: request.pageSize,
  };
};

export async function getCurrentSession(): Promise<CurrentSession> {
  await wait();
  return platformAdminSession;
}

export async function listApplications(request: PageRequest & { keyword?: string }): Promise<PageResult<Application>> {
  await wait();
  const keyword = request.keyword?.trim().toLowerCase();
  const filtered = keyword
    ? applications.filter((item) => item.name.toLowerCase().includes(keyword) || item.code.toLowerCase().includes(keyword))
    : applications;
  return paginate(filtered, request);
}

export async function createApplication(input: CreateApplicationInput): Promise<Application> {
  await wait();
  const now = new Date().toISOString();
  const app: Application = {
    id: `app_${input.code.replaceAll('-', '_')}`,
    name: input.name,
    code: input.code,
    status: 'active',
    appKey: `app_${input.code}_${Date.now()}`,
    appSecretPreview: 'sec_****_new',
    apiKey: `api_${input.code}_${Date.now()}`,
    apiSecretPreview: 'api_****_new',
    callbackUrls: input.callbackUrls,
    allowedOrigins: input.allowedOrigins,
    ownerFeishuUserId: input.ownerFeishuUserId,
    agentPrompt: [
      '使用环境变量接入 feishu-iam。',
      'IAM_APP_SECRET=${IAM_APP_SECRET}',
      'IAM_API_SECRET=${IAM_API_SECRET}',
      '禁止把真实 secret 写入 AGENTS.md、CLAUDE.md、README、示例代码或 Git。',
    ].join('\n'),
    createdAt: now,
    updatedAt: now,
  };
  applications.unshift(app);
  return app;
}

export async function listAuditLogs(request: PageRequest & { action?: AuditLog['action'] }): Promise<PageResult<AuditLog>> {
  await wait();
  const filtered = request.action ? auditLogs.filter((item) => item.action === request.action) : auditLogs;
  return paginate(filtered, request);
}
