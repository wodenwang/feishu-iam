## 发现与处理

1. `server:build` 原本只执行 TypeScript typecheck，没有产出 Docker runtime 需要的 `dist-server`。
   - 处理：改为 `tsc --noEmit` 做类型检查，再用 Vite SSR build 生成 `dist-server/main.js`，并复制 SQL migrations。
   - 复验：`npm run server:build` 通过；Docker build 通过。
2. 首次 Docker build 的前端仍显示 `Mock data`。
   - 原因：`VITE_IAM_API_MODE` 是 Vite 构建期变量，Docker build 未显式设置。
   - 处理：在 Dockerfile build stage 设置 `ARG/ENV VITE_IAM_API_MODE=http`，默认同源 HTTP mode。
   - 复验：Playwright 截图 `local-compose-login-http-1440.png` 显示 `HTTP runtime`。
3. GitHub Actions 上部分既有 Ant Design 多步交互测试超过默认 5 秒。
   - 原因：GitHub runner 比本机慢，且相关测试包含 Drawer、Popover、Table、TreeSelect 等多步交互。
   - 处理：将 frontend Vitest 全局 `testTimeout` 调整为 10 秒，匹配本项目既有长交互测试耗时。
   - 复验：重新推送后以 GitHub Actions `verify` 为准。
