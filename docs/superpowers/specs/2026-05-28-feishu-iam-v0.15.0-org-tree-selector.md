# Feishu IAM v0.15.0 组织树与组织用户选择器规格

日期：2026-05-28
状态：实施前规格

## 1. 背景

`v0.14.2` 已收口应用详情 Tab、角色管理列表 CRUD 和后台信息密度问题。当前 GitLab 仍有三类打开事项：

- `#19`：飞书同步着陆页重复展示页面内二级侧栏，本地部门树仍是列表式表达。
- `#20`：飞书同步部门树和角色组织用户绑定需要统一成熟 Tree / TreeSelect 类交互。
- `#24`：应用详情 `角色管理` Tab 的 `停用` 按钮仍有文字换行和操作列变形问题。

本版本定位为 `v0.15.0`，主线是“组织树和组织用户选择体验重构”，同时合并 `#24` 这种小 UI 回归修复。`#21` 的角色最终权限点一览和权限组内权限点查看属于权限可解释性主题，留到后续小版本处理。

## 3. 版本号

推荐版本号：`v0.15.0`。

原因：

- `#19/#20` 不只是局部样式修复，而是引入可复用组织树数据模型和组织用户选择器，属于新的后台复杂交互能力。
- `#24` 单独发补丁价值偏小，合并到本版本作为回归修复更合适。
- 本版本不改权限模型、SSO 协议、飞书同步后端逻辑或部署拓扑，仍能保持清晰的验收边界。

## 4. 目标

- 飞书同步页面主体移除重复的 `系统管理 / 飞书同步` 内嵌侧栏，让页面直接聚焦同步健康、组织用户、字段诊断和同步历史。
- 本地部门树使用真正的 Tree 组件表达组织层级，支持展开、收起、当前节点、搜索命中和部门/用户视觉区分。
- 角色详情的组织与用户绑定改为统一组织用户选择器，采用左侧待选、右侧已选的双栏模式。
- 组织用户选择器复用同一套节点数据模型、搜索策略、选择状态和已选汇总逻辑，避免飞书同步和角色绑定各写一套一次性列表。
- 应用详情 `角色管理` Tab 操作列稳定，`停用` 或 `启用` 按钮不再换行、挤压或导致行高异常。

## 5. 纳入范围

### 5.1 GitLab issue #19：飞书同步页面结构和本地部门树

- 移除飞书同步页面主体内重复的 `系统管理 / 飞书同步` 左侧栏。
- 页面主体直接呈现飞书同步工作区。内部导航如仍需要分区，应使用顶层 Tab、分段控件或清晰区块，不再嵌套系统管理侧栏。
- 本地部门树改为 Tree 组件或项目内可复用 Tree 封装。
- Tree 节点至少支持：
  - 层级缩进。
  - 展开和收起。
  - 当前选中节点。
  - 部门节点和用户节点的视觉区分。
  - 搜索后的过滤、命中高亮或展开父级策略。
  - 键盘焦点和基础可访问语义。
- 飞书同步页面使用只读或轻量浏览模式，不承担角色绑定保存逻辑。

### 5.2 GitLab issue #20：统一组织用户选择器

- 建立一套可复用的组织/用户树节点模型，能表达：
  - 节点类型：部门、用户。
  - 稳定节点 ID。
  - 父子关系。
  - 展开状态。
  - 是否可选。
  - 选中、半选或禁用状态。
  - 节点主标题、辅助文本、图标和标签。
- 角色详情 `组织与用户绑定` 使用双栏选择器：
  - 左侧为待选区，展示组织树和组织下用户。
  - 顶部搜索覆盖用户姓名、部门名称、`user_id`、部门 ID 等本地镜像字段。
  - 用户和部门均可勾选，二者在视觉和已选结果中清晰区分。
  - 右侧为已选区，分别展示已选用户和已选部门数量及列表，并支持移除单项。
  - 保存前展示本次变更摘要，包括新增用户、移除用户、新增部门和移除部门。
  - 保存成功后刷新角色详情，并保留当前角色详情上下文。
- 选择器在桌面下优先使用双栏布局；窄屏下可上下堆叠或使用适合移动端的分区布局，但必须保持搜索、勾选、移除和确认可用。
- 如当前本地镜像接口已能满足数据展示，优先在前端构造成树；如缺少必要字段，只补充最小 API 响应字段，不新增 DDL。

### 5.3 GitLab issue #24：应用详情角色管理操作列稳定性

- 修复应用详情 `角色管理` Tab 表格操作列中 `停用` 或 `启用` 按钮文字换行和按钮形态变形问题。
- 操作列宽度、按钮间距和按钮尺寸应稳定。
- 空间不足时可以改用图标按钮、短文案、操作菜单或更稳定的列宽策略，但不能让按钮文字在内部断行。
- 危险操作仍保留清晰的危险样式、确认流程和可访问标签。
- 本修复不改变应用内角色管理的业务模型、后端接口或信息架构。

## 6. 不纳入范围

- 不纳入 GitLab issue `#21`。
- 不实现角色最终权限点一览。
- 不实现权限组内权限点查看。
- 不实现最终权限点按 key 或标题搜索。
- 不改变权限模型，不新增 deny、ABAC、资源级权限或数据范围权限。
- 不新增 Prisma DDL。
- 不改变 SSO、OAuth、管理员 session、管理员权限校验或生产部署拓扑。
- 不实现飞书实时通讯录树；本版本继续基于本地镜像数据。
- 不实现飞书角色同步或飞书用户组同步。
- 不恢复权限管理中的角色元数据新增、编辑或启停；角色元数据继续归应用管理。
- 不把权限点和权限组 CRUD 做成后台常驻主流程。

## 7. 组件和数据边界

### 7.1 组织树节点模型

组织树相关组件应围绕统一节点模型设计，而不是直接绑定某个页面的接口响应。节点模型至少覆盖部门和用户两类节点，并允许不同页面决定是否展示 checkbox、是否允许选中用户、是否允许选中部门。

飞书同步页面使用浏览模式：展示部门层级、当前节点和搜索命中，必要时展示节点下用户，但不提供保存动作。

角色组织用户绑定使用选择模式：展示 checkbox、已选区、变更摘要和保存动作。

### 7.2 页面集成

飞书同步页面应保留 `v0.13.0` 已完成的同步总览、组织与用户查询、同步历史详情、字段诊断、轻量同步和全量同步强确认。`v0.15.0` 只重排页面主体和组织树表达，不削弱排障主旅程。

角色详情继续作为权限配置主工作区。组织与用户绑定在角色详情内完成，权限组绑定继续保留现有能力。本版本只替换组织与用户绑定交互，不把角色基础信息管理放回权限管理。

## 8. 错误处理和状态

- 组织树数据加载中、为空、加载失败和搜索无结果必须有稳定中文提示。
- 搜索输入为空时恢复默认组织树状态。
- 搜索命中较多时需要保持可扫描，不得造成页面明显卡顿或布局跳动。
- 保存角色组织用户绑定失败时，错误展示在选择器或提交区附近，不暴露堆栈、token、cookie、secret 或框架默认错误。
- 保存中状态应禁用重复提交，成功后提供稳定反馈并刷新当前数据。
- 飞书同步页面中只读树加载失败不应影响同步历史、字段诊断等其他区块的错误边界。

## 9. 验收标准

- 打开 `系统管理 / 飞书同步` 后，页面主体不再出现重复的系统管理内嵌侧栏。
- 飞书同步页面首屏直接展示同步工作区，组织用户区使用 Tree 表达本地部门层级。
- Tree 支持展开、收起、当前节点、部门/用户视觉区分和搜索命中展示。
- 角色详情 `组织与用户绑定` 使用左待选、右已选的双栏选择器。
- 可以搜索用户姓名、部门名称、`user_id` 或部门 ID。
- 可以勾选用户和部门，已选区能区分用户和部门并支持移除。
- 保存前展示新增和移除摘要，保存成功后角色详情数据刷新。
- 应用详情 `角色管理` Tab 操作列按钮不换行、不变形，行高稳定。
- 桌面和 390px 窄屏下，树、选择器、搜索框、已选区、底部操作、表格操作列无明显错位、遮挡、溢出、异常空白或按钮文字断裂。
- 浏览器 console 无新增错误，Network 无非预期失败请求。
- `pnpm check`、相关前端测试、管理后台构建和 Browser 自检通过。
