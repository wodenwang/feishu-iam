## ISSUE-001：E2E runtime DB 隔离不足

等级：High

现象：在先运行 `server:test` 后，再运行 v0.1.3 HTTP mode E2E，`/applications` 显示 403，测试找不到“创建应用”按钮。

根因：`server:test` 会重置并写入本地 runtime DB，最后残留的 `platform_admins` 可能不是 E2E 固定 mock 用户 `ou_v012_verify_admin`。E2E 只登录固定 mock 用户但不保证数据库初始化前置状态，因此进入 Admin Console 后没有 `application:create` 权限。

修复：

- 新增 Playwright `globalSetup`。
- 当 E2E 进程提供 `DATABASE_URL` 或 `TEST_DATABASE_URL` 时，开始前重置并迁移 runtime DB。
- README 补充 v0.1.3 HTTP mode E2E 命令，说明 DB 会被重置。

复验：

- `DATABASE_URL=postgres://feishu_iam:feishu_iam@127.0.0.1:55432/feishu_iam_manual VITE_IAM_API_MODE=http npm run e2e -- tests/e2e/v0.1.3-admin-console-http.spec.ts`
  - 结果：3 个项目通过，覆盖 `desktop-1440`、`laptop-1280`、`tablet-768`。
