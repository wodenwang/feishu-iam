# Feishu IAM v0.13.0 飞书同步运维控制台设计规格

日期：2026-05-27
状态：已通过原型复评和工程评审，待进入实施计划

## 1. 背景

`v0.12.0` 已完成 `feishu-iam-sso-demo` 真实第三方接入验收，证明第三方系统可以通过 Feishu IAM 完成登录、授权码换 token、读取 userinfo、查询权限清单，并根据权限结果控制前端展示和后端接口。

下一版本定为 `v0.13.0`，范围锁定 GitLab issue `#13`：把 `系统管理 / 飞书同步` 从当前偏状态页和同步历史页的形态，升级为面向运维管理者的数据同步控制台。

这个版本不继续扩大 SSO 协议面，也不新增应用或权限模型能力。它服务的是 Feishu IAM 的身份主数据可信性：飞书是唯一身份和组织来源，后续登录排障、角色授权、组织关系核对和第三方系统权限判断，都依赖本地飞书镜像可查、可解释、可修复。

## 2. 目标

`v0.13.0` 的目标是让运维或平台管理员能在后台直接回答以下问题：

1. 当前 Feishu IAM 与飞书的同步状态是否可信。
2. 本地镜像中某个飞书部门、用户和用户部门关系到底是什么样子。
3. 最近一次同步失败在哪里，失败原因和排查建议是什么。
4. 是否可以只对某个用户、部门或诊断项做轻量刷新，而不是每次都触发全量同步。
5. 全量同步作为重操作触发前，管理员是否清楚影响范围、审计含义和查看结果的位置。

## 3. 纳入范围

### 3.1 同步总览

- 展示配置状态、最近同步、当前运行任务、有效用户数、有效部门数、失败风险和字段诊断摘要。
- 明确区分 `running`、`success`、`failed`、`stale running`、`config failed`、`not configured` 等状态。
- 保留已有可信诊断能力，继续展示失败阶段、request id、run id 和稳定中文错误摘要。
- 总览不把“全量同步”作为默认最醒目的日常主操作。

### 3.2 本地组织和用户查询

- 支持搜索本地飞书部门，查看部门基础字段、父部门、子部门、部门下用户、启停或 orphaned 状态和最近同步时间。
- 支持搜索本地飞书用户，查看 `user_id`、`open_id`、`union_id`、姓名、邮箱或手机号脱敏展示、状态、所属部门和最近同步时间。
- 支持按部门树浏览本地组织结构，但本版本只读取本地镜像，不直接调用飞书通讯录实时树。
- 长 ID、邮箱、request id 等字段必须带清晰字段标签，并提供截断、换行或复制能力，不能撑破列表。

### 3.3 轻量同步和诊断操作

- 提供刷新字段诊断、配置连通性或 tenant token 诊断的轻量入口。
- 用户级和部门级轻量同步在 `v0.13.0` 必须真实可用，不能只交付占位入口。
- 实施时必须核对现有飞书 OpenAPI 能力和当前同步服务边界；若用户直读接口不可用，应采用基于本地用户所属部门的局部刷新 fallback，并把不可定位作为失败结果展示，不能伪造同步成功。
- 轻量操作必须记录操作者、来源、目标对象、操作类型、结果、IP、User-Agent、时间和 request id。

### 3.4 全量同步高级操作区

- 全量同步放入高级或危险操作区。
- 触发前必须强确认，说明影响范围、预计耗时、运行中限制、审计记录和结果查看方式。
- 已有同步运行中时，必须禁用重复触发并展示当前 run id、开始时间和状态入口。

### 3.5 同步历史和详情诊断

- 同步历史列表只展示关键字段：状态、触发来源、开始时间、结束时间、部门统计、用户统计、关系统计、错误摘要和详情入口。
- 详情中展示错误摘要、失败阶段、request id、run id、统计、阶段耗时和诊断建议。
- 不向浏览器展示明文 secret、token、cookie、飞书 app_secret、原始错误堆栈或不必要的原始 payload。

## 4. 不纳入范围

- 不实现完整 OIDC Discovery、JWKS、ID Token、SAML、默认 refresh token。
- 不实现 ABAC、资源级权限、deny 规则或数据范围权限。
- 不实现飞书角色同步或飞书用户组同步。
- 不把 Feishu IAM 改成多租户 SaaS。
- 不新增 HTTPS、反向代理、高可用、滚动升级或部署拓扑改造。
- 不改变生产路径的管理员 session 和管理员权限校验机制。
- 不恢复前端注入 `PLATFORM_ADMIN_TOKEN` 或破窗 Web 登录。

## 5. 信息架构和页面形态

入口保持在：

```text
系统管理 / 飞书同步
```

页面按后台运维控制台组织，而不是营销式 dashboard：

1. 顶部为 `PageHeader`，面包屑固定为 `后台 / 系统管理 / 飞书同步`。
2. 第一屏展示同步总览、当前风险和日常轻量操作。
3. 主工作区使用标签页或分区承载：
   - `组织与用户`
   - `同步历史`
   - `字段诊断`
   - `高级操作`
4. 组织、用户、同步记录详情使用 `DetailSheet`，复杂详情默认宽屏或填满。
5. 窄屏下保留查询、列表、详情和确认操作可用性，允许表格横向滚动，但不能出现元素遮挡或异常空白。

## 6. 后端契约建议

实施计划阶段应优先复用现有 `feishu` 模块和 `apps/api/prisma/schema.prisma` 中的本地镜像表：

- `FeishuDepartment`
- `FeishuUser`
- `FeishuUserDepartment`
- `FeishuSyncRun`

建议新增或整理为管理端 API：

```text
GET  /api/v1/admin/feishu/overview
GET  /api/v1/admin/feishu/departments
GET  /api/v1/admin/feishu/departments/{department_id}
GET  /api/v1/admin/feishu/users
GET  /api/v1/admin/feishu/users/{user_id}
GET  /api/v1/admin/feishu/sync-runs
GET  /api/v1/admin/feishu/sync-runs/{run_id}
POST /api/v1/admin/feishu/diagnostics/refresh
POST /api/v1/admin/feishu/sync-runs
```

用户级和部门级轻量同步接口作为设计目标保留：

```text
POST /api/v1/admin/feishu/sync-users/{user_id}
POST /api/v1/admin/feishu/sync-departments/{department_id}
```

如果实施阶段确认当前飞书 OpenAPI 或同步服务不适合一次性补齐这两个接口，本版本可以将其拆为后续版本，但必须在规格或实施计划中明确原因和替代验收标准。

## 7. 权限、安全和审计

- `platform_admin` 可以查看和触发所有飞书同步运维操作。
- `sync_admin` 可以查看飞书同步、刷新诊断和触发允许范围内的同步操作。
- 其他管理员如无权限，应看到稳定中文无权限说明，不发起敏感接口请求。
- 所有写操作和触发类操作必须写审计日志。
- 前端和文档不得记录明文密钥、token、cookie、密码、飞书 `app_secret` 或一次性 secret。
- 错误信息必须稳定、可排查，不暴露堆栈、内部异常对象或框架默认错误页。

## 8. 测试和验收

### 8.1 自动化验证

- 后端测试覆盖部门查询、用户查询、同步历史详情、权限不足、诊断刷新和全量同步触发确认路径。
- 前端测试覆盖组织/用户查询、详情抽屉、同步历史详情、全量同步确认、权限不足和错误状态。
- 运行项目相关检查，优先 `pnpm check`；如果范围过大，至少运行受影响包的类型检查、lint、单测和构建。

### 8.2 Browser 自检

涉及管理后台页面，完成实现后必须使用 `@Browser` 打开本地页面自检，默认入口：

```text
http://localhost:3000/
```

自检覆盖：

- 桌面和窄屏布局无明显错位、溢出、遮挡或异常空白。
- 同步总览、组织/用户查询、详情抽屉、同步历史、字段诊断和高级操作区可用。
- 全量同步确认不会被误触发，运行中状态能阻止重复触发。
- console 无新增错误，Network 无非预期失败请求。

### 8.3 生产验收

- 在 `192.168.2.112:~/feishu-iam` 完成停机升级验证。
- `/ready` 返回 `ready`，`/version` 返回 `0.13.0 / v0.13.0`。
- 生产后台可以搜索至少一个真实飞书用户和一个真实飞书部门。
- 可以查看该用户所属部门、该部门下用户和最近同步时间。
- 可以打开最近同步记录详情，看到 run id、统计、错误摘要或成功结果。
- 如触发全量同步，必须有强确认、运行状态和审计记录。

## 9. 文档和发布收口

版本收口时必须同步：

- `README.md`：Quick Start、版本历史、镜像或部署信息、相关文档索引。
- `CHANGELOG.md`：新增 `v0.13.0` 发布记录。
- `AGENTS.md`：当前阶段、优先事项和不做事项。
- `docs/codex-sessions/`：中文会话归档。
- 必要时补充飞书同步运维说明或验收文档。

## 10. 实施前置评审结论

本规格已完成 Pencil 原型复评和工程评审，实施计划必须同时遵守：

- `design/v0.13.0-feishu-sync-console-prototype.md`
- `docs/superpowers/reviews/2026-05-27-feishu-iam-v0.13.0-feishu-sync-console-eng-review.md`

进入实施计划时不再保留以下不确定项：

1. `v0.13.0` 只锁定 `#13` 飞书同步运维控制台。
2. 用户级和部门级轻量同步必须在本版本真实落地，不得只做 UI 占位或稳定“暂不支持”说明。
3. 生产验收默认不触发真实全量同步，只验证查询、详情、历史、诊断、轻量同步、审计和全量同步预确认路径；真实全量同步需要部署时再次明确授权。
