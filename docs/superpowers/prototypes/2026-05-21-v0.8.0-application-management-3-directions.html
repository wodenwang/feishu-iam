Feishu IAM v0.8.0 应用管理页设计 shotgun

目标是重构后台前端，但不改变现有业务模型和业务流程。以下三个方向都保留 v0.7.0 已确认边界：应用管理吸收 SSO 接入配置，权限管理仍按角色分配，权限组和权限点不在后台 UI 中做 CRUD。

只做原型，不改前端代码

共同约束

  • 三个方向都不改变现有业务模型：应用、环境、redirect URI、client、secret、IAM 角色、权限组绑定、审计和安全事件边界保持不变。
  • 三个方向都不把权限组和权限点 CRUD 放回后台 UI；只从“权限管理”里消费已有权限组。
  • 高风险动作使用确认弹窗、短时展示和审计追溯；前端不保存明文 secret，不在原型和文档中记录真实密钥。
  • 确认方向后再进入 v0.8.0 实施计划；本轮不修改现有 `apps/admin-web` 代码。
