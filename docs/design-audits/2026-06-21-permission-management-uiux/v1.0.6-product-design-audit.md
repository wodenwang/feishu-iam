# Feishu IAM v1.0.6 权限管理 UI/UX 产品设计调研

## 调研目标

本轮调研聚焦 `权限管理` 模块的角色配置工作台 UI/UX，不改变后端权限模型、不新增 DDL、不扩展 SSO 或权限协议面。

调研对象：

- 生产入口：`https://feishu-iam.riversoft.com.cn/admin/permissions`
- 角色配置工作台：`组织与用户`、`应用权限`、`总览`、`基础信息`
- 本地复现入口：`http://localhost:5173/admin/permissions`

## Product Design 工具状态

当前会话尝试加载 Product Design 对应工具，但可调用工具集中没有可直接使用的 Product Design `get-context` / `ideate` 工具。为保证版本继续推进，本轮采用以下替代设计证据：

- 项目设计基线 `DESIGN.md`。

- 本地 Playwright 三视口截图和结构审计。

## 用户标注问题

### 1. 组织与用户待选区高度被右侧撑开

生产标注位置：角色配置工作台 `组织与用户` Tab 的 `待选组织与用户` 区域。

问题判断：

- 左右两栏在桌面布局中使用 grid 时默认拉伸到同一行高度。
- 右侧已选摘要内容或后续区域变高时，左侧待选区被拉伸，导致控件间距和空白高度异常。

设计结论：

- 待选区应按内容自然高度展示。
- 桌面双栏父容器应使用 `items-start`。
- 左侧工作区应使用 `content-start`，避免内部 grid 子项被拉开。

### 2. 应用权限可选区高度异常，应用切换方式不合理

生产标注位置：角色配置工作台 `应用权限` Tab 的 `可选权限组` 区域。

问题判断：

- 左侧 `可选权限组` 也存在同类高度拉伸问题。
- 当前应用切换使用下拉框，会把“当前配置哪个应用”隐藏在表单控件里，不符合角色绑定多应用的工作台心智。

设计结论：

- 应用切换改为纵向 tab，展示应用名称、`appKey`、应用状态和绑定状态。
- 纵向 tab 保持常显，便于管理员理解正在编辑的应用上下文。
- `添加应用` 继续作为次级动作保留在同一工作区。

## 额外发现

### 3. 角色配置工作台标题重复

本地审计发现：

- `总览` 页面在首屏出现多个“角色配置工作台”语义标题。
- 页面顶栏、详情卡片和角色名称同时重复时，会消耗首屏空间，降低扫描效率。

设计结论：

- 页面级标题保留 `角色配置工作台`。
- 内容卡片标题改为 `角色上下文`。
- 总览基础信息区标题改为 `基础信息概览`，角色名称作为字段展示。

### 4. 绑定结果预览过长

本地审计发现：

- `应用权限` 右侧 `绑定结果预览` 在权限点对比数据较多时会形成非常长的列。
- 右侧栏变成长页面后，保存摘要、最终权限点和权限点对比之间缺少稳定边界。

设计结论：

- 桌面端右侧预览栏应作为 sticky 核对栏，设定最大高度并内部滚动。
- 权限点对比表应设定最大高度，横纵向都允许在表格内部滚动。
- `最终权限点` 保持当前列表上限，避免长来源文本撑破页面。

## 前端设计方案

### 组织与用户

- `OrgUserSelector` 桌面双栏父容器使用 `items-start`。
- `OrgBrowser` 待选区域使用 `content-start`。
- 不改变选择模型：组织主体和用户主体继续在同一列表中展示和保存。
- 不新增后端接口。

### 应用权限

- `GroupsTab` 顶层左右布局使用 `items-start`。
- `可选权限组` 区域使用 `content-start`。
- 当前应用使用纵向 tablist：
  - `role="tablist"`
  - `aria-orientation="vertical"`
  - 选中项使用 `aria-selected="true"`
  - 每个应用展示名称、`appKey`、状态、绑定状态。
- 移除当前应用下拉框。
- 保留 `添加应用` 下拉和按钮作为绑定新应用的次级动作。

### 总览与上下文

- 页面顶栏继续承载页面级标题。
- 内部详情卡片标题改为 `角色上下文`。
- 总览第一块标题改为 `基础信息概览`。
- 角色名称在字段中展示，避免标题重复。

### 绑定结果预览

- 桌面端 `绑定结果预览` 使用 sticky 核对栏。
- 右侧栏设置 `max-height` 和内部滚动。
- 权限点对比表设置 `max-height` 和 `overflow-auto`。
- 不隐藏权限点对比，不改为弹窗。

## 本版本不做

- 不新增 DDL。
- 不改变角色、应用、权限组、权限点的数据模型。
- 不改变管理员 session 或权限校验。
- 不恢复应用管理中的角色管理。
- 不实现完整移动端重构，只保证无横向溢出、无遮挡、基础链路可用。
- 不做全站 UI 主题改造。

## 完成标准

- 用户标注的两个高度异常在桌面视口消除。
- `应用权限` 当前应用切换改为纵向 tab。
- 角色工作台不再重复展示多个“角色配置工作台”标题。
- 权限点对比数据较多时，右侧预览栏不再无限拉长页面。
- Vitest focused tests 通过。
- admin-web typecheck、lint、build 通过。
- Playwright 三视口复查无 console error、无失败请求、无横向溢出。
