# feishu-iam v0.1.8 真实飞书登录工程评审

日期：2026-05-24
分支：`codex/v0.1.8-real-feishu-auth`

## 结论

本版本目标锁定为 `v0.1.8` 真实飞书 Admin Console 登录闭环。它只补齐平台管理员进入 IAM 的真实飞书身份路径，不实现第三方应用 OAuth server、通讯录全量同步、refresh token 轮换或完整 OIDC discovery。

## 已确认边界

- `FEISHU_AUTH_MODE=real` 下启用真实飞书登录。
- `/api/auth/feishu/start` 负责生成飞书授权地址并设置短期 state cookie。
- `/api/auth/feishu/callback` 负责校验 state、用 code 换取 `user_access_token`、获取用户信息、创建 IAM session。
- 不持久化 `user_access_token`、`refresh_token`、飞书 App Secret 或任何真实 token。
- 成功登录只写入 `feishu_users` 和最小 `directory_users` 投影。
- 首次平台管理员绑定继续通过 `/api/initialization/bind-platform-admin` 使用当前 session。
- 开发模式保留 mock 登录；生产模式继续拒绝 `FEISHU_AUTH_MODE=mock`。
- 真实飞书应用凭证只通过运行时环境变量注入，不提交仓库。

## 架构决策

### 1. 真实飞书能力放在 adapter 边界

扩展 `FeishuAuthAdapter`，新增真实登录所需方法：

- `buildAuthorizationUrl(state, redirectUri)`
- `exchangeCodeForUser(code, redirectUri)`

`RealFeishuAuthAdapter` 内部访问飞书 OpenAPI。测试通过 fake adapter 覆盖成功和失败路径，避免 CI 依赖真实飞书。

### 2. state 使用短期 HttpOnly cookie

本切片使用 `iam_oauth_state` HttpOnly cookie 存储随机 state，`Max-Age=300`，`SameSite=Lax`。callback 必须同时有 query state 和 cookie state，且完全匹配。callback 后立即清除 state cookie。

不新增数据库表，原因是 state 只服务单次浏览器跳转，短期 cookie 足以覆盖当前风险，且避免额外清理任务。

### 3. 登录 token 不落库

callback 只把 `user_access_token` 用于当次获取用户信息，不保存 token。当前 Admin Console session 由本系统 `iam_sessions` 控制，默认 8 小时。refresh token、offline access 和长期授权续期留给后续版本。

### 4. 用户投影保持最小

真实登录成功后写入：

- `feishu_users.feishu_user_id`
- `feishu_users.name`
- `feishu_users.email`
- `feishu_users.status`
- 对应 `directory_users` 最小记录

完整部门、手机号、离职状态和通讯录字段由后续 sync 子系统负责。

### 5. 前端登录采用浏览器跳转

HTTP mode 下 `/login` 的“使用飞书登录”直接 `window.location.href = /api/auth/feishu/start`。后端负责 redirect 到飞书授权页。callback 成功后后端 redirect 到 `/initialize` 或 `/applications`，由前端继续判断初始化和权限。

## 风险与处理

| 风险 | 处理 |
|---|---|
| OAuth state 被伪造或重放 | state 必须同时匹配 query 和 HttpOnly cookie；callback 后清除 cookie。 |
| Secret 泄露 | 只在 `.env.example` 保留占位符；真实 secret 不写入仓库、测试 fixture、日志和文档。 |
| 飞书 API 不稳定或返回错误 | adapter 把 token/user_info 失败转成明确 IAM error code，并写失败审计。 |
| 用户已登录但没有后台权限 | `GET /api/session/current` 返回 authenticated 但 permissions 为空，前端显示 403 或初始化页。 |
| 本地 E2E 无法访问真实飞书 | 用 fake adapter 和 route-level tests 覆盖 callback contract；真实飞书仅做人工 smoke。 |
| redirect URI 不匹配飞书后台配置 | README 和 deploy report 明确检查 `FEISHU_REDIRECT_URI` 与飞书后台配置一致。 |

## 测试策略

- `server/tests/auth.real-feishu.test.ts`
  - start endpoint 设置 state cookie 并 redirect 到飞书授权地址。
  - callback 缺失 code/state/cookie 被拒绝。
  - callback state mismatch 被拒绝。
  - callback 成功创建 session、upsert 用户、写审计。
  - 飞书 token/user_info 失败返回明确错误且不创建 session。
- `server/tests/env.test.ts`
  - real mode 必须具备 App ID、App Secret、redirect URI。
- 前端组件测试
  - HTTP mode 登录按钮触发真实 start URL。
  - 开发 mock 登录按钮仍只在 DEV + HTTP mode 显示。
- Browser/Playwright
  - 用本地 fake/mock 验证 `/login`、callback processing 或失败状态、`/initialize`、登录后后台页面。
- 验证命令
  - `npm run server:build`
  - `npm run server:test`
  - `npm test`
  - `npm run build`
  - `npm run e2e`
  - secret grep
