# feishu-iam v0.2.2 Access Diagnostics Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 交付 `v0.2.2 应用接入诊断与可观测性补强`，让第三方系统接入失败时可以从应用详情快速定位 redirect URI、secret、权限注册、角色授权、登录态、目录用户投影或应用 scope 问题。

**Tech Stack:** React 19、TypeScript、Vite、Ant Design、TanStack Query、Fastify、PostgreSQL、Vitest、Playwright。

## Scope

### In Scope

- `GET /api/applications/:id/diagnostics` 返回脱敏诊断摘要。
- `POST /api/applications/:id/diagnostics/copy` 写入 `application.diagnostics.copy` 审计。
- 应用详情新增 `接入诊断` Tab，展示健康状态、发现项、最近失败和修复建议。
- 支持复制 Markdown 诊断包，默认不包含 secret、token、authorization code、cookie、signature 或 hash。
- 新增 `scripts/verify-v0.2-access-diagnostics.sh` 自动验收正常和异常诊断状态。
- README 与 `docs/integration/thirdparty-demo.md` 增加接入失败排查矩阵。
- 更新 `VERSION`、`package.json`、`package-lock.json`、`CHANGELOG.md`。
- 增加 `design/implementation-screenshots/v0.2.2-access-diagnostics/` 证据目录。

### Out Of Scope

- OIDC discovery、JWKS、PKCE、refresh token、复杂 consent。
- SDK、CLI、Helm、Terraform。
- 飞书 webhook、event sync、incremental sync 或同步告警。
- `/directory` 编辑、导入、导出或目录治理。
- 多租户 SaaS IdP。
- username/password、本地 root 或独立账号体系。

## Data Contract

```ts
export type ApplicationDiagnosticStatus = 'healthy' | 'warning' | 'failed';
export type ApplicationDiagnosticSeverity = 'info' | 'warning' | 'critical';

export interface ApplicationDiagnosticFinding {
  code: string;
  severity: ApplicationDiagnosticSeverity;
  title: string;
  description: string;
  nextAction: string;
  relatedRequestId?: string;
}

export interface ApplicationDiagnosticRecentEvent {
  action: string;
  result: 'success' | 'failed';
  requestId: string;
  createdAt: string;
  message: string;
}

export interface ApplicationDiagnostics {
  applicationId: string;
  appKey: string;
  status: ApplicationDiagnosticStatus;
  checkedAt: string;
  endpoints: {
    oauthAuthorize: string;
    oauthToken: string;
    applicationPermissions: string;
  };
  redirectUris: {
    active: string[];
    disabled: string[];
  };
  secrets: {
    appSecret: { status: 'issued' | 'missing'; rotatedAt?: string };
    apiSecret: { status: 'issued' | 'missing'; rotatedAt?: string };
  };
  counts: {
    applicationAdmins: number;
    permissionGroups: number;
    permissionPoints: number;
    roles: number;
    roleBindings: number;
    syncedUsers: number;
  };
  findings: ApplicationDiagnosticFinding[];
  recentEvents: ApplicationDiagnosticRecentEvent[];
}
```

```ts
import type { ApplicationDiagnostics } from '../../features/iam/types';

export function buildApplicationDiagnosticsMarkdown(input: ApplicationDiagnostics): string {
  return [
    `# feishu-iam 应用接入诊断包`,
    ``,
    `- appKey: ${input.appKey}`,
    `- status: ${input.status}`,
    `- checkedAt: ${input.checkedAt}`,
    `- OAuth authorize: ${input.endpoints.oauthAuthorize}`,
    `- OAuth token: ${input.endpoints.oauthToken}`,
    `- Application permissions: ${input.endpoints.applicationPermissions}`,
    ``,
    `## Active Redirect URIs`,
    ...input.redirectUris.active.map((uri) => `- ${uri}`),
    ``,
    `## Findings`,
    ...input.findings.map((item) => `- [${item.severity}] ${item.code}: ${item.title}。下一步：${item.nextAction}${item.relatedRequestId ? `。requestId: ${item.relatedRequestId}` : ''}`),
    ``,
    `## Recent Events`,
    ...input.recentEvents.map((item) => `- ${item.createdAt} ${item.action} ${item.result} requestId=${item.requestId}`),
    ``,
    `## Secret Handling`,
    `- appSecret 和 apiSecret 必须从运行时环境变量或 secret manager 读取。`,
    `- 本诊断包不包含 secret 明文、token、authorization code、cookie、HMAC signature 或 hash。`,
  ].join('\n');
}
```
