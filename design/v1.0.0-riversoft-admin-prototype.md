## Riversoft Token 使用方式

| Token | 建议值 | 使用范围 | 约束 |
| --- | --- | --- | --- |
| `--primary` | `#0E76BD` | 主按钮、当前导航、链接、focus ring、关键操作图标 | 不用于大面积背景铺满，避免全站过蓝。 |
| `--blue-500` | `#1C8CCA` | hover、选中态、信息型强调 | 与 `--primary` 形成层级，不替代语义色。 |
| `--blue-400` | `#26A4D8` | 辅助图标、浅色状态提示 | 只做小面积点缀。 |
| `--foreground` | `#231C16` | 标题、正文、表格主文本 | 保持足够对比度，避免使用纯浅灰正文。 |
| `--sidebar` | `#0A2540` | 左侧导航、移动顶部栏 | 作为后台稳定框架色，不做渐变 hero。 |
| `--background` | `#F0F4F8` | 页面底色、分隔带 | 冷灰蓝背景衬托白色工作区。 |
| `--card` | `#FFFFFF` | 表格容器、表单区、详情区 | 使用细边框和轻阴影，不做漂浮卡片堆叠。 |
| `--lemon` | `#A6C511` / `#D8E000` | 小面积健康、图表或数量强调 | 不用于主按钮、危险按钮或大面积面板。 |
| `--radius` | `8px` | 按钮、输入框、表格容器、弹窗 | 保持 shadcn/ui 后台质感，避免过度圆角。 |

建议在实现阶段通过 tweakcn / Tailwind CSS variables 统一替换全局 token，再对少量项目级组件做定向修正。

## 组件视觉升级范围

| 组件 | 升级方向 | 不改变项 |
| --- | --- | --- |
| AppShell / Sidebar | 深蓝框架、当前项高亮、分组展开 chevron、移动 Sheet 视觉统一 | 不改变一级/二级导航结构和路由。 |
| PageHeader | 标题层级、面包屑、操作区密度统一 | 不新增页面入口或改变页面层级。 |
| DataTable | 表头层级、行高、固定操作列、横向滚动、状态 badge 对比度 | 不改变字段、排序、分页参数和行操作行为。 |
| StatusBadge | 启用、维护中、停用、成功、失败、同步中语义色统一 | 不改变状态枚举和后端语义。 |
| FormDialog / DetailSheet | 标题、主次按钮、错误展示、提交中样式统一 | 不改变表单字段、校验规则和提交流程。 |
| PageState | 空、错、加载 skeleton、重试入口样式统一 | 不新增业务动作，不暴露堆栈或敏感信息。 |
| Confirm / Danger Zone | 危险边框、确认文案、主次按钮层级 | 不启用常规硬删除，不改变软删除策略。 |
| Trace / Error Page | request id 输入、复制、跳转排障入口突出 | 不恢复整段问题信息复制、粘贴或本地提取。 |

## UX 不允许改变

- 不改变管理后台的路由、信息架构和导航分组。
- 不改变应用管理列表、应用详情页、角色授权工作区、飞书同步页、操作审计/追踪页、统一问题提示页的主流程。
- 不改变清单、筛选、详情、Tab、弹窗、确认、分页、表单提交、禁用/启用等 CRUD 行为。
- 不改变后端数据契约、字段命名、分页参数、状态枚举、权限校验和审计写入规则。
- 不把角色元数据管理恢复到权限管理；角色元数据能力继续归应用管理。
- 不新增第三方应用权限、demo client、回调地址或业务角色数据。
- 不展示真实 secret、token、cookie、飞书 app_secret、数据库密码、生产导出数据或 raw payload。
- 数据初始化只处理 IAM 管理权限，确保“王文哲”成为第一个平台管理员。

## 状态矩阵

| 状态 | 原型位置 | 视觉规则 | 实现注意 |
| --- | --- | --- | --- |
| 空状态 | 应用管理列表 | 图标 + 短说明 + 次级建议；背景使用冷灰蓝浅面板 | 不新增额外业务入口。 |
| 错误状态 | 统一问题提示页、表单错误区 | 红色语义色只用于错误边框、图标、标签；正文只展示中文说明和 request id | 不展示堆栈、secret、token、cookie 或 authorization。 |
| 加载状态 | 应用管理列表 skeleton、飞书同步提交中 | skeleton 行宽稳定，不推动表格列宽变化 | 加载时保留页面结构，避免跳动。 |
| 禁用态 | 角色授权工作区禁用权限点、不可用按钮 | 降低对比度但文字可读，保留 tooltip / title 解释 | 禁用来自后端权限或业务规则，前端不伪造授权。 |
| 提交中 | 飞书同步全量同步进度 | 主按钮展示 spinner / 进度文案，相关危险按钮不可重复提交 | 写操作期间禁止重复提交，失败后回到可操作态。 |
| 确认态 | 应用详情危险操作 | 红色边框、明确标题、主次按钮；危险按钮不自动聚焦 | 确认文案不改变删除/软删除业务策略。 |

## 响应式规则

| 视口 | 布局规则 | 组件规则 |
| --- | --- | --- |
| 1440 | 左侧导航固定，内容区展示完整表格、Tab、筛选、分页和主要操作；页面信息密度适中 | 操作列固定宽度，长 key / URL 截断并提供 tooltip 或复制。 |
| 768 | 保留后台工作区结构，导航可收缩；筛选区允许换行；详情页 Tab 不挤压主内容 | 表格启用横向滚动，按钮保持不换行，弹窗/抽屉宽度受视口约束。 |
| 390 | 使用移动 Header + Sheet 导航；筛选折叠为图标入口；表格横向滚动；底部或顶部主导航触控稳定 | 图标按钮和行操作不小于 44px，表单按钮纵向排列，文本不溢出容器。 |

## 可访问性要求

- focus ring 使用 Riversoft primary，至少 `2px` 可见，不被容器裁切。
- 当前导航项必须通过视觉高亮表达 `aria-current`。
- 可展开导航必须通过 chevron 方向和展开背景表达 `aria-expanded`。
- 所有图标按钮必须有 `aria-label`，并在桌面端提供 `title` 或 tooltip。
- 移动端可点击区域不小于 `44px`，包含筛选、详情、导航、复制、关闭、确认和取消按钮。
- 表单错误靠近字段展示；接口错误在稳定区域展示中文说明。
- 状态色不得作为唯一信息来源，必须配合文字、图标或标签。

## D1-A 迭代修正记录（2026-06-06）

基于 `/plan-design-review` 的 D1-A 推荐方案，对 v1.0.0 原型执行以下小范围修正：

### 1. 桌面 AppShell 恢复

**问题**: 360px 状态矩阵 mockup 使用深色 topbar（`$sidebar-bg`），可能误导为 IA/UX 改造（顶部导航替代侧边栏）。

**修正**: 
- 新增 `D1-A 桌面工作台 1440` 截图：232px 左侧 Sidebar（`#0F172A`）+ 1208px 内容工作区（`$background`），导航项「工作台」高亮为 `#1E293B`。
- 新增 `D1-A 桌面应用管理 1440` 截图：相同 AppShell 结构，导航项「应用管理」高亮。
- Sidebar 包含四条导航：工作台、应用管理、权限管理、系统管理，与现有 v0.16.1 后台结构一致。
- 360px 窄屏 mockup 保持不变（390px 使用移动 Header + Sheet 导航，符合响应式规则）。

### 2. 应用详情安全凭证表达

**问题**: 早期应用详情页将凭证信息表达为密钥字段，容易误导工程实现展示密钥形态。

**修正**:
- 应用详情截图改为 `凭证状态`、`最近轮换` 和“创建或轮换后仅展示一次”的安全表达。
- 新增 768px 平板应用详情截图示范安全凭证表达方式：凭证状态 + 最近轮换时间，不展示任何密钥原文。
