# feishu-iam v0.2.1 应用接入闭环补缺工程评审

## 结论

通过，按窄切片执行。`v0.2.1` 是 `v0.2.0 应用接入生产化` 之后的补缺版本，不开启新的协议产品化、目录治理或部署产品化范围。

## 版本边界

### In Scope

- 应用详情 `接入配置` 中提供一键复制第三方应用接入提示词，用于 Codex / Claude Code 创建或维护第三方项目 `AGENTS.md` / `CLAUDE.md`。
- 提示词包含明文 SSO 接入信息、redirect URI、OAuth 端点、Application API 端点、`appKey`、HMAC 鉴权规则和权限注册/查询约定。
- 角色授权支持绑定 permission group，并在权限查询时展开到组内启用权限点。
- 登录页支持 `loginRequired` 未登录引导，优化登录态检查、失败恢复和 Admin Console 视觉层级。
- 版本元数据更新为 `0.2.1` / `v0.2.1`，补充验证报告。

### Out Of Scope

- OIDC discovery、JWKS、PKCE、refresh token、复杂 consent。
- SDK、CLI、Helm、Terraform 或安装升级产品化。
- 飞书 webhook、event sync、incremental sync 或同步告警。
- `/directory` 编辑、导入、导出或目录治理。
- 多租户 SaaS IdP。
- username/password、本地 root 或独立账号体系。
- push、PR、tag、GitHub Release、merge 或 deploy，除非用户另行明确授权。

## 工程边界

### 应用提示词

提示词在前端由当前应用详情和 redirect URI 列表生成。后端继续只记录复制审计，不返回或保存 prompt 全文，也不重新暴露已 hash 的 `appSecret` / `apiSecret` 明文。

默认提示词只写入非 secret 接入配置和 secret 环境变量名。真实 secret 仍只能在创建或轮换后的一次性明文结果中复制，并要求二次确认。

### 角色权限组绑定

新增 `role_permission_groups` 表，保存角色到权限组的直接绑定。权限查询将直接权限点绑定和权限组展开后的权限点合并去重，并只返回启用角色、启用权限组、启用权限点下的 code。

前端 `RolesPage` 继续使用 Ant Design `Tree`，父节点代表权限组，子节点代表权限点；保存时拆分为 `permissionGroupCodes` 和 `permissionPointCodes`。

### 登录页

登录页不引入账号密码，不做营销页。保留企业后台密度和可恢复错误状态，新增 `loginRequired` 状态，给出重新发起飞书登录的明确动作。

## 测试策略

- 后端：`server/tests/roles.test.ts` 覆盖权限组绑定、权限查询展开、停用权限组语义和审计。
- 后端：`server/tests/applications.test.ts` 覆盖复制提示词审计不保存 secret 明文。
- 前端：`src/pages/Roles/index.test.tsx` 覆盖授权 Drawer 权限组选择与摘要。
- 前端：`src/pages/Applications/Detail.test.tsx` 覆盖复制应用提示词按钮与内容脱敏。
- 前端：`src/pages/Login/index.test.tsx` 覆盖 `loginRequired` 和视觉/恢复文案。
- 收口：运行 server tests、frontend tests、build 和必要 Playwright/browser 验证。

## 风险

| 风险 | 控制 |
|---|---|
| Prompt 泄露 secret | 默认只包含环境变量名；复制审计不保存 prompt 全文；测试断言不包含 secret preview。 |
| 权限组绑定导致跨应用越权 | 后端按角色所属应用解析 permission group code；应用管理员继续走 application scope。 |
| 组绑定与点绑定重复 | 权限查询 `distinct` 去重；列表计数按 distinct group/point 统计。 |
| 登录页过度营销化 | 只使用 Admin Console 入口布局、状态卡和部署元信息，不使用落地页式 hero。 |
