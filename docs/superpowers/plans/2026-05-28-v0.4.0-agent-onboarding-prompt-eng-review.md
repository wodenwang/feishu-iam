# feishu-iam v0.4.0 Agent 接入初始化提示词工程评审

日期：2026-05-28

## 结论

通过，按窄切片执行。`v0.4.0` 只增强第三方应用接入初始化提示词和一次性 secret 复制体验，不改 OAuth、Application API、secret hash 存储或权限模型。

## 架构决策

- D1：prompt 在前端构建。前端已经持有应用详情、redirect URI 和一次性 secret 结果，后端不需要新增 prompt API。
- D2：不新增可解密 secret 存储。创建和轮换接口继续只返回一次性明文，数据库仍只保存 hash。
- D3：复制行为继续走 `POST /api/applications/:id/secret-copy-events`，但扩展 kind：
  - `runtime_env`
  - `agent_prompt_placeholder`
  - `agent_prompt_onetime_plaintext`
- D4：审计 metadata 只保存 `appKey`、`kind` 和 `containsPlaintextSecret: boolean`，不保存 prompt 正文或 secret。
- D5：`buildApplicationPrompt()` 支持可选 secret 输入；没有输入时输出占位符。

## 数据流

```text
Application detail + active redirect URIs
  -> buildApplicationPrompt({ mode: placeholder })
  -> copy prompt
  -> secret.copy audit kind=agent_prompt_placeholder

Create application result / Rotate secret result
  -> one-time plaintext secret exists only in frontend state
  -> buildApplicationPrompt({ oneTimeSecrets })
  -> copy prompt
  -> secret.copy audit kind=agent_prompt_onetime_plaintext
```

## 风险与处理

| 风险 | 处理 |
|---|---|
| 一次性 prompt 泄露到审计或日志 | 审计接口不接收 prompt 正文，只接收 kind；测试扫描 secret 不在 audit metadata。 |
| 用户误以为普通详情页能找回 secret | 普通 prompt 使用占位符，UI 文案说明只有创建/轮换当次可复制明文。 |
| 轮换单个 secret 后 prompt 错填另一个 secret | prompt builder 接收局部 secret，未知值保持占位符。 |
| prompt 与真实接口漂移 | prompt 中的 endpoint 和 HMAC canonical string 与当前代码路径保持一致，并由测试断言关键字符串。 |
| 创建应用 Modal 过载 | 不改变主布局，只增加复制完整 prompt 的次级按钮和 warning。 |

## 测试策略

- 前端单测：
  - 普通 prompt 包含 `AGENTS.md`、`CLAUDE.md`、接口 SOP 和 secret 占位符。
  - 普通 prompt 不包含 secret preview。
  - `appSecret` one-time prompt 只填 app secret 明文。
  - `apiSecret` one-time prompt 只填 api secret 明文。
  - 创建应用结果 prompt 同时填入两个 secret。
- 后端单测：
  - `secret-copy-events` 接受新增 kind。
  - audit metadata 不包含测试 secret 明文。
- 文档检查：
  - README 和 integration doc 出现 v0.4.0 Agent 初始化 SOP。
