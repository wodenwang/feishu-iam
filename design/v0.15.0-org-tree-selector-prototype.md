## 设计边界

- `v0.15.0` 覆盖 GitLab issue `#19/#20/#24`。
- GitLab issue `#21` 明确排除到后续小版本，不在本原型中加入最终权限点查看器或权限点解释面板。
- 飞书同步页只浏览本地镜像组织和用户，不引入飞书通讯录实时树。
- 飞书同步页顶部默认操作区不放全量同步；全量同步只能作为高级 / 危险操作进入强确认流程，并保留审计线索。
- 组织浏览和组织用户绑定采用项目内封装的 `OrgBrowser` / `OrgUserSelector`，不把第三方树控件视觉和状态模型直接暴露给页面。
- 组织浏览器使用下钻式信息架构：搜索、面包屑、混合部门/用户列表、部门详情和选择摘要分区清晰。
- 错误状态采用区域隔离：当前部门或当前查询失败时，只影响左侧浏览区域，不清空右侧已选草稿和保存摘要。
- 角色绑定中，选择部门表示绑定组织主体，不等于自动绑定全部用户；用户可单独选择，重复覆盖、父子半选、禁用和移除摘要需要在组件内明确呈现。

## 实现提示

- 桌面角色绑定态建议保持双栏高度稳定，左侧待选区承载搜索、面包屑、下钻列表和局部失败；右侧承载已选对象、变更摘要和保存错误。
- 390px 窄屏不要压缩成左右双栏，应切换为 `待选 / 已选 / 摘要` 分步面板；已选页负责组织 / 用户分组和移除，摘要页负责新增 / 移除差异、保存中和保存失败，保存区固定在当前流程底部。
- `OrgUserSelector` 保存语义应按主体类型提交：`org_subjects` 与 `user_subjects` 分开；部门主体不在前端展开为用户列表，覆盖去重只用于摘要提示和后端校验前的用户理解。
- `#24` 属于表格操作列稳定性修复，应通过固定操作列宽、按钮最小宽度和不换行文案解决，不扩大到 `#21` 的权限可解释性主题。
- `停用` 必须使用危险确认弹窗，按钮保留稳定 tooltip 和 `aria-label=停用角色`；确认文案说明授权立即失效、审计保留。
