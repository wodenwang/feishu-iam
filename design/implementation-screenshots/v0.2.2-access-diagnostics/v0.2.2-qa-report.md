## Runtime 验收覆盖

- 健康检查通过。
- mock 飞书登录、平台管理员绑定和目录用户投影通过公开 API 完成。
- 新建临时应用后，诊断返回 warning，并包含 `NO_PERMISSION_REGISTRATIONS` 与 `NO_ROLE_BINDINGS`。
- 注册权限组/权限点、创建角色并绑定飞书用户后，诊断返回 healthy 且无 findings。
- 停用 active redirect URI 后，诊断返回 failed，并包含 `NO_ACTIVE_REDIRECT_URI`。
- `POST /api/applications/:id/diagnostics/copy` 写入 `application.diagnostics.copy` 审计，脚本不上传诊断包正文。
- 诊断 API 输出检查不包含一次性 `appSecret` / `apiSecret` 原文。

## 剩余风险

- 本版本只聚合现有表和审计事件；真实生产日志、trace、飞书开放平台配置错误仍需要外部系统或人工排查。
- 真实飞书验收仍依赖飞书开放平台手动配置 redirect URI、通讯录权限和环境白名单。
