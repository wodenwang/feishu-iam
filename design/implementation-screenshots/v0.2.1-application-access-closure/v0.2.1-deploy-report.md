## 部署动作

1. 使用 `git archive --format=tar HEAD` 将仓库受控文件同步到远端部署目录。
2. 保留远端 `.env` 与 `data/postgres`，并创建 `.env.backup-v0.2.1-20260528012318`。
3. 将远端 `.env` 中的 `FEISHU_IAM_IMAGE_TAG` 更新为 `v0.2.1`。
4. 在远端执行 `docker compose up -d --build` 完成镜像构建、容器重建和启动。

## 边界确认

- 本版本交付角色绑定权限组、应用接入提示词复制和未登录飞书重登提示。
- 系统仍保持飞书唯一身份源，未新增 username/password、本地 root 或独立账号体系。
- 应用提示词只输出接入端点、redirect URI、`appKey`、HMAC 规则和 secret 环境变量名，不输出真实 secret 明文。
- 远端 `.env`、飞书应用凭证、tokens、同步快照和 PostgreSQL 数据目录未纳入仓库同步。
