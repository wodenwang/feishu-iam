## 部署动作

1. 使用 `git archive --format=tar HEAD` 将仓库受控文件同步到远端部署目录。
2. 保留远端 `.env` 与 `data/postgres`，并创建 `.env.backup-v0.3.0-*`。
3. 将远端 `.env` 中的 `FEISHU_IAM_IMAGE_TAG` 更新为 `v0.3.0`。
4. 在远端执行 `docker compose up -d --build --remove-orphans` 完成镜像构建、容器重建和启动。

## 边界确认

- 本版本只交付飞书事件接收、去重、状态记录和重试入口。
- 系统仍保持飞书唯一身份源，未新增 username/password、本地 root 或独立账号体系。
- 事件记录不保存 secret、token、signature、cookie 或完整原始事件体。
- 远端 `.env`、飞书应用凭证、tokens、同步快照和 PostgreSQL 数据目录未纳入仓库同步。
