# Feishu IAM v0.7.0 管理后台信息架构与关键闭环设计

日期：2026-05-20
状态：已确认设计方向，待用户审阅书面规格

## 1. 版本目标

`v0.7.0` 的目标是重新梳理 Feishu IAM 后台管理界面，把当前长页面式控制台重构为真正的模块化管理后台，并补齐关键管理闭环。

一句话边界：

`v0.7.0 = 后台信息架构重构 + 应用管理、权限管理、管理员授权、记录查询、系统设置的独立模块闭环`。

版本完成后，后台应满足：

1. 左侧菜单每个一级模块都是独立页面，不再通过 `#锚点` 或 scroll-to 定位到同一长页面。
2. 每个模块进入首页后展示当前信息列表或概览，并提供对应管理操作闭环。
3. 复杂交互先形成可确认原型，再进入实现。
4. 尚未实现的子能力保留空白页，并在显眼位置显示“建设中”。
5. 本版本不处理 GitLab issue 修复；issue 后续单独开版本或补丁版本 triage。

## 2. 版本承接关系

`v0.7.0` 承接 `v0.6.0` 已完成的生产化 Docker Compose 部署、管理员登录和生产入口边界，也承接 `v0.6.1` 规划中的管理后台手工配置能力。

如果进入实施时 `v0.6.1` 仍未完全落地，`v0.7.0` 实施计划需要先复核并补齐必要的后台 API 能力，不能只做前端路由重排。

## 3. 不纳入范围

`v0.7.0` 不做以下能力：

- GitLab issue 修复或 issue triage。
- HTTPS、反向代理、高可用或多实例部署。
- 完整 OIDC Discovery、JWKS、ID Token、refresh token 或 SAML。
- 资源级权限、ABAC、deny 规则或数据范围权限。
- 飞书用户组同步或飞书角色同步。
- 权限点和权限组的 UI 创建、编辑、删除。
- 统一记录时间线聚合后端。
- 备份升级、任务调度或复杂系统运维后台。
- 应用硬删除、管理员硬删除或 client 硬删除。

## 4. 一级菜单与路由

后台一级菜单固定为：

- `工作台`
- `应用管理`
- `权限管理`
- `管理员授权`
- `记录查询`
- `系统设置`

每个一级菜单必须对应独立路由。现有 `AdminShell` 中的 `href="#模块名"` 锚点导航需要替换为真实路由或等价的模块状态路由。模块切换不能依赖滚动定位。

每个模块都必须具备：

- 独立首页。
- 加载态。
- 空态。
- 错误态。
- 权限不足态。
- 未实现子能力的“建设中”状态。

可见按钮必须满足以下任一条件：

1. 有真实动作。
2. 有明确加载、成功、失败反馈。
3. 因权限或前置条件不足被禁用，并展示原因。
4. 未实现时不显示为可点击按钮，改用“建设中”占位。

`SSO 接入` 不再作为独立一级菜单或独立模块名。所有应用级 SSO 接入配置都收拢到 `应用管理` 的应用详情中。

## 5. 工作台

`工作台` 只做轻量概览和快捷入口，不承载具体管理表单。

工作台展示：

- 系统运行状态。
- 应用数量和状态摘要。
- 飞书同步摘要。
- 近期风险入口。
- 跳转到应用管理、权限管理、记录查询和系统设置的快捷入口。

工作台不提供创建应用、权限授权、管理员授权、触发同步等主操作，避免重新退化成长页面。

## 6. 应用管理

### 6.1 模块定位

`应用管理` 是“应用概念”的总入口。凡是围绕应用本身的配置，都在这里管理。

包括：

- 应用基础信息。
- 应用启停。
- dev、test、prod 环境。
- redirect URI。
- client。
- client secret 查看与轮换。

### 6.2 首页

模块首页默认展示应用清单。

应用清单支持：

- 按关键词搜索。
- 按状态筛选。
- 创建应用。
- 进入应用详情。
- 显示应用状态、名称、`app_key`、owner、更新时间等关键字段。

平台管理员可查看和管理全部应用。应用管理员只能看到自己被授权的应用。

### 6.3 应用详情结构

应用详情采用主从式结构：保留应用列表或返回清单能力，详情区使用二级导航组织应用配置。

详情二级导航：

- `基础信息`
- `环境与回调`
- `Client`
- `操作记录`

`基础信息` 支持查看和编辑应用名称、描述、owner、状态。应用不做硬删除，只支持停用和启用。

`环境与回调` 支持管理应用环境和每个环境下的 redirect URI。回调地址仍遵守既有环境安全规则：`dev` 可允许 localhost 和 RFC1918 私有 IPv4 HTTP 回调，`test` 和 `prod` 仍要求 HTTPS。

`Client` 支持：

- client 列表。
- 创建 client。
- 查看 client secret。
- 轮换 client secret。
- 停用和启用 client。

`操作记录` 用于从应用上下文进入记录查询。如果后端暂不支持应用过滤，必须显示“建设中”或“当前筛选暂不可用”，不能假装过滤成功。

### 6.4 应用停用

应用停用不是删除。

停用后必须保留：

- 应用基础信息。
- 环境。
- redirect URI。
- client。
- 权限、角色和成员绑定。
- 审计日志和安全事件。

停用后后端应拒绝或使失效：

- 该应用发起的登录。
- 该应用 client 换取 token。
- 该应用权限查询。

### 6.5 Client Secret 查看与轮换

`v0.7.0` 在安全和效率之间做平衡：平台管理员和对应应用管理员都可以查看和轮换本应用的 `client_secret`。

安全规则：

- 新增和轮换后的 secret 必须服务端加密存储，不能明文落库。
- 仍保留用于 token 校验的 hash 或等价不可逆校验材料。
- 查看 secret 前需要二次确认。
- 每次查看和轮换都写审计日志。
- 每次查看和轮换都写安全事件。
- 前端只短时展示 secret。
- secret 不得写入日志、文档、会话归档、测试快照或浏览器持久化存储。
- 非授权应用管理员不能查看其他应用 secret。

历史 client 如果当前只保存了不可逆 `client_secret_hash`，不能伪造可查看能力。此类 client 在 UI 中应显示“历史 secret 不可查看，请轮换后查看新 secret”。轮换后才进入可查看的加密存储路径。

建议安全事件类型：

- `secret_viewed`
- `secret_rotated`

## 7. 权限管理

### 7.1 模块定位

`权限管理` 管业务应用权限分配，不管理后台管理员身份。后台管理员身份归 `管理员授权`。

权限分配不按人直接分配，而按 IAM 角色分配。

主路径：

1. 选择应用。
2. 创建、编辑、禁用 IAM 角色。
3. 给 IAM 角色绑定已有权限组。
4. 给 IAM 角色绑定飞书用户或飞书部门。
5. 用户权限由角色推导出权限组和权限点。

### 7.2 入口关系

权限配置更多从应用往下穿透。

`应用管理` 的应用详情中提供“管理权限”入口，跳转到 `权限管理` 并带上应用上下文。

`权限管理` 也保留独立一级入口，用于：

- 全局检索。
- 切换应用。
- 批量查看角色授权状态。

平台管理员可管理全部应用。应用管理员只能管理自己被授权的应用。

### 7.3 IAM 角色

IAM 角色必须在管理 UI 中维护。

平台管理员和对应应用管理员可以在授权应用范围内：

- 创建 IAM 角色。
- 编辑 IAM 角色名称和描述。
- 禁用 IAM 角色。
- 启用 IAM 角色。
- 给 IAM 角色绑定已有权限组。
- 给 IAM 角色绑定飞书用户或飞书部门成员。

应用管理员在自己被授权应用下拥有完整权限授权闭环。平台管理员拥有同等能力且跨全部应用。

### 7.4 权限组和权限点边界

权限组和权限点暂不在 `v0.7.0` UI 中创建、编辑或删除。

权限点和权限组由第三方应用开发者通过 API 管理。后台 UI 只消费已有权限组，用于角色授权。

后台不常驻展示“权限目录”。“已绑定权限组”只显示当前 IAM 角色已经绑定的权限组结果。

点击 `绑定权限组` 后，在弹窗中：

- 支持按 key 或名称搜索已有权限组。
- 支持勾选多个权限组。
- 保存时替换或更新当前角色的权限组绑定。
- 不提供权限组创建、编辑、删除入口。

角色直接绑定单个权限点不作为本版本主流程；如已有底层能力，可先不在 UI 暴露。

### 7.5 成员绑定

IAM 角色成员绑定支持：

- 飞书用户。
- 飞书部门。

用户和部门都只从本地飞书镜像选择，不直连飞书实时搜索。

部门绑定采用动态语义：某部门绑定到角色后，该部门当前有效成员获得角色权限；飞书同步更新部门成员后，权限计算自然变化，不保存一次性用户快照。

不纳入本版本：

- 飞书用户组成员绑定。
- 飞书角色成员绑定。
- 组织范围限制授权。

## 8. 管理员授权

### 8.1 模块定位

`管理员授权` 只管理 Feishu IAM 后台管理员，不管理第三方应用里的业务用户权限。业务权限分配归 `权限管理`。

### 8.2 管理员类型

`v0.7.0` 后台 UI 只暴露两类管理员：

- `平台管理员`
- `应用管理员`

平台管理员拥有 IAM 平台所有操作能力，包括创建和停用应用、应用接入配置、IAM 角色维护、角色权限组绑定、角色成员绑定、分配应用管理员、查看记录查询和系统设置。

应用管理员绑定到一个或多个特定应用，只能在授权应用范围内维护该应用权限授权闭环。

历史角色如 `audit_viewer`、`sync_admin` 如果数据库已有，可以在管理员列表中只读展示，但新增和编辑授权主流程不主动露出。

### 8.3 首页与操作

模块首页默认展示管理员列表。

列表字段：

- 管理员姓名。
- 飞书 `user_id`。
- 管理员类型。
- 应用范围。
- 状态。
- 最近登录时间。
- 创建时间。

支持操作：

- 新增管理员。
- 编辑应用管理员的应用范围。
- 禁用管理员。
- 启用管理员。

新增管理员流程：

1. 搜索本地飞书用户。
2. 选择平台管理员或应用管理员。
3. 如果选择应用管理员，必须选择一个或多个应用。
4. 保存后刷新管理员列表。

不支持硬删除管理员。

禁用管理员后，其已有管理 session 应失效或在下一次请求被拒绝。

非平台管理员进入该模块时显示权限不足说明，不能展示会失败的可点击按钮。

所有新增、编辑、启停操作都必须写审计日志。

## 9. 记录查询

### 9.1 模块定位

`记录查询` 收拢所有查询类能力。首页使用页签或卡片入口，不做统一聚合时间线。

子能力：

- `审计日志`
- `安全事件`
- `同步记录`
- `登录与 Token 记录`

### 9.2 审计日志

审计日志接入现有审计查询能力。

需要支持从应用管理和权限管理带应用上下文进入。如果接口暂不支持应用过滤，界面要明确显示“当前筛选暂不可用”或“建设中”。

### 9.3 安全事件

安全事件接入现有安全事件查询能力。

新增 secret 高风险事件筛选：

- `secret_viewed`
- `secret_rotated`

查看和轮换 `client_secret` 必须能在安全事件中被检索到。

### 9.4 同步记录

同步记录展示飞书同步历史。如果当前接口不足，先显示“建设中”。

### 9.5 登录与 Token 记录

登录与 Token 记录展示 SSO 登录、token 异常、revoke 等记录。如果当前接口不足，先显示“建设中”。

## 10. 系统设置

`系统设置` 只承接系统层能力，不混入应用级配置。

本版本系统设置包含：

- `飞书同步`
- `系统运行`
- `版本信息`

`飞书同步` 展示同步状态、字段诊断、手动触发同步和同步历史入口。

`系统运行` 展示 API 健康、数据库 ready、运行状态。

`版本信息` 展示当前版本、部署入口说明和升级提示。

备份升级、任务调度、SSO 全局设置等不在 `v0.7.0` 承诺范围；如需要占位，必须显示“建设中”。

## 11. 后端能力影响

实施前需要复核现有后端能力，并补齐缺口。

可能需要新增或调整的能力：

- 应用编辑、停用、启用接口。
- client 启用接口。
- client secret 加密存储字段、加密密钥配置和查看接口。
- secret 查看和轮换的审计日志与安全事件。
- IAM 角色创建、编辑、启停接口。
- IAM 角色绑定权限组接口。
- IAM 角色绑定飞书用户和飞书部门接口。
- 权限计算服务支持部门动态成员语义。
- 管理员编辑应用范围、启停管理员接口。
- 管理员禁用后 session 失效或请求拒绝。
- 记录查询按应用上下文过滤，若本版本不补齐则前端必须显示不可用状态。

数据库层面可能需要：

- `application_clients` 增加加密 secret 字段和加密元数据字段。
- 迁移历史 client 为“不可查看，需要轮换”状态。
- 保持 `client_secret_hash` 或等价不可逆校验材料用于 OAuth token 校验。
- 如现有 `iam_role_subjects.subject_type` 未覆盖部门语义，需要明确 `feishu_user`、`feishu_department` 等枚举值。

## 12. 前端结构建议

前端应避免继续扩大 `App.tsx`。建议按模块拆分：

- `routes/WorkspacePage`
- `routes/ApplicationManagementPage`
- `routes/PermissionManagementPage`
- `routes/AdminAuthorizationPage`
- `routes/RecordQueryPage`
- `routes/SystemSettingsPage`

共享组件：

- 模块页头。
- 列表筛选栏。
- 空态、错误态、权限不足态、建设中态。
- 确认弹窗。
- Secret 查看弹窗。
- 应用选择器。
- 飞书用户搜索选择器。
- 飞书部门搜索选择器。

应用详情和权限管理都需要可从 URL 表达当前应用上下文，便于刷新、复制链接和从其他模块跳转。

## 13. 权限与审计

后台权限边界必须由后端校验，前端隐藏按钮只是体验优化。

平台管理员：

- 可操作全部应用和系统设置。
- 可管理所有管理员授权。
- 可查看全部记录查询能力。

应用管理员：

- 只能查看和管理自己被授权的应用。
- 可在授权应用下维护 IAM 角色、角色权限组绑定、用户/部门成员绑定。
- 可查看和轮换授权应用下的 client secret。
- 不能创建应用。
- 不能管理管理员授权。
- 不能管理系统级设置。

所有写操作必须写入审计日志。高风险操作必须写入安全事件，至少包括：

- 查看 client secret。
- 轮换 client secret。
- 停用应用。
- 停用 client。
- 禁用管理员。
- 修改管理员应用范围。

## 14. 测试与验收

### 14.1 自动化验证

实施完成后至少运行：

- `pnpm check`
- 后端相关单元测试和 e2e 测试。
- 前端测试。
- 构建验证。

如果某项无法运行，必须说明原因和风险。

### 14.2 浏览器验证

复杂 UI 需要用浏览器做真实交互验证，并覆盖：

- 一级菜单切换不触发 scroll-to 长页面行为。
- 每个模块独立加载。
- 应用管理主从详情。
- client secret 查看、轮换和短时展示。
- 权限管理角色绑定权限组弹窗。
- 权限管理绑定用户和部门。
- 管理员授权新增、编辑范围、启停。
- 记录查询安全事件筛选 `secret_viewed`、`secret_rotated`。
- 系统设置不出现应用级 SSO 配置。

### 14.3 验收标准

本版本验收标准：

1. 左侧菜单不再使用 `#锚点`，一级菜单进入独立路由。
2. 每个一级模块都有独立首页、加载态、空态、错误态和权限不足态。
3. 未实现子能力显示“建设中”。
4. 应用管理能完成应用基础信息、环境、回调、client、secret 查看和轮换闭环。
5. 权限管理能完成 IAM 角色、绑定权限组、绑定用户和部门成员闭环。
6. 权限组和权限点不在 UI 创建、编辑、删除。
7. 管理员授权能完成新增、编辑应用范围、禁用、启用闭环。
8. 管理员授权 UI 只暴露平台管理员和应用管理员两层。
9. 记录查询能查审计和安全事件，并能筛选 secret 查看和轮换事件。
10. 系统设置承接飞书同步、系统运行、版本信息，不混入应用级配置。
11. 相关测试、类型检查、构建通过。

## 15. 后续版本建议

后续版本可以单独规划：

- GitLab issue triage 和修复版本。
- 权限点和权限组开发者 API 文档增强。
- 批量成员授权和导入。
- 更细的应用管理员子权限。
- 组织范围限制授权。
- 统一记录时间线。
- 备份升级和任务调度后台。
