# v0.13.1 左侧导航层级修复 Pencil 原型说明

日期：2026-05-28

## 1. 原型目标

本原型用于验证 GitLab issue `#12` 的交互方向：后台左侧导航必须清楚表达一级目录和二级目录，尤其是 `系统管理` 与 `飞书同步 / 管理员授权 / 操作审计 / 系统信息` 的父子关系。

本原型不改变业务页面内容，只验证导航壳层。

## 2. 原型文件

- Pencil 源文件：`design/admin-console-v0.13.1.pen`
- 导出目录：`design/exports/v0.13.1-sidebar-nav/`

导出截图：

- `fAmZJ.png`：桌面展开态，当前位于 `系统管理 / 操作审计`。
- `ePJlH.png`：桌面展开态，`系统管理` 折叠。
- `k3FzN.png`：桌面收缩态，当前仍位于系统管理二级页。
- `p4oOj.png`：390px 移动端导航 Sheet 展开态。

## 3. 设计决策

- 桌面展开态中，`系统管理` 是带 chevron 的一级分组，不再只是普通链接。
- 二级菜单使用左侧层级线、缩进和较弱文字色表达从属关系。
- 当前二级项使用浅青绿色背景和左侧强调线高亮。
- 当前二级页下，`系统管理` 父级也保持 active 背景，提供上下文。
- 桌面收缩态下，`系统管理` 图标保持 active，并通过 tooltip 展示其二级入口摘要。
- 移动端 Sheet 展示完整层级，不退化成只有一级图标。

## 4. 工程映射

- `AppShell` 应拆出或重构项目级 `SidebarNav` / `SidebarNavItem`。
- 父级分组应支持 `aria-expanded`。
- 二级页面应使用 `aria-current="page"`。
- 当前二级页所在父级必须自动展开。
- 收缩态的 tooltip 只作为视觉辅助，不能替代 accessible name。

## 5. 自检结果

- `snapshot_layout`：未发现布局问题。
- 已导出 4 张关键状态截图。
- 移动端初版曾出现遮罩后页面文字逐字换行，已修正为纯背景，避免干扰 Sheet 层级判断。
