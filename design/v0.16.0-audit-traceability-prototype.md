## 设计边界

- `#27` 的第一验收对象仍是未登录提示页；统一问题提示页是复用同一信息架构的扩展，不改变管理员 session、飞书登录 OAuth 后端流程或破窗登录边界。
- 问题提示页只展示稳定、可复制、可排障的信息；复制内容不得包含 access token、client secret、developer api token、cookie、authorization header、堆栈或原始敏感 payload。
- 追踪视角保留在 `系统管理 / 操作审计` 内，不替代原有 `审计日志 / 安全事件 / 同步记录 / 登录与 Token` Tab。
- 时间线用于排障叙事，不做复杂 BI 图表或导出中心。
- 应用详情跳转只携带应用、client、时间窗口和返回路径上下文，不改变应用详情现有 Tab 模型。

## 实现提示

- 统一问题提示页建议抽成可复用组件，输入场景、标题、说明、主操作、错误码、`request id`、路径、应用和 client 上下文。
- `request id` 必须有独立复制按钮；“一键复制问题信息”使用稳定多行文本模板，空字段不展示占位符。
- 390px 下主操作独占一行，复制操作下沉为次级整行按钮；排障信息超过四行时允许折叠，但 `request id`、错误码和发生时间必须默认可见。
- 追踪 Tab 初始态应优先引导粘贴终端用户复制的问题信息；完整命中态先给诊断摘要，再给时间线和原始事件入口。
- 部分命中、无结果和权限不足必须是明确状态，不应只显示空表格。
- 长 `request id`、长回调 URL 和脱敏字段在列表中截断或换行，复制内容始终使用完整值。
