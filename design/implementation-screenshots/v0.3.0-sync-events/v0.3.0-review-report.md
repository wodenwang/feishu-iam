# v0.3.0 飞书事件同步 Pre-landing Review

## 结论

通过，未发现阻塞发布的问题。

## Diff 风险检查

- 身份边界：未新增 username/password、本地 root 或独立账号体系。
- 飞书边界：事件回调使用飞书 Verification Token；配置 Encrypt Key 时校验 Lark 签名并解密事件体。
- 敏感信息：`sync_events` 只保存事件 id、类型、资源标识、状态和摘要，不保存 secret、token、signature、cookie 或完整原始事件体。
- 权限边界：事件回调不依赖 IAM session；事件状态、列表和重试 API 均限制平台管理员访问。
- 同步边界：事件重试使用 system actor 触发同步 run，不伪造飞书用户。
- 版本边界：未实现目录编辑、告警平台、完整 incremental worker、OIDC/JWKS/PKCE/refresh token、SDK/CLI/Helm/Terraform。

## 测试覆盖

- 后端：URL verification、事件接收、去重、事件状态、事件列表、重试、非平台管理员禁止访问。
- 前端：同步页事件区渲染、事件表格和既有同步任务交互回归。
- Runtime：`scripts/verify-v0.3-sync-events.sh` 覆盖 challenge、pending、duplicate、list、retry 和敏感信息扫描。
