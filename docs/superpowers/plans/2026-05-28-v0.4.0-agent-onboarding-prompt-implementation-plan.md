# v0.4.0 Agent Onboarding Prompt Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `v0.4.0` Agent-first third-party onboarding prompts with placeholder and one-time plaintext secret modes, updated docs, tests, and release metadata.

**Architecture:** Keep prompt generation in the frontend because one-time plaintext secrets already exist only in create/rotate result state. Extend copy-audit kinds so runtime can record placeholder versus one-time plaintext prompt copies without receiving prompt text or secret values. Reuse existing Application Detail and create-result modal UI patterns.

**Tech Stack:** React, TypeScript, Ant Design, TanStack Query, Fastify, PostgreSQL, Vitest, Playwright.

---
