# v1.0.7 权限管理 PRD 定稿会话

## 会话目标

规划 Feishu IAM 下个版本，继续优化权限管理模块，并结合线上审计和 GitHub issues 定稿 PRD。

## 用户关键要求摘要

- 使用 Product Design 插件调查线上版本 `https://feishu-iam.riversoft.com.cn/admin/permissions`。
- 结合用户自行登记在 issues 中的改进项。
- 梳理版本改动范围。
- 通过互动沟通最终定稿 PRD。

## 使用或形成的关键约束

- 遵守 `AGENTS.md`：中文文档、先读治理、非平凡变更要有验证说明、权限管理不回退既有能力。
- 当前任务是规划和 PRD 定稿，不进入实现。
- 不处理 secret、token、cookie 或其他敏感凭证。
- 线上审计只做只读浏览器观察，不执行保存、停用、删除、发布或部署动作。

## 重要过程

1. 读取项目治理、`my-harness-next-action` 和 Product Design `audit` 规则。
2. 查询 GitHub open issues，确认目标仓库为 `wodenwang/feishu-iam`，当前相关 issue 为 `#6/#7/#8/#9`。
3. 首次打开线上权限管理入口时，内置浏览器没有管理员会话，记录登录门禁截图。
4. 用户完成管理员登录后，继续审计权限管理角色列表、角色工作台总览、组织与用户、应用权限、基础信息和变更记录。
5. 与用户交互确认 PRD 决策：`D1/B D2/A D3/A D4/A`。

## 重要设计决策

- `v1.0.7` 同时覆盖 `#6-#9`，拆成两个 vertical slice。
- 用户确认两个 slice 同属一个版本一次性完成，不拆分成 `v1.0.8`。
- S1：角色工作台减负，移除冗余总览区域、基础信息 tab、变更记录 tab，重做应用选择器，移除角色工作台中的权限点对比。
- S2：新增权限矩阵 MVP，用于按用户或组织查询最终权限。
- 组织权限矩阵首版只统计直接绑定该组织的角色，不展开组织下所有用户。
- 移除角色关联应用采用软解除或停用绑定，并写操作审计，不做硬删除。
- 平台管理员登录下 `创建角色` disabled 纳入本版本调查和修复。
