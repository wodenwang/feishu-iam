## 必须进入工程计划的设计要求

1. `/roles` 在 HTTP mode 不得 fallback 到 mock role 数据。
2. 角色列表加载失败时必须显示可恢复错误块，并展示 requestId。
3. 新建 / 编辑 / 停用 / 授权保存失败时必须给出 message 反馈，包含 requestId。
4. 授权 Drawer 可继续读取 v0.1.4 的部门和用户只读接口，但不得新增 `/directory` 能力。
5. 权限树只展示当前应用已注册的 permission groups / permission points，不做权限点编辑。
6. HTTP mode 下批量停用可以通过逐个 `PATCH /api/roles/:id` 实现，不新增批量后端 API。
7. 409 duplicate role code 需要保留明确错误信息。
8. 768 视口下操作列、授权 Drawer、确认 Modal 不得遮挡核心按钮。
