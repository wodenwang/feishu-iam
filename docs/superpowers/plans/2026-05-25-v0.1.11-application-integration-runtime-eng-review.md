# v0.1.11 Application Integration Runtime 工程评审

## 结论

通过。`v0.1.11` 应实现应用接入配置 runtime 闭环，而不是扩大到第三方 OAuth server 或 Sync runtime。

## 范围锁定

In scope:

- 应用详情 runtime endpoint。
- 权限注册结果 runtime endpoint。
- secret copy audit endpoint。
- audit list target filters。
- 前端 HTTP service 替换 unsupported 方法。
- 应用详情和接入向导使用 runtime 数据。
- README 公共入口文档重构。

Out of scope:

- 第三方 OAuth server、token endpoint、consent。
- OIDC discovery。
- Sync runtime。
- `/directory` 能力扩展。
- 真实 secret 明文再展示。

## 架构决策

1. 不新增应用配置 migration。第一版接入配置使用 runtime 事实和占位符模板，避免把 callback/origin 配置模型挤进本切片。
2. 应用详情读取先保持 platform-admin only。application-admin 权限边界需要独立数据模型支持，不能在本切片用硬编码补。
3. `secret.copy` 只记录行为和 kind，不接收、不存储、不回显 secret。
4. 审计过滤复用 `GET /api/audit-logs`，新增 `targetId` / `targetType` 查询条件。
5. README 面向首次浏览者，删除 `bpmt-120` 和私有部署细节。

## 风险与控制

| 风险 | 控制 |
|---|---|
| secret 泄露 | 后端不返回 secret；copy audit metadata 不含 secret；README 不含真实配置。 |
| 权限边界过宽 | v0.1.11 只允许 platform admin 读取详情；应用管理员后续单独做。 |
| 接入检查误报完成 | 接入检查只表达 runtime 可见事实，不宣称 OAuth demo 已完成。 |
| README 继续像运维记录 | 重写成项目价值与快速上手入口，私有部署移出 README。 |

## 测试策略

- 后端测试覆盖 endpoint 权限、返回字段、权限注册结果、secret copy audit 和 audit target filters。
- 前端单测覆盖 HTTP API 方法和页面基本行为。
- Playwright 覆盖 HTTP mode 应用创建、详情、接入向导和复制审计路径。
- 构建与 server build 作为发布门禁。
