## Findings

### REVIEW-001：HTTP applications filters were visible but ignored

等级：High

问题：HTTP mode `/applications` 页面保留了 keyword/status/date 筛选控件，但 `httpApi.listApplications` 只传 `page/pageSize`，后端也只分页，不处理筛选。用户点击筛选会看到静默无效的结果。

修复：

- `src/features/iam/httpApi.ts` 转发 `keyword`、`status`、`createdAtFrom`、`createdAtTo`。
- `server/src/modules/applications/applicationRoutes.ts` 使用参数化 SQL 支持 keyword/status/createdAt filters，并同步 total。
- 新增 `src/features/iam/httpApi.test.ts`。
- 扩展 `server/tests/applications.test.ts`。

验证：

- `npm run test -- src/features/iam/httpApi.test.ts src/features/iam/httpClient.test.ts src/pages/Applications/List.test.tsx`
- `TEST_DATABASE_URL=postgres://feishu_iam:feishu_iam@127.0.0.1:55432/feishu_iam_manual npm run server:test -- server/tests/applications.test.ts`

### REVIEW-002：HTTP mode direct routes could enter out-of-scope pages

等级：High

问题：HTTP mode 菜单只显示 `/applications` 和 `/audit-logs`，但路由仍注册 Dashboard/Roles/Directory/Sync/Application Detail/Onboarding。平台管理员直达这些 URL 会进入后续切片页面，并触发 unsupported HTTP methods；面包屑“首页”也指向 `/dashboard`。

修复：

- `src/app/App.tsx` 在 HTTP mode 只注册 `/applications` 和 `/audit-logs` 可用 route。
- HTTP mode 下后续切片 route redirect 到 `/applications`。
- `src/layouts/AdminLayout.tsx` 的 breadcrumb 首页在 HTTP mode 指向 `/applications`。
- E2E 增加 `/roles` 直达应回到 `/applications` 的断言。

验证：

- `npm run test -- src/router/routes.test.tsx src/pages/Login/index.test.tsx src/pages/Initialize/index.test.tsx`
- `E2E_RESET_DATABASE=true DATABASE_URL=postgres://feishu_iam:feishu_iam@127.0.0.1:55432/feishu_iam_manual VITE_IAM_API_MODE=http npm run e2e -- tests/e2e/v0.1.3-admin-console-http.spec.ts`
