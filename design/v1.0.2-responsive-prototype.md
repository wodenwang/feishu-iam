# Feishu IAM v1.0.2 响应式原型说明

日期：2026-06-13

## 原型定位

本原型说明用于承接 `v1.0.2` 的 Product Design 视觉目标、现有 `v1.0.0` Pencil 原型和线上审计截图，指导后续实现阶段的第一个 UI/UX vertical slice。

本轮不改变后台信息架构、路由、权限模型、SSO 协议、后端 API 或数据库结构。目标是在现有 shadcn/ui + tweakcn + Tailwind 前端基线上修复明显 UI/UX 缺陷，尤其是 390px 移动端可读性、Tab 溢出和公开问题页复制边界。

## Pencil MCP 降级记录

本轮尝试使用 Pencil MCP 编辑 `design/admin-console-v1.0.2.pen`：

1. 调用 `get_editor_state(include_schema: true)`，返回错误：当前没有编辑器打开的 `.pen` 文件。
2. 复制 `design/admin-console-v1.0.0.pen` 为 `design/admin-console-v1.0.2.pen`。
3. 调用 `snapshot_layout(filePath=design/admin-console-v1.0.2.pen)`，仍返回需要在编辑器中打开文件。
4. 使用本机 `open design/admin-console-v1.0.2.pen` 后再次调用 `get_editor_state(include_schema: true)`，仍无法被 MCP 识别为打开状态。

因此，本轮将 Pencil 第 3 步降级为“保留 v1.0.2 `.pen` 基线副本 + Product Design 视觉目标 + 响应式原型说明”。这不是跳过设计门禁，而是记录工具限制并把后续实现所需的设计约束固化到项目文件中。

## 关键画面模型

### 1. 移动端资源列表

适用页面：

- 应用管理
- 权限管理
- 管理员授权
- 操作审计结果列表

390px 下的资源卡片必须展示：

- 资源名称或主体名称
- 关键 ID，例如 `app_key`、角色 key、管理员 user id 或 request id
- 状态
- 最近更新时间或关键时间
- 主操作入口

长字段必须换行、截断并提供复制或详情入口，不允许逐字竖排。

### 2. 移动端 Tabs

适用页面：

- 操作审计
- 应用详情
- 角色详情
- 飞书同步内部分区
- 系统管理二级页

390px 下超过 4 项的 Tabs 不允许静态撑破视口。实现可选择：

- 横向滚动 Tabs，并保留滚动容器和可见焦点。
- Select / overflow menu 收纳低频项。
- 分组折行，但不得让内容区被 Tab 挤压。

### 3. 公开问题页

适用页面：

- `/`
- `/oauth/authorize`
- `/oauth/feishu/callback`
- `/api/auth/feishu/callback`
- 管理后台未登录、session 失效、无权限 HTML 页面

页面首屏必须包含：

- 品牌和状态图标
- 用户可读标题
- 简短恢复建议
- `request id`
- `复制 request id`
- 场景化主操作

不得出现 `复制问题信息`、整段错误复制、token、secret、cookie、authorization、授权码、token hash、state hash 或 raw payload。

## 状态矩阵

| 区域 | 加载 | 空 | 错误 | 成功 | 移动端要求 |
|---|---|---|---|---|---|
| 资源列表 | skeleton 保持卡片高度稳定 | 展示创建或筛选提示 | 展示中文错误和重试 | 卡片字段完整可读 | 首屏至少可读一条资源主信息。 |
| Tabs | 不改变 Tab 宽度 | 当前 Tab 空态在内容区展示 | Tab 不承载错误文案 | 当前 Tab 高亮清楚 | 不出现页面级横向溢出。 |
| 问题页 | 不展示无意义 spinner | 不适用 | 只展示 request id 和恢复建议 | 不适用 | 390px 第一屏能看到 request id 和主操作。 |
| 复杂配置 | 保留步骤/摘要骨架 | 已选为空时解释下一步 | 字段错误贴近字段 | 保存后反馈清楚 | 待选 / 已选 / 摘要分步，不遮挡保存入口。 |
