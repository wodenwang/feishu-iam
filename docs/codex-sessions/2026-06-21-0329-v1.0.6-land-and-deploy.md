# v1.0.6 land-and-deploy

## 会话目标

完成 Feishu IAM `v1.0.6` 权限管理 UI/UX 小版本的 Step 15 `gstack /land-and-deploy`：基于 GitHub Release 和 release commit 构建 `linux/amd64` 镜像、上传生产机、执行停机升级，并读回健康和版本证据。

## 用户关键要求摘要

- 基于 GitHub Release `https://github.com/wodenwang/feishu-iam/releases/tag/v1.0.6` 和 release commit `31db6aa96b5dedac89d8943150b599561843cff7` 执行镜像构建、上传和生产部署。
- 执行前复核 git status、tag、release、README/CHANGELOG 口径、部署目标、镜像 tag、远端环境和敏感信息边界。
- 执行后输出镜像信息、远端部署目录、`/ready`、`/version`、smoke/canary 证据和回滚信息。
- 更新 `.my-harness/` 快速索引；第三方技能生成文档继续保留在其规范目录中。

## 使用或形成的关键约束

- 不输出 secret、token、cookie、authorization、数据库密码或飞书密钥。
- 构建镜像必须基于 `v1.0.6` release tag，而不是 main 上的 release 后 handoff 文档提交。
- 生产部署继续使用单机 Docker Compose 停机升级，不改变部署拓扑。
- 远端 `.env` 不全量展示；仅用命令行覆盖版本和镜像相关变量。

## 重要决策

- 使用 detached worktree `/tmp/feishu-iam-v1.0.6-build` checkout `v1.0.6` 进行镜像构建，确保镜像源码为 release commit `31db6aa`。
- 使用离线 tar 上传到 `bpmt@120.24.236.92:/home/bpmt/feishu-iam`，生产机使用 `docker load` 导入镜像。
- 执行升级时显式传入 `FEISHU_IAM_GIT_SYNC=false FEISHU_IAM_PULL_POLICY=never FEISHU_IAM_IMAGE_TAG=v1.0.6 APP_VERSION=1.0.6 GIT_COMMIT=31db6aa ./upgrade.sh`，避免远端 `.env` 旧版本默认值影响升级。


结果：本地 main 干净；`v1.0.6^{}` 指向 `31db6aa96b5dedac89d8943150b599561843cff7`；GitHub Release 已存在、非 draft、非 prerelease。

```bash
docker buildx build --platform linux/amd64 -f deploy/api.Dockerfile -t feishu-iam:v1.0.6 --load .
```

结果：镜像构建成功；本地镜像 ID `sha256:1bf04b6ec9c7c04c331dce7ff3e3bc09ef370d989099d1e2ee9616aad65624b5`；平台 `linux/amd64`；离线包 SHA-256 `0e79ad15c748318769d372c6d98131340410dc589c7df98c2a86da2a48709e08`。

