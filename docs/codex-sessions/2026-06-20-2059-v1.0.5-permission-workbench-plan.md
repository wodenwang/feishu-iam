## 用户关键要求摘要

## 重要设计和工程决策

- `v1.0.5` 是本轮唯一版本号。
- 实施计划保留现有 `/api/v1/admin/applications/:appKey/iam-roles` 路径，但改变语义为“应用上下文下的全局角色绑定”。
- 权限管理首屏无应用筛选时，计划先由前端读取可见应用并 fan-out 拉取角色，再按 `role.id` 去重；当前接入规模小，接受这个低成本方案。
- 实施计划要求从数据库和 API vertical slice 开始，不先做前端重皮。
