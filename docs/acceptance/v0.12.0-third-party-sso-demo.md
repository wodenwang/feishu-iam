# v0.12.0 第三方接入验收：feishu-iam-sso-demo

## 目标

本验收用于确认 Feishu IAM 第一版成功标准已经通过一个真实第三方 Web 系统闭环验证：第三方系统只对接 Feishu IAM，不直接保存飞书应用密钥；用户通过 Feishu IAM 登录后，第三方后端可以换取 Feishu IAM access token、读取用户信息和权限清单，并在前端与接口侧根据权限结果控制功能展示。

## 验收对象

| 项目 | 内容 |
| --- | --- |
| 第三方系统仓库 | [feishu-iam-sso-demo](https://github.com/wodenwang/feishu-iam-sso-demo) |
| 第三方系统版本 | `v1.0.0` |
| 第三方系统入口 | `http://feishu-iam-sso-demo.dev.tangtring.com` |
| Feishu IAM 入口 | `http://feishu-iam.dev.tangtring.com` |
| 应用 key | `feishu-iam-sso-demo` |
| 回调地址 | `http://feishu-iam-sso-demo.dev.tangtring.com/oauth/callback` |

## 数据流

1. 用户访问 `feishu-iam-sso-demo`。
2. demo 后端生成 OAuth `state`，跳转到 Feishu IAM `/oauth/authorize`。
3. Feishu IAM 按需要跳转飞书登录，登录成功后回调 demo 的 `/oauth/callback`。
4. demo 后端用授权码调用 Feishu IAM `/oauth/token`，换取 Feishu IAM access token。
5. demo 后端调用 Feishu IAM `/oauth/userinfo`，把用户信息写入 httpOnly session。
6. demo 后端调用 Feishu IAM `/api/v1/apps/feishu-iam-sso-demo/me/permissions`，读取当前用户在本应用下的权限组和权限点。
7. demo 前端只调用自身 `/api/me` 和 `/api/permissions`，根据后端返回的权限检查结果展示 2 个权限组、5 个权限点和授权状态。

## 权限目录

demo 使用固定权限目录，权限 key 全部以 `feishu-iam-sso-demo.` 开头：

| 类型 | key | 说明 |
| --- | --- | --- |
| 权限组 | `feishu-iam-sso-demo.group.one` | 组 1：基础访问，包含点 1 和点 2 |
| 权限组 | `feishu-iam-sso-demo.group.two` | 组 2：操作能力，包含点 3 和点 4 |
| 权限点 | `feishu-iam-sso-demo.point.one` | 查看工作台 |
| 权限点 | `feishu-iam-sso-demo.point.two` | 查看报表 |
| 权限点 | `feishu-iam-sso-demo.point.three` | 管理任务 |
| 权限点 | `feishu-iam-sso-demo.point.four` | 导出数据 |
| 权限点 | `feishu-iam-sso-demo.point.five` | 访问高级设置，独立权限点 |

## 安全约束

- 本文档不记录 `client_secret`、developer API token、session secret、access token、authorization code、cookie、密码或其他敏感凭证。
- demo 的敏感配置只允许保存在本地 `.env.local`、远程 `.env` 或密钥管理系统中。
- demo 登录失败、token 换取失败、userinfo 失败和权限读取失败都应展示稳定脱敏错误，不暴露上游响应中的 token、secret、authorization code 或 cookie。
- Feishu IAM 回调地址必须精确匹配，不使用通配符回调。

## v0.12.0 收口结论

`v0.12.0` 可作为 Feishu IAM 第一版真实第三方接入验收版本收口。后续版本应优先处理生产运维可观测性和后台体验问题，例如 `#13` 飞书同步运维控制台；完整 OIDC、SAML、ABAC、资源级权限、HTTPS、高可用和滚动升级仍不属于本版本范围。
