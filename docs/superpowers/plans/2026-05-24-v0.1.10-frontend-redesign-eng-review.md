## 架构决策

1. **只使用 Ant Design 组件体系。**
   - Layout：`Layout.Sider`、`Layout.Header`、`Layout.Content`
   - 登录：独立 `LoginPage`，不使用 username/password 输入。
   - 表格状态：继续复用 `AppTable`，但要支持稳定容器高度和 error/empty/loading/search-empty。

2. **新增小型项目组件，不引入新 design system。**
   - `src/components/BrandLogo/`
   - `src/components/PageHeader/`
   - `src/components/UserMenu/`
   - `src/components/StatusTag/`
   - 必要时扩展 `AppTable` 和 `SearchForm`。

3. **主题 token 集中在 `src/theme/theme.ts`。**
   - 使用 `#0f4c81`、`#0b2e4f`、`#1f67b2`、`#f28c28`、`#eef4f8`。
   - 页面里不散落品牌色，除非是组件内局部 token 映射。

4. **退出登录需要后端 API。**
   - 新增 `POST /api/auth/logout`。
   - 清除 `iam_session` cookie。
   - 前端 mutation 成功后清空 TanStack Query cache 并跳转 `/login`。
   - 失败时保留当前会话，展示可重试错误。

5. **实现阶段优先 subagent-driven-development。**
   - 任务必须按不重叠文件边界拆分。
   - 若并行 worker 产生冲突，由主 agent 集成并修正。

## 数据流

```text
LoginPage
  ├── Production: /api/auth/feishu/start
  ├── Local DEV: /api/dev/feishu/mock-login
  └── callback/error status from query string

AdminLayout
  ├── useCurrentSession()
  ├── BrandLogo + route Menu
  ├── Header env/status tags
  └── UserMenu
       └── useLogout()
            ├── POST /api/auth/logout
            ├── queryClient.clear()
            └── navigate('/login')

ApplicationsListPage
  ├── SearchForm
  ├── PageHeader primary action
  └── AppTable stable state container
       ├── data
       ├── loading
       ├── empty
       ├── error
       └── search-empty
```

## 风险与约束

| 风险 | 处理 |
|---|---|
| 右上角继续抖动 | Header 高度固定 56px，UserMenu 固定高度，长 ID 在 Dropdown 内截断。 |
| 登录页误出现本地账户体系 | 组件测试断言没有 username/password/password input，DEV mock 只在 `import.meta.env.DEV` 可见。 |
| logout API 失败造成会话状态不明 | 失败时不清 cache，不跳转，Modal 内或 message 显示错误并允许重试。 |
| 表格 loading/error 改变高度 | `AppTable` 支持 `minHeight` 和分页占位，状态只替换 body 区。 |
| 768 表格内容溢出 | Table 使用 `scroll.x`，保留关键列，其他列横向可达。 |
| subagent 并行冲突 | 按文件边界拆任务，主 agent 只做集成和冲突修正。 |
