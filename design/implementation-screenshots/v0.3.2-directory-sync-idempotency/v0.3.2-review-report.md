# v0.3.2 pre-landing review

日期：2026-05-28

## 结论

通过。diff 范围符合 `v0.3.2` 补丁边界：只修改同步幂等、目录 active 查询、版本元数据、计划/评审/QA 证据，不新增目录编辑、目录治理、事件 worker、OIDC、SDK/CLI 或本地账号体系。

## 审查点

- `server/src/modules/sync/syncService.ts`
  - full sync 成功后将本次 snapshot 外的 active 部门标记为 disabled，不删除历史记录，避免破坏 `role_department_bindings` 外键。
  - `directoryDepartmentCount` 改为 active 部门计数，避免 `/sync` 页面继续把 disabled stale 部门算入当前投影。
- `server/src/modules/directory/directoryRoutes.ts`
  - 部门树、descendants、部门筛选和用户路径递归均限制 active 部门。
  - 用户列表响应字段不变，disabled 部门路径回落为 `-`。
- `server/tests/sync.test.ts`
  - 覆盖 #27：历史 seed 部门被 disabled 且不出现在默认目录 API。
  - 覆盖连续 full sync 幂等：第二次同步不新增 active 部门。
  - 覆盖合法同名部门：不同父级、不同 ID 的两个“it部”都保留。

## 风险

- 历史角色部门绑定仍可能引用 disabled 部门。本版本保留记录不做治理，符合补丁边界；后续如需清理应另开目录治理版本。
- 如果飞书返回孤儿部门，即 parent 不在 snapshot，该 active 子部门不会出现在树中。这与既有递归树语义一致，没有在本版本扩大处理范围。
