# v1.0.4 OAuth silent SSO 验收记录

## 范围

本版本支持第三方 iframe 场景下的安全 silent SSO，不让第三方 iframe 直接渲染 IAM 登录页。

## 本地验证

已通过：

```bash
pnpm --filter @feishu-iam/api typecheck
pnpm --filter @feishu-iam/api test -- test/oauth.service.spec.ts test/oauth.controller.e2e-spec.ts test/oauth-error.filter.spec.ts
pnpm check
pnpm build
```

结果：

- API 类型检查通过。
- OAuth 定向测试 3 个文件 59 个用例通过。
- 完整检查通过：API 41 个测试文件 475 个用例通过，Admin Web 17 个测试文件 160 个用例通过。
- 生产构建通过：后端 NestJS 构建完成，前端 Vite 构建完成；Vite chunk size warning 为既有构建提示。

覆盖点：

- `prompt=none` + 已登录 IAM SSO session：302 到第三方 `redirect_uri`，包含 authorization code 和原始 `state`。
- `prompt=none` + 未登录：302 到第三方 `redirect_uri`，包含 `error=login_required` 和原始 `state`，不渲染 IAM 登录页。
- `prompt=none` + 应用未启用 silent SSO：302 到第三方 `redirect_uri`，包含 `error=unauthorized_client` 和原始 `state`。
- `redirect_uri` 不匹配仍走安全失败，不向不可信地址回跳。
- 普通 OAuth 登录成功后设置 httpOnly、Secure 的 IAM SSO session cookie。
- 普通 OAuth 登录、token exchange、userinfo 和 revoke 相关既有测试不回退。
- OAuth HTML 错误页设置 `X-Frame-Options: DENY` 和 `Content-Security-Policy: frame-ancestors 'none'`。

## 安全结论

- Base Portal 不传递 token、cookie、authorization code 或 secret。
- IAM 不要求 Base Portal 代理第三方鉴权。
- silent SSO 不放宽 `redirect_uri` exact match、client、scope、authorization code TTL 或一次性使用校验。
- 文档和测试未记录真实 secret、token、cookie、authorization code 或 env 值。
