# v0.1.8 Review Report - Real Feishu Auth

日期：2026-05-24
范围：落地前代码审查。

## 结论

通过，已修复审查中发现的问题。

## Findings

### Finding 1: 生产 cookie 缺少 Secure

风险：真实飞书登录会创建 OAuth state cookie 和 IAM session cookie。生产 HTTPS 部署下缺少 `Secure` 会降低 cookie 传输边界。

修复：

- `main.ts` 在 `NODE_ENV=production` 时传入 `secureCookies: true`。
- `authRoutes.ts` 的 state cookie 和 session cookie 支持 `Secure`。
- `auth.real-feishu.test.ts` 增加生产 cookie 模式断言。

### Finding 2: route fake 覆盖不足以证明 adapter 解析

风险：只有 route fake adapter 测试会漏掉真实飞书 token response 和 user_info response 的解析错误。

修复：

- 新增 `server/tests/realFeishuAuthAdapter.test.ts`。
- 覆盖授权 URL 生成、token exchange、user info 映射和飞书错误码失败路径。

### Finding 3: 敏感信息边界

风险：用户已提供真实飞书 App Secret，任何文档、测试 fixture、部署报告或 git diff 都不得落真实值。

修复：

- 文档只保留 `<replace-me>` 占位符。
- secret grep 未发现真实 secret；只命中占位符和字段名。

## 剩余风险

- 真实飞书 smoke 仍依赖目标环境 `.env` 和飞书后台 redirect URI 完全一致。
- 当前版本不做 refresh token 持久化，session 到期后用户需重新走飞书登录。
